Pending confirmation triage

页面定位

本页收束 Mars Wiki 中的 待确认 与源码疑点。它不是“未完成页面”清单，而是把已经写入各页的不确定性分级：哪些需要运行验证，哪些需要外部数据或文献，哪些只是源码边界尚未集中复核。

处理原则：

  • 能由源码直接确认的内容，不再保留为旧缺页或旧登记占位。
  • 不能由源码单独确认的内容，保留 待确认，并说明需要哪类证据。
  • 不把推断写成事实；需要运行、构建日志、外部数据版本或维护者判断的内容继续显式标注。

优先级分类

等级 含义 处理方式
P0 可能影响数值结果、守恒性、数组索引或 restart 复现 优先用最小运行案例或维护者确认。
P1 影响物理解释、外部数据版本或诊断可比性 需要文献、数据文件版本或实验配置佐证。
P2 影响构建、并行行为或工具链复现 用 build log、脚本和实际工具运行核验。
P3 遗留变量、注释、命名或未使用导入 保留在页面内，批量清理时再处理。

topmons_mod qbar_number_pbl 等 PBL 平均量的单位和数浓度解释仍需确认。 追踪调用方输入单位，或用诊断输出核对量纲。

P1 物理参数和外部数据

页面 疑点 需要的证据
reaction-rates 热层化学反应速率常数的外部版本来源尚未完全归档。 对照 chemthermos_reactionrates.def 的历史来源、JPL/UMIST 或项目文档。
radiation 与辐射相关文件页 气溶胶、辐射表或典型光学参数的推荐范围不能仅由源码判断。 数据表版本、运行配置和基准实验说明。
aeropt_mod / aeroptical ASCII 光学性质表的具体文件版本、半径范围和波长范围需要与 datadir 实物对应。 实际 datadir 文件清单和表头检查。
soilwater 与 soil 土壤吸附、地下冰和热惯量常数的物理来源需要外部资料补强。 对照配置、论文或维护者说明。
simu_MCS MCS 分箱文件版本、Ls-sol 换算和观测高度坐标约定需与实际观测产品一致。 观测文件元数据和一组已知对比案例。

P2 构建和边界契约

页面 疑点 需要的证据
common-dependencies temps_mod 串行/并行版本选择取决于构建目标。 实际 make/cmake 编译日志或目标配置。
xios-config XIOS server/client 模式、重复 field id 和实际输出开关组合需要运行验证。 运行日志、iodef.xml 和 XIOS 输出文件。

startarchive2icosa XIOS/MPI 编译链和硬编码文件名对现代环境的兼容性。 本机编译和一次 start archive 转换运行。
mod_const_mpi / parallel_lmdz 1D XIOS、MPI、OpenMP 条件路径的实际组合。 1D 不同编译选项下的运行日志。

后续处理规则

  1. 新增 待确认 时必须说明它需要源码、运行、外部数据、构建日志还是维护者判断。
  2. 仅凭“看起来合理”不得移除 待确认。
  3. 若运行验证通过，应在对应页面补一句验证证据，并在本页把条目移到“已确认”或删除。
  4. 若发现真实源码问题，应新建维护任务或 issue，不在 Wiki 中直接修改源码。
