testphys1d 专用配置

路径

LMDZ.MARS\deftank\run.def.1d                  (91 行, 1D 专用)
LMDZ.MARS\deftank\callphys.def.GCM6          (245 行, 物理参数开关, 1D/3D 共用)
LMDZ.MARS\libf\phymars\dyn1d\testphys1d.F90  (289 行, 1D 主程序)
LMDZ.MARS\libf\phymars\dyn1d\init_testphys1d_mod.F90 (850 行, 1D 初始化)

类型

配置——控制 testphys1d (1D 物理列模式) 的初始条件、时间积分和物理参数化。

作用

testphys1d 是 LMDZ.MARS 的 1D 列模式，用于在单一大气柱中离线运行全部物理过程（辐射、湍流、凝结、微物理、尘埃、水循环等），不包含 3D 动力学。

源码依据（testphys1d.F90:28-46 注释）：

PROGRAM useful to run physical part of the martian GCM in a 1D column

Can be compiled with a command like (e.g. for 25 layers) "makegcm -p mars -d 25 testphys1d" It requires the files "testphys1d.def" "callphys.def" and a 'run.def' file (containing a "INCLUDEDEF=callphys.def" line) and a file describing the sigma layers (e.g. "z2sig.def")

用途：

  • 测试单个物理参数化（辐射、微物理、土壤等）在给定条件下的行为。
  • 在无需运行完整 3D 动力学的情况下快速验证物理改动。
  • 生成 1D 气候平衡解（如极冠、水循环的日-季平均剖面）。

文件加载链

源码依据（testphys1d.F90:110-146）：

testphys1d.e
  ├── 读取 "run.def" (必须, 内含 INCLUDEDEF=callphys.def)  ← 报错停止如果缺
  ├── 读取 "traceur.def" 获取示踪物名和数量              ← 报错停止如果缺
  ├── 如果 startfiles_1D=true:
  │     ├── start1D.txt (ASCII 初始廓线, 可选)
  │     └── startfi.nc (NetCDF 物理初始场, 可选)
  └── 如果不存在 startfi.nc → 写一个新的 (physdem0/physdem1)

被读取的位置

读取例程 文件 说明

1D 与 3D 的差异（源码依据：testphys1d.F90:199-232,234-260）：

  • 风场：不使用 Coriolis 力，而是向地转风 (gru, grv) 做松弛，时间尺度 1e4 秒（第 206-207 行）。
  • 水示踪物：ctrl_h2ovap / ctrl_h2oice 为真时，物理计算的倾向被覆盖或松弛到 qref_h2ovap / qref_h2oice 廓线（第 214-232 行）。
  • 地表气压：物理倾向 dpsurf 直接加到 psurf（3D 由动力学保证守恒）。

影响的计算流程

主题 影响
辐射 callrad + callnlte + tauvis 共同决定 1D 大气的日变化辐射加热
尘埃循环 dustbin, doubleq, lifting, rdstorm, topflows 开启完整尘埃物理
水循环 water + microphys + caps + ctrl_h2ovap/ctrl_h2oice 决定水循环闭环方式
CO₂ 循环 callcond + co2ice 初始值 + psurf 决定 CO₂ 凝结/升华
土壤 callsoil + inertia + 土壤分层 (comsoil_h) 决定地表热惯量
对流/边界层 calldifv + calladj + calltherm + callrichsl + u/v 决定垂直混合
重力波 calllott + calllott_nonoro 影响高层大气耗散

ichoice 温度初始廓线选项（1-8）

源码依据（run.def.1d:68-87）：

ichoice 类型 说明
1 常数 T = tref
2 Savidjari 类 Seiff 但 dT/dz = cte
3 Lindner 极区廓线
4 inversion 逆温
5 Seiff 标准廓线（基于 Viking 进入数据），默认推荐
6 常数 + 高斯扰动 (levels) 在 pic (层数) 处加扰动，宽度 largeur，幅度 hauteur
7 常数 + 高斯扰动 (km) 在 pic (km) 处加扰动
8 ASCII 读取 从文件 profile_temp 逐层读取一列温度值

复现要求

  • run.def 必须物理存在且含 INCLUDEDEF=callphys.def 行（源码依据：testphys1d.F90:114-120）：否则 getin 无法解析 callphys.def 字段，程序停止。
  • traceur.def 必须存在（源码依据：init_testphys1d_mod.F90:180-186）：第一行为示踪物数量 nq，后续每行一个示踪物名。同位素可用 tname tnom_transp 格式声明父-子关系。
  • day_step 必须整除 output_per_sol（源码依据：callphys.def.GCM6:7 与 time_phylmdz_mod）：否则输出步与物理步不匹配。
  • ndt 与 day_step 决定总时间步数（源码依据：init_testphys1d_mod.F90）：总步数 = ndt * day_step。
  • hybrid=.true. 与 3D 一致（源码依据：run.def.1d:63）：1D 必须使用与 3D 相同的垂直坐标定义（来自 z2sig.def）。
  • 水循环松弛设置（源码依据：run.def.1d:54-61）：
    • ctrl_h2ovap=.true. + relaxtime_h2ovap<0 → 强制到 profile_ref_h2o_vap 文件廓线。
    • ctrl_h2ovap=.true. + relaxtime_h2ovap≥0 → 指数松弛。
    • ctrl_h2ovap=.false. → 完全由物理决定。
  • ichoice=5 (Seiff) 是标准初始廓线（源码依据：run.def.1d:77 注释：standard profile, based on Viking entry）。
  • 复现风险：1D 风场采用向地转风 (gru, grv) 的 1e4 秒松弛而非 3D Coriolis 力（源码依据：testphys1d.F90:199-208 注释："For some tests: No coriolis force at equator"）。此简化使得 1D 高层风场可能与 3D 不同。
  • 复现风险：psurf 在 1D 中每时间步由物理倾向 dpsurf 直接累加（testphys1d.F90:254），没有 3D 动力学的守恒修正；长 run 可能出现质量漂移。
  • 复现风险：callphys.def.GCM6 中 nonoro_gwd_* 参数在 1D 仍可被调用（testphys1d.F90:194 调 physiq），但其参数化假设水平不均匀，1D 结果物理解释需谨慎。
  • 复现风险：当 startfiles_1D=.false. 且无 startfi.nc 时，程序调用 physdem0/physdem1 自动生成一个 startfi（源码依据：testphys1d.F90:152-159），其内部字段值由 1D 参数决定，复现时若修改 albedo/inertia/tauvis 必须重新生成 startfi。

已核验输入格式与运行边界

  • profile_ref_h2o_vap / profile_ref_h2o_ice 不从 datadir 搜索，而是在当前运行目录按字面文件名打开；init_testphys1d_mod.F90:794-801 和 823-830 逐层读取 nlayer 行，文件缺失或行数不足都会停止。
  • start1D.txt 的 ASCII 格式可由 writerestart1D_mod 写出的 restart1D.txt 反推；复用时仍需保证 tracer 顺序、nslope 和 nlayer 一致。
  • profile_temp 文件只在 ichoice>=8 时需要，profile_temp_mod.F90:184-190 按运行目录下的字面文件名逐行读取 nlev=nlayer+1 个温度值；样例文件未随 deftank 提供时，应由用户运行目录自行放置。
  • ndt 在 init_testphys1d_mod.F90:74 声明为 integer，getin("ndt",ndt) 读取后先表示 sol 数，再在 init_testphys1d_mod.F90:309 乘以 day_step 转为总物理步数；配置中应写整数 sol 数。
