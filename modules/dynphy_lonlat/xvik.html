文件定位

xvik.F 定义交互式后处理程序 xvik。它逐个读取用户给出的 diagfi NetCDF 文件，提取 Viking Lander 1、Viking Lander 2 和 InSight 站点附近的地表压力，按真实站点高度做静力订正，同时输出全球平均地表压、南北极 CO2 冰等效压力、每日平均压力以及最多 6 阶的季节谐波拟合。它依赖 readhead_NC 从 diagfi 文件读取网格头信息和地表位势，再用 ps/temp/co2ice/Time 做诊断。

输出

prestot_yearN 当前工作目录 文本 全球平均地表压、南北极 CO2 冰等效压力和总 CO2 等效压力。

共享状态与副作用

  • 交互式从标准输入读取路径、输出时间轴和多个 diagfi 文件名。
  • 打开并关闭多个文本文件；pathsor 变量被读取但未拼接到 OPEN(file=...)，实际输出在当前工作目录。
  • 修改 pi、pa、preff 等模块变量。
  • 使用固定站点经纬度和高度：Viking 1、Viking 2、InSight；站点高度先按 3.73 转为位势。
  • 第一条时间记录不输出站点压力，因为 firstcal 初始为 .true.，站点循环在 .NOT.firstcal 后执行。

核心逻辑

  1. 设置 pi、pa=20、preff=610.，定义火星年长度 unanj=669.。
  2. 初始化三站点经纬度和真实高度，打印站点信息并把经纬度转为弧度。
  3. 从标准输入读取 diagfi 目录、输出目录、时间轴选项和第一个文件名。
  4. 对每个非空文件名，打开 pathchmp/nomfich.nc，调用 readhead_NC 读取头信息和 phis，再初始化 GCM 常数/几何。
  5. 读取 Time 维长度和时间数组。
  6. 对三个站点，利用 rlonu/rlatv 找到包围站点的 2x2 GCM 网格，计算双线性权重 zw(0:1,0:1,iv)。
  7. 用权重插值得到每个站点的模型地表位势 phisim。
  8. 对每个时间记录读取 ps、temp 或 temp7、co2ice。
  9. 计算当前 sol 和火星年编号；跨年时打开新的站点输出文件和全球压力输出文件，按 Time_unit 写表头。
  10. 用面积加权得到全球平均地表压；按南北半球累加 co2ice 并乘 g 得等效压力。
  11. 对每个站点，在 2x2 网格上对 log(ps) 做双线性插值，得到 GCM 高度压力 zp1；用第 7 层温度估计 gh=constR*t7，将压力订正到真实站点高度 zp2。无温度时使用 10 km 标高的备用尺度高度。
  12. 写逐时站点文件和 prestot_yearN。
  13. 所有 diagfi 文件处理完后，读取逐时站点文件，按 floor(sol) 分组写日平均文件。
  14. 读取日平均文件；若年内 Ls 覆盖到 350 度以上，计算 0-6 阶 Fourier 系数，并重建 1-669 sol 的谐波压力序列。

伪代码

read input directory, output directory, time axis option
read first diagfi basename
while basename is non-empty:
    open basename.nc
    readhead_NC -> day0, phis, constR, geometry
    read Time
    locate 2x2 GCM cells around Viking1, Viking2, Insight
    interpolate model topography at sites
    for each Time record:
        read ps, temp/temp7, co2ice
        open new yearly output files if needed
        compute planetary mean pressure and CO2 ice equivalents
        if not first record:
            interpolate log(ps) around each site
            correct pressure from model altitude to true site altitude
            write station pressure files
        increment year at sol ~= 669
    close NetCDF
    read next basename

for each produced yearly station file:
    group rows by integer sol and write diurnal means

for each diurnal file:
    if year is complete enough:
        compute Fourier coefficients n=0..6
        write coefficients and reconstructed annual curve

参与的主题流程

主题 参与方式
Viking/InSight 压力后处理 从 diagfi 网格输出生成站点压力、日平均和谐波拟合。
CO2 库存诊断 输出全球平均地表压、南北极 CO2 冰等效压力和总 CO2 等效压力。
util/xvik 拟合链 util/xvik/fit_Iceinertia_MONSicedepth.F 的配置和 README 说明它消费 xvik 输出。

写法特点

  • 固定格式 Fortran，主程序后直接附带三个辅助例程。
  • 输出路径变量 pathsor 被读取但没有实际用于文件名。
  • sol2ls 使用 year_day=669.，ls2sol 使用 year_day=668.6d0，两者常数略不一致。
  • temp 缺失时不终止，而是尝试 temp7；若也不存在，则温度置零并用 10 km 标高备用公式。
  • 谐波拟合只在最后一个日平均点 Ls>350 时执行，否则写 “not computed because year is not complete”。

复现要点

  • 输入 diagfi 文件必须至少有 Time、ps、co2ice，并应有 temp 或 temp7。
  • readhead_NC 要求同一文件中有 controle、经纬度、面积、phisinit、aps/bps。
  • 需要从标准输入依次提供目录、输出时间轴和一个或多个文件名。
  • 输出文件实际写到当前工作目录；运行前应切到希望保存结果的目录，或在源码/脚本层处理路径。

待确认

  • pathsor 未使用是否为 bug 或历史残留。
  • sol2ls 和 ls2sol 火星年常数不一致是否会影响 Time_unit==2 的日平均分组。
  • util/xvik/fit_Iceinertia_MONSicedepth.F 文档提到 xpsol/xprestot，而当前 xvik.F 写 ps_VL*_yearN/prestot_yearN，两者文件名约定是否已分叉。
  • firstcal 跳过首个站点压力输出的原因需结合诊断时间轴约定确认。
