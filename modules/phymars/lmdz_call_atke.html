文件定位

lmdz_call_atke.F90 定义 lmdz_call_atke 模块，是 ATKE（Atelier TKE）1.5 阶湍流闭合方案的调用接口模块。它被 vdifc_mod.F 调用，负责协调 lmdz_atke_exchange_coeff 模块中的核心计算，并实现可选的显式风切变预测以支持前向-后向时间步进。

本模块是 ATKE 方案的入口层，不包含核心物理计算，而是组织计算流程和处理时间步进策略。

核心逻辑

call_atke

  1. 第一次调用 atke_compute_km_kh：

    • 使用当前风场计算初始 Km/Kh 和 TKE

  2. 可选的显式风切变预测（iflag_num_atke = 1）：

    • 计算第一层风速 wind1
    • 对纬向风和经向风分别调用 atke_explicit_prediction
    • 使用预测的风场再次调用 atke_compute_km_kh
    • 实现前向-后向时间步进策略

atke_explicit_prediction

  1. 计算 "big K"：

    • 将界面交换系数转换为层交换系数
    • K_big = rg * K_in / dp * rho²

  2. 预测变量：

    • 对 2 ≤ k ≤ nlay-1：使用中心差分格式
    • 对 k = 1：使用地表通量边界条件
    • 对 k = nlay：使用零通量顶部边界条件

伪代码

call_atke(dtime, ngrid, nlay, cdrag_uv, cdrag_t, u_surf, v_surf,
          temp_surf, wind_u, wind_v, temp, qvap, play, pinterf,
          zlay, zinterf, tke, Km_out, Kh_out):

    ! 1. 第一次计算 Km/Kh
    atke_compute_km_kh(wind_u, wind_v, ..., tke, Km_out, Kh_out)

    ! 2. 可选的显式风切变预测
    if iflag_num_atke == 1:
        wind1 = sqrt(wind_u(:,1)² + wind_v(:,1)²)
        atke_explicit_prediction(wind_u, Km_out, u_surf, wind_u_predict)
        atke_explicit_prediction(wind_v, Km_out, v_surf, wind_v_predict)

        ! 3. 使用预测风场重新计算 Km/Kh
        atke_compute_km_kh(wind_u_predict, wind_v_predict, ..., tke, Km_out, Kh_out)

atke_explicit_prediction(ngrid, nlay, rg, rd, dtime, pinterf, play,
                         temp, wind1, x_in, K_in, x_surf, cdrag, x_predict):

    ! 1. 计算 K_big
    for k = 2 to nlay-1:
        rho = pinterf(k) / rd / avg(temp)
        K_big(k) = rg * K_in(k) / dp * rho²

    ! 2. 预测变量
    for k = 2 to nlay-1:
        ml = dp / rg
        x_predict(k) = x_in(k) - dtime/ml * (-K_big(k+1)*x_in(k+1) + ...)

    ! 3. 地表边界条件
    ml = dp(1) / rg
    F1 = -rho(1) * wind1 * cdrag * (x_in(1) - x_surf)
    x_predict(1) = x_in(1) - dtime/ml * (-K_big(2)*(x_in(2)-x_in(1)) - F1)

    ! 4. 顶部边界条件
    ml = dp(nlay) / 2 / rg
    x_predict(nlay) = x_in(nlay) + dtime/ml * K_big(nlay) * (x_in(nlay)-x_in(nlay-1))

参与的主题流程

主题 参与方式
边界层湍流 vdifc_mod.F 调用本模块作为 ATKE 方案入口
TKE 时间步进 本模块实现可选的前向-后向时间步进策略
垂直扩散 本模块协调 ATKE 方案的计算流程

写法特点

  • .F90 自由格式模块，代码简洁（162 行）。
  • 作为调用接口层，不包含核心物理计算。
  • 实现了可选的显式风切变预测（iflag_num_atke = 1）。
  • atke_explicit_prediction 使用中心差分格式和地表通量边界条件。

复现要点

  • 必须确保 lmdz_atke_turbulence_ini 模块中的配置参数已正确初始化。
  • iflag_num_atke 的选择会影响时间步进策略。
  • atke_explicit_prediction 中的 "big K" 计算需要正确的密度和压力梯度。
  • 地表通量使用 rho * wind * cdrag * (x - x_surf) 公式。

待确认

  • iflag_num_atke = 1 的前向-后向时间步进策略是否对所有条件都稳定。
  • atke_explicit_prediction 中 "big K" 的计算公式是否正确。
  • cdrag_t（热量拖曳系数）在本模块中未使用，需确认是否为遗留参数。
