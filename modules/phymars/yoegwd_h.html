文件定位

yoegwd_h.F90 定义 yoegwd_h 模块，是地形重力波拖曳链的共享调参状态容器。它本身不执行计算，也不读取配置文件；所有变量都在模块内以 save 或模块变量形式保存，并通过 OpenMP THREADPRIVATE 声明成为每个线程私有的地形 GW 参数副本。实际数值由 sugwd 在 calldrag_noro 首调用时写入，随后被 orosetup、gwstress_mod、gwprofil_mod 和 orodrag_mod 读取。

共享状态与副作用

yoegwd_h 的全部变量都是共享模块状态，且都带有 OpenMP THREADPRIVATE 语义。源码没有提供显式初始化例程或默认值；正常路径依赖 calldrag_noro_mod.F90 首调用时先调用 sugwd，再进入 ORODRAG/OROSETUP/GWSTRESS/GWPROFIL 链。复现时不能只检查本文件声明，还必须确认 SUGWD 已经在同一线程中执行过，否则这些 threadprivate 变量可能保留未定义或旧值。

核心逻辑

  1. 声明 12 个 real,save 地形 GW 参数，分成四组 THREADPRIVATE。
  2. 声明 1 个 integer 层索引 NKTOPG，同样设为 THREADPRIVATE；它不是 real,save 参数。
  3. 不执行任何计算；参数写入由 sugwd.F90::SUGWD 完成，下游例程直接 use yoegwd_h, only: ... 读取。

伪代码

module yoegwd_h:
    declare saved real tuning constants for orographic GW drag
    mark each group as OpenMP threadprivate
    declare integer NKTOPG
    mark NKTOPG as OpenMP threadprivate

runtime expectation:
    SUGWD initializes these variables before the orographic GW chain reads them

参与的主题流程

主题 参与方式
地形重力波 / 次网格地形拖曳 保存 SUGWD 初始化的调参常量和安全阈值，供 OROSETUP 预处理、GWSTRESS 应力初始化、GWPROFIL 应力廓线重算和 ORODRAG wake drag 使用。

写法特点

  • 这是 Fortran 90 “头文件式”共享状态模块，只有声明，没有 contains 区段。
  • 所有 real 参数都带 save，保证模块状态跨调用保留；NKTOPG 是模块变量，本身也具有模块生命周期。
  • 每组变量都用 !$OMP THREADPRIVATE(...) 标注。并行运行时，每个线程需要自己的初始化副本，不能假设 master 线程写入后自动覆盖其他线程副本。
  • 本文件没有默认赋值；实际默认数值集中在 sugwd.F90 行 45-76。

复现要点

  • 使用地形 GW 链前，必须确认 calldrag_noro 的 firstcall 路径已调用 SUGWD。
  • 修改 GKDRAG 时要同步检查 GKDRAGL=4.*GKDRAG 的派生关系。
  • NKTOPG 的有效值依赖 SUGWD 对半层 sigma 坐标 sigtest 的扫描结果。
  • OpenMP 运行中要关注 threadprivate 初始化顺序；本模块没有 COPYIN 或集中广播逻辑。

待确认

  • GHMAX 在当前地形 GW 链中的实际消费位置未在本任务内找到，后续若复现最大地形高度限制，需要继续沿调用链核验。
  • THREADPRIVATE 变量在所有线程上的初始化时机需结合物理驱动的并行区边界确认；本页只记录源码声明和直接调用关系。
