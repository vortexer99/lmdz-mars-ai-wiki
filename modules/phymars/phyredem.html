libf/phymars

文件定位

phyredem.F90 是物理 start/restart 文件写出模块。它通过 iostart 的 open_restartphy、put_var、put_field、close_restartphy 创建或追加 startfi.nc/restartfi.nc 物理状态文件：physdem0 写控制表、网格和静态地表字段；physdem1 写带时间的表面、土壤、tracer、尘埃、云分数、重力波、吸附、古气候和 tracer 质量修正状态。

读入端对应 phyetat0_mod。复现一次 restart 续跑时，phyredem 写出的字段名、是否带时间轴、以及哪些开关控制字段存在，都会决定下一次 phyetat0 能恢复哪些物理记忆。

共享状态与副作用

  • physdem0 和 physdem1 都会打开、写入并关闭由 filename 指定的 NetCDF 文件；这是本模块的主要副作用。
  • physdem0 不只使用实参，也读取多个 module 级物理常数和地表共享状态来构造 controle 与静态字段。
  • physdem1 的字段集合受运行开关控制：CLFvarying、dustscaling_mode==2、dustinjection>0、calltherm、calllott_nonoro、adsorption_soil、paleoclimate、call_mass_fixer_dyn。
  • physdem1 会打印跳过 h2o_vap、hdo_vap、co2_ice surface tracer 的诊断信息。
  • watercaptag 是 logical module array，写出前被转换为 0/1 real field。

核心逻辑

  1. physdem0 打开目标文件，构造长度为 iostart::length 的 tab_cntrl 控制表。
  2. 控制表写入网格总点数、垂直层数、初始日/时刻、火星物理常数、行星轨道参数、PBL 参数、极冠反照率/发射率、尘埃平均光学厚度和土壤体积热容量。
  3. physdem0 写 controle_descriptor、soildepth、经纬度、网格面积、地表位势、裸地反照率、次网格地形统计量、土壤热惯量、粗糙度、水冰标签和坡面分布，然后关闭文件。
  4. physdem1 打开目标文件，并首先写 Time；iostart 用该变量推进/设置后续 time-dependent 字段的时间索引。
  5. physdem1 写水冰库、可选常年 CO2 冰、表面温度、土壤热惯量、土壤温度、反照率第一谱段、发射率和 PBL q2。
  6. 根据开关写云分数、尘埃缩放、尘埃辐射调整、dtau、热羽流 wstar。
  7. 扫描 tracer_mod::noms 找水汽/水冰和 HDO 汽/冰 tracer；随后逐 tracer 写地表 qsurf，但跳过已由其他字段表示或无地表储量意义的 tracer。
  8. 根据开关写非地形随机 GW 记忆、地热通量、吸附土壤水字段、古气候地下冰字段和 tracer 质量修正标量。
  9. 关闭目标文件。

伪代码

physdem0(filename, ...)
  open_restartphy(filename)
  tab_cntrl = 0
  fill tab_cntrl with grid size, date, constants, orbit, PBL, caps, dust, soil
  put_var controle and descriptor
  put_var soildepth
  put_field longitude, latitude, area
  put_field static surface/topography/roughness/water tag fields
  put_var def_slope
  put_field subslope_dist
  close_restartphy

physdem1(filename, ...)
  open_restartphy(filename)
  put_var Time first
  put_field main time-dependent surface and soil states
  if option enabled: put cloud, dust, thermal, GW, adsorption, paleoclimate fields
  for each tracer name:
    skip h2o_vap/hdo_vap when matching ice tracer exists
    skip co2_ice because CO2 ice is included in qsurf(:,igcm_co2)
    put_field surface tracer using trim(name)
  if call_mass_fixer_dyn:
    put_var mass_predyn_* for existing target tracers
  close_restartphy

参与的主题流程

主题 参与方式
物理 restart 输出 写 restartfi.nc，供下一段运行的 phyetat0_mod 读取
1D startfi 生成 testphys1d.F90 在缺少 start 文件时调用 physdem0/physdem1 写 startfi.nc
newstart 初始场生成 newstart.F 写动力 restart 后调用本模块写物理 restartfi.nc
次网格坡面 写 def_slope 与 subslope_dist，供下一次初始化恢复坡面分布
水循环 / HDO 写 watercap、地表 water/HDO tracer 和可选土壤吸附水字段
CO2 循环 / 古气候 写 perennial_co2ice、CO2 地表 tracer 关系和古气候地下 CO2 冰深度
尘埃循环 写 tauscaling、可选 dust_rad_adjust_prev/next 和 dtau
非地形重力波 写 du_nonoro_gwd、dv_nonoro_gwd 记忆项
tracer 质量修正 写 mass_predyn_* 标量，供下一次 phyetat0_mod 恢复质量修正参考值

写法特点

  • physdem0 的 airefi 实参没有被源码使用，实际写出的面积来自 geometry_mod::cell_area；局部变量 ig 也未使用。
  • physdem1 的 phystep 实参没有被源码使用，时间轴只由 time 写入。
  • physdem1 中 paleoclimate 被 use paleoclimate_mod 导入两次，属于源码重复导入但不改变行为。
  • albedo 实参是 (ngrid,2,nslope)，但写出 albedo 字段时只写 albedo(:,1,:)。
  • 古气候 h2oice_depth/co2ice_depth/coef_ssdif 在 physdem1 中未带 time 实参写出，而 perennial_co2ice 带 time 写出；读端仍在 paleoclimate 分支中读取这些字段。

复现要点

  • 写 time-dependent 字段前必须先写 Time；iostart 的 timeindex 由 put_var("Time",...) 更新。
  • 是否能从 restart 恢复某个可选状态，取决于写出时对应开关是否开启。例如 CLFvarying 控制 totcloudfrac，calltherm 控制 wstar，calllott_nonoro 控制随机 GW 记忆项。
  • co2_ice 地表 tracer 不单独写出；源码注释说明它已加入 qsurf(:,igcm_co2)，因此读端需要按 CO2 地表状态路径恢复。
  • 对 h2o_vap / hdo_vap，若没有对应 ice tracer，源码会改写为 h2o_ice / hdo_ice 字段名；若 ice tracer 存在，则跳过 vapor surface tracer，等待 ice tracer 自己写出。
  • watercaptag 写成 0/1 real field，读端再按阈值转换回 logical。
  • mass_predyn_* 只为存在的 8 个目标 tracer 写出；缺少对应 tracer 索引时不会产生该标量。

待确认

  • physdem0 的 airefi 实参和 comsoil_h::inertiedat 导入未被直接使用，需确认是否为历史接口遗留。
  • physdem1 的 phystep 实参未使用，需确认是否保留给旧版接口兼容。
  • 古气候地下冰字段不带 time 写出，而 phyetat0_mod 读取时传入 indextime；需以 iostart 当前维度分派行为为准复现。
