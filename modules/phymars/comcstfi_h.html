文件定位

comcstfi_h.F90 定义 comcstfi_h，是 phymars 中集中保存行星半径、重力、气体常数、比热、平均摩尔质量和自转角速度的共享物理常数模块。它本身不读取配置文件，也不做物理计算；主要职责是在初始化阶段接收上游给定的常数，并让辐射、沉降、边界层、云微物理、坡面和诊断等下游方案通过 use comcstfi_h 读取同一套常数。

这个模块的复现意义在于：很多物理公式只在下游文件中出现，但公式中的 g、r、cpp、rcp、rad、pi 等都来自这里。若初始化路径不同，常数来源也可能不同，例如 GCM 主路径由 phys_state_var_init_mod 调用 init_comcstfi_h 初始化一部分常数，而 tabfi.F 和 1D 初始化路径会直接写入 rad/g/mugaz/omeg/rcp 等模块变量。

共享状态与副作用

  • rad、g、r、cpp、rcp、mugaz、omeg 都是 SAVE 模块变量。
  • 源码用 OpenMP THREADPRIVATE 分三组声明：rad/g、r/cpp/rcp、mugaz/omeg。并行复现时需要确认每个线程已获得正确副本。
  • init_comcstfi_h 只写 rad、cpp、g、r 和 rcp；它不写 mugaz 或 omeg。
  • tabfi.F 在读取或默认生成 tab_cntrl 时会直接写 rad、omeg、g、mugaz、rcp，并把这些值回传到 p_rad/p_omeg/p_g/p_mugaz/p_daysec。
  • 1D dyn1d/init_testphys1d_mod.F90 也直接写 rad、omeg、g、mugaz、rcp、r、cpp。
  • 本模块没有文件 I/O、没有分配内存、没有显式结束/释放例程。

核心逻辑

  1. 编译/加载模块时，pi 作为 parameter 由 acos(-1.d0) 固定。
  2. 模块暴露一组 SAVE 常数变量，供下游 use comcstfi_h, only: ... 读取。
  3. GCM 初始化路径中，phys_state_var_init_mod.F90 行 46 导入 init_comcstfi_h，行 118 调用 init_comcstfi_h(prad,pcpp,pg,pr)。
  4. init_comcstfi_h 将 prad、pcpp、pg、pr 分别赋给 rad、cpp、g、r。
  5. init_comcstfi_h 最后计算 rcp=r/cpp，保证 rcp 与本次传入的 r 和 cpp 一致。
  6. mugaz 和 omeg 必须由其他初始化路径赋值；源码证据包括 tabfi.F 行 114-119、182-186，以及 1D 初始化行 133-140。

伪代码

module comcstfi_h:
    define pi = acos(-1)
    keep SAVE constants:
        rad, g, r, cpp, rcp, mugaz, omeg
    mark these constants THREADPRIVATE

init_comcstfi_h(prad, pcpp, pg, pr):
    rad = prad
    cpp = pcpp
    g = pg
    r = pr
    rcp = r / cpp

写法特点

  • 这是自由格式 .F90 模块，但变量命名保留 LMD 历史公共常数风格。
  • 模块注释写明“理想情况下所有 module variables 应该 protected”，但当前源码没有使用 PROTECTED 属性；任何 use comcstfi_h 且未限制写保护的代码理论上可写这些变量。
  • pi 是唯一编译期常量；其余常数依赖初始化顺序。
  • mugaz 和 omeg 与 rad/g 同属本模块状态，但不在 init_comcstfi_h 中初始化，这是复现时最容易混淆的点。

复现要点

  • 复现 GCM 主路径时，应确认 phys_state_var_init 传入的 prad/pg/pr/pcpp 来源，并确认 tabfi 或其他路径是否随后覆盖了 rad/g/rcp/mugaz/omeg。
  • rcp 既可能由 init_comcstfi_h 计算，也可能由 tabfi 从 tab_cntrl(tab0+9) 读取；两者不一致会影响位温、Exner 因子和垂直插值相关计算。
  • mugaz 不由 init_comcstfi_h 赋值，但 dust_coagulation_mod、iniwrite.F、phyetat0_mod 等会读取它；需要确认启动文件或 1D 初始化已设置该值。
  • OpenMP THREADPRIVATE 意味着共享常数在多线程环境中不是普通全局单例；复现线程初始化问题时，应检查线程私有副本是否完成同步或初始化。
  • tabfi.F 的默认 Mars 常数为 rad=3397200.、daysec=88775.、g=3.72、mugaz=43.49、rcp=.256793，但这些默认值只代表 tabfi 的 nid=0/默认路径，不应覆盖所有实验配置。

待确认

  • GCM 主路径中 mugaz 和 omeg 是否总是在 phys_state_var_init 后由 tabfi 或等价流程赋值，需结合完整初始化调用顺序确认。
  • THREADPRIVATE 变量在当前并行框架下的拷贝/广播策略未在本文件内体现，需要查看 OpenMP 初始化约定。
