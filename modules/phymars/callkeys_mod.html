文件定位

callkeys_mod.F90 定义 callkeys_mod 模块（第 1 行），集中保存物理包运行时开关、少量辐射/尘埃参数和水循环/CO2 云控制量。该文件本身没有子程序、函数、use 依赖或文件 I/O；它的职责是声明 SAVE 模块变量，并把几乎全部变量声明为 OpenMP THREADPRIVATE，供 conf_phys.F、restart/初始化路径和各物理过程模块读写。

从源码看，callkeys_mod 不是默认值定义页。除 integer,parameter :: swrtype=2（第 109 行）外，其余变量只在本模块中声明存储；常见默认值和 callphys.def 覆盖路径在 conf_phys.F 中完成，例如 conf_phys.F 第 54-79 行导入本模块变量，第 201-203 行读取 iaervar，第 230-354 行读取核心物理开关，第 532-545 行读取尘埃注入参数，第 706-755 行读取水/CO2 云开关，第 1160-1186 行读取高层大气开关。

共享状态与副作用

  • 本模块的核心副作用是提供全局可见的 SAVE 状态。多数变量紧跟 !$OMP THREADPRIVATE(...) 指令，使 OpenMP 线程各自持有一份配置状态。
  • conf_phys.F 是主要写入者：它从 callphys.def 风格配置读取并覆盖默认值，再写入这些模块变量。
  • 许多物理模块只读其中一小组变量，例如 watercloud_mod.F 读取 activice/microphys/supersat/hdo/scavenging/CLFvarying，updatereffrad_mod.F 读取 water/activice/microphys/rdstorm/topflows/co2clouds/activeco2ice/co2useh2o，vdif_cd_mod.F90 读取 callrichsl/callatke，read_dust_scenario_mod.F90 读取 iaervar/dustinjection/swrtype。
  • 复现风险：如果并行初始化路径没有在每个 OpenMP 线程上正确传播这些 THREADPRIVATE 值，后续物理分支可能在线程间不一致。

核心逻辑

  1. 声明 callkeys_mod 模块并启用 IMPLICIT NONE。
  2. 按物理主题声明 logical,save、real,save、integer,save 和一个 character(len=100),save 状态变量。
  3. 紧随变量声明为 OpenMP THREADPRIVATE，保证并行线程私有副本。
  4. 在文件尾部定义唯一编译期常量 swrtype=2，注明短波方案 1=Fouquart, 2=Toon。

伪代码

module callkeys_mod:
    declare saved logical switches for physics branches
    mark each switch group threadprivate

    declare saved real/integer/character runtime parameters
    mark each parameter group threadprivate

    define swrtype = 2
end module

写法特点

  • 这是纯状态模块，没有初始化例程；读者不能只看本文件判断运行默认值。
  • 变量按历史功能追加排列，而不是按严格子系统拆成多个模块；因此同一模块被辐射、尘埃、水、CO2、PBL、高层大气和 restart 路径共同使用。
  • 大量 THREADPRIVATE 指令是复现关键点。串行复现只需关注变量值；OpenMP 复现还要关注每个线程的变量副本何时被初始化。

复现要点

  • 复现实验配置时，应同时记录 callphys.def.*、conf_phys.F 中的默认值/互斥检查，以及本模块的 THREADPRIVATE 状态。
  • swrtype 在当前源码中是 parameter，不是运行时 getin_p 开关；短波方案默认固定为 Toon，旧 Fouquart 路径依赖源码改动或历史分支。
  • callkeys_mod 只保存开关；具体数组维度、tracer 索引、辐射表和云微物理常数仍在各自模块中定义。
  • 若某个 downstream 页面说“来自 callkeys_mod”，应进一步查 conf_phys.F 或对应 def 样例确认默认值和合法组合。

待确认

  • semi 源码注释为 ! ?!?，本文件无法说明其物理含义；需结合 conf_phys.F 和长波半隐式路径确认。
  • 除 swrtype=2 外，所有变量默认值不在本文件中定义；本页只记录声明位置和用途，不把 conf_phys.F 的默认值重写为本模块事实。
