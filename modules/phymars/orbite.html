文件定位

orbite.F 定义 orbite_mod，只提供 orbite 一个例程。它把太阳经度 Ls 转换成当前太阳-行星距离 dist_sol 和太阳赤纬 declin，是 physiq_mod.F 中辐射几何准备的一步。

当前主路径中，physiq_mod.F 先用 solarlong 得到共享状态 zls，再在 callrad 分支中调用 orbite(zls,dist_sol,declin)。随后 declin 进入 solang 或 mucorr 计算 mu0/fract，dist_sol 则用于辐射、坡面直接辐射、近红外 CO2 吸收和热层等路径的太阳通量缩放。

共享状态与副作用

  • 本文件没有本地 SAVE 变量、module 状态、common block、文件 I/O 或诊断输出。
  • orbite 只读取 planete_h 和 comcstfi_h 中的共享常数/轨道状态，并通过两个 intent(out) 参数返回结果。
  • 上游 physiq_mod.F 把输出写入 comsaison_h::dist_sol/declin；这些共享状态随后被辐射、近红外 CO2、坡面和热层路径读取。
  • planete_h::iniorbit 在源码行 73-100 计算 e_elips、p_elips 和 lsperi；若绕过该初始化，orbite 读到的轨道状态可能未定义或过期。

核心逻辑

  1. 接收当前太阳经度 pls，不对角度范围做归一化。
  2. 用椭圆轨道公式计算太阳-行星距离：
pdist_sol = p_elips / (1 + e_elips * cos(pls - lsperi))
  1. 用倾角和太阳经度计算太阳赤纬：
pdecli = asin(sin(pls) * sin(obliquit * pi / 180.))
  1. 直接返回两个标量；没有额外限幅、异常处理或日志输出。

伪代码

orbite(pls):
    distance = p_elips / (1 + e_elips * cos(pls - lsperi))
    declination = asin(sin(pls) * sin(obliquit * pi / 180))
    return distance, declination

参与的主题流程

主题 参与方式
轨道 / 季节换算 把 solarlong 给出的 zls 转换成距离太阳的 AU 值和太阳赤纬。
太阳几何 declin 是 solang 与 mucorr 计算 mu0/fract 的上游输入。
辐射调用链 dist_sol 进入 callradite 和其他太阳通量缩放路径，用于按 1/dist_sol^2 调整 1 AU 太阳常数。
近红外 CO2 吸收 nirco2abs 接收 dist_sol/mu0/fract/declin 计算近红外 CO2 加热率。
坡面和热层路径 physiq 坡面直接辐射和 thermosphere 调用也读取 dist_sol，坡面参数还读取 declin。

写法特点

  • 固定格式 .F 文件，但内容是一个 module 和一个 contained subroutine。
  • 输入/输出均为单精度 REAL 标量。
  • obliquit 在 planete_h 中以度保存，本例程在赤纬公式中显式乘 pi/180.；pls 和 lsperi 则按弧度参与三角函数。
  • 公式没有对 1 + e_elips*cos(pls-lsperi) 做分母保护，也没有对 asin 参数做限幅；复现时应保留源码行为。

复现要点

  • 调用前必须确保 planete_h 中的 e_elips/p_elips/lsperi/obliquit 已经按当前运行的轨道参数初始化。
  • p_elips 在 planete_h::iniorbit 中除以 unitastr，因此 pdist_sol 返回 AU；不要再重复换算。
  • pls 必须是弧度。solarlong 输出 zls 时也使用弧度。
  • 若使用 1D/testphys 的古气候或自定义轨道输入，要确认 peri_day、lsperi、e_elips 与 p_elips 的更新路径一致；本页只确认 orbite 字面读取 planete_h 变量。
  • dist_sol 和 declin 在 physiq_mod.F 中只在 callrad 分支下由 orbite 更新；关闭辐射时仍读取这两个量的自定义流程需要单独设置或确认上一次值。

待确认

  • 1D 古气候初始化中自定义 eccentric/Lsperi 后，是否所有运行路径都会同步得到与 iniorbit 一致的 p_elips；需要结合 init_testphys1d_mod.F90 和 tabfi.F 的初始化顺序逐步核对。
  • asin 输入的极端数值保护不在本文件中处理；当前源码依赖轨道参数范围使公式保持在有效区间。
