libf/phymars

文件定位

tabfi.F 是物理初始化路径中的控制表读取和常数恢复模块。tabfi 从 startfi.nc 的 controle 数组读取火星半径、自转角速度、重力、气体摩尔质量、日长、物理时间步、轨道量、地表粗糙度、湍流下限、CO2 冰反照率/发射率、尘埃可见光学厚度和土壤体积热容；当 nid=0 时，它不读文件，而写入一组默认 Mars 常数。若 Lmodif=1 且处于串行运行，它还允许交互修改控制表中的若干物理常数，最后调用 iniorbit 生成轨道派生量。

共享状态与副作用

tabfi 会直接写入多个共享模块变量：comcstfi_h 的 rad/omeg/g/mugaz/rcp，time_phylmdz_mod 的 daysec/dtphys，planete_h 的 year_day/periheli/aphelie/peri_day/obliquit/lmixmin/emin_turb，surfdat_h 的地表和 CO2 冰辐射参数，dimradmars_mod::tauvis，以及 comsoil_h::volcapa。它会向标准输出打印控制表前后状态；修改模式还会从标准输入读取关键字和值。并行运行时若 Lmodif=1 会 abort_physic。

核心逻辑

  1. 打印 nid/tab0/Lmodif。
  2. 若 nid=0，清零 tab_cntrl 并写入默认 Mars 常数、轨道量、地表粗糙度、CO2 冰辐射参数、尘埃光学厚度和土壤热容。
  3. 若 nid!=0，调用 get_var("controle", tab_cntrl, found)，随后按 tab0+index 读取各控制量并写入共享模块。
  4. 打印修改前的控制表值；若 Lmodif=1 且并行运行则终止。
  5. 串行修改模式中循环读取用户关键字，允许修改 day_ini/z0/emin_turb/lmixmin/emissiv/emisice/albedice/iceradius/dtemisice/tauvis/obliquit/peri_day/peri_ls/periheli/aphelie/volcapa。
  6. 修改后再次打印控制表值，调用 iniorbit()。
  7. 若旧 start 文件没有 iceradius，把南北 CO2 雪粒径设为 100.e-6，dtemisice 设为 0.4。

伪代码

if nid == 0:
    set default Mars physical constants and surface/radiative parameters
else:
    read controle into tab_cntrl
    copy tab_cntrl(tab0 + indices) into shared modules and output args

print control values before modification
if Lmodif == 1:
    abort if parallel
    while user enters a keyword:
        read replacement value and update matching shared variable
    print control values after modification

call iniorbit()
patch old start files if iceradius is zero

参与的主题流程

主题 参与方式
初始状态 / restart phyetat0_mod 调用本例程恢复或默认生成物理控制表。
轨道和太阳几何 写入基础轨道量并调用 iniorbit，影响 solarlong、orbite 等下游。
CO2 冰和地表辐射 初始化 albedice/emisice/iceradius/dtemisice/emissiv/z0_default 等参数。
土壤和边界层 初始化 volcapa、lmixmin、emin_turb。

写法特点

  • 固定格式 Fortran，包含交互式 read(*,*) 修改循环。
  • tab_cntrl 本地长度固定为 100；物理参数通过 tab0+index 访问。
  • getin_p 被 use 但当前文件未直接调用。
  • surfdat_h 的 only 列表中 iceradius 重复出现。

复现要点

  • nid=0 与 nid!=0 是两条完全不同的初始化路径；前者使用默认常数，后者依赖 startfi.nc 的 controle 数组。
  • Lmodif=1 只能在串行路径执行；并行修改会直接终止。
  • 修改 peri_ls 不直接写入 restart 控制表字段，而是通过 lsp2solp 反推 peri_day。
  • 老 start 文件若 iceradius(1)=0，会触发 CO2 雪粒径和 metamorphism 时间尺度回退值。

待确认

  • getin_p use 是否仍有计划用途，当前源码未调用它。
  • 控制表 tab0+12/13/28-30 等空档字段是否由历史版本或其他工具占用，需要结合写出端 phyredem 继续核对。
