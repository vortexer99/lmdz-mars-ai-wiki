文件定位

planete_h.F90 定义 planete_h 共享模块，用来保存火星轨道参数、年长、倾角、边界层湍流下限，以及由轨道输入派生出的偏心率、半通径和近日点太阳经度。它本身不推进物理时间步，而是在初始化阶段由 tabfi 或 1D 初始化路径写入基础量，再由 iniorbit 计算 e_elips/p_elips/lsperi，供 solarlong 和 orbite 把 sol 日期、太阳经度、日火距离和太阳赤纬联系起来。

输出

输出 去向 类型 / 维度 单位 含义
solp tabfi 的 peri_day real scalar sol lsp2solp 由近日点太阳经度反推的近日点日期。

共享状态与副作用

  • 所有可变模块量都是 REAL,SAVE，并用 OpenMP THREADPRIVATE 分组声明；并行运行时每个线程持有私有副本。
  • lsp2solp 不只返回 solp，还会重算并写入模块变量 e_elips。
  • iniorbit 写入 e_elips/p_elips/lsperi，并向标准输出打印输入轨道量、偏心率、半通径、平均近点角、偏近点角和近日点太阳经度。
  • coefvis/coefir 在本文件中声明为共享状态，但本次 rg 未发现 phymars 内实际读写点；复现时不要假定它们已参与当前主路径。

核心逻辑

  1. 初始化路径先写基础轨道量：aphelie/periheli/year_day/peri_day/obliquit。3D 路径来自 tabfi 默认值或 startfi.nc 控制表；1D 路径来自默认常数或 paleomars 配置。
  2. 如用户在 tabfi 交互修改 peri_ls，lsp2solp 先用远日点/近日点距离求 e_elips，把输入的近日点 Ls 转成偏近点角，再由 Kepler 方程形式反推出 peri_day。
  3. iniorbit 重新由远日点/近日点距离计算 e_elips，并计算 AU 单位的 p_elips。
  4. iniorbit 用 (year_day - peri_day)/year_day 得到平均近点角，Newton 迭代求偏近点角 zx0，收敛阈值为 1.e-12，最多 100 次。
  5. iniorbit 由 zx0 和 e_elips 计算 lsperi，再用 modulo(lsperi,2*pi) 归一化。
  6. 后续 solarlong 用 peri_day/year_day/e_elips/lsperi 把日期转为 Ls；orbite 用 e_elips/p_elips/lsperi/obliquit 把 Ls 转为距离和赤纬。

伪代码

initialize base orbit values from tabfi, restart, or 1D configuration

if user changes peri_ls in tabfi:
    e_elips = (aphelie - periheli) / (aphelie + periheli)
    zx0 = eccentric_anomaly_at_Ls0(peri_ls, e_elips)
    solp = year_day * (1 - (zx0 - e_elips * sin(zx0)) / (2*pi))
    peri_day = solp

iniorbit:
    e_elips = (aphelie - periheli) / (aphelie + periheli)
    p_elips = 0.5 * (periheli + aphelie) * (1 - e_elips^2) / unitastr
    zanom = 2*pi * (((year_day - peri_day) / year_day) - nearest_integer)
    solve zx0 - e_elips*sin(zx0) = abs(zanom) by Newton iteration
    if zanom < 0:
        zx0 = -zx0
    lsperi = modulo(-2 * atan(sqrt((1+e_elips)/(1-e_elips)) * tan(zx0/2)), 2*pi)

主题 参与方式

写法特点

  • 这是自由格式 .F90 模块，但变量名沿用较老的 planete_h 头文件风格。
  • 共享变量未声明 PROTECTED，源码第 23 行注释也提示理想情况下应保护；当前实现依赖初始化顺序约束。
  • lsp2solp 的局部 degrad = 180.d0/pi 让输入 lsp 按 degree 解释；lsperi 模块变量则以 rad 保存。
  • iniorbit 的 Newton 迭代结束后无论是否因收敛退出，都会再执行一次 zx0 = zx0 + zdx；复现时应保留该字面行为。

复现要点

  • 调用 solarlong 或 orbite 前必须保证 iniorbit 已在当前线程的 planete_h 副本上执行，否则 e_elips/p_elips/lsperi 可能未定义或过期。
  • aphelie/periheli 用 Mkm，p_elips 输出为 AU；不要在 orbite 中重复除以天文单位。
  • 3D restart 路径恢复的是基础量，再由 tabfi 调 iniorbit 重建派生量；restart 文件不直接保存 e_elips/p_elips/lsperi。
  • 1D paleomars 分支会把 Lsperi 从 degree 转为 rad，仅用于 call_dayperi；随后仍需依赖初始化链路调用 iniorbit 来生成 lsperi/p_elips/e_elips。
  • read_dust_scenario_mod.F90 中 realday=mod(zday,669.) 仍硬编码 669，但时间维一致性检查用 year_day；古气候或非标准年长配置下要分别检查这两处行为。

待确认

  • coefvis/coefir 在当前 phymars 搜索范围内没有除声明外的读写点；是否由预处理、外部分支或历史接口使用，需后续全仓库或构建配置核对。
  • 1D paleomars 分支中 Lsperi 范围检查源码写成 if (eccentric < 0. .or. eccentric > 360.)，疑似应检查 Lsperi；本页按源码字面记录，不判断修复。
