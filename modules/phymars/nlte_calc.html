文件定位

nlte_calc.F 是 CO2 15 微米 NLTE 冷却快速方案中的光学厚度、escape probability 和 Curtis-matrix 系数计算文件。文件顶部保留了注释掉的 MODULE nlte_calc_mod / USE nlte_aux_mod / CONTAINS 包装，当前源码实际提供一组外部子程序，由 nlte_tcool_mod.F 直接调用。

本文件负责把 nlte_commons_h 中的温度、CO2 数密度、混合比、谱线参数和历史谱线表转换成 c110/vc110、c121/vc121、taustar11_cts、taustar12 等冷却矩阵输入。它不读文件；历史谱线表读取、插值定位、等效线宽和归一化工具来自 nlte_aux。

共享状态与副作用

  • 本文件主要通过 nlte_commons_h 读写共享工作数组；例如 c110/vc110/c121/vc121/taustar11_cts/taustar12 是下游 nlte_tcool_mod.F 继续使用的状态。
  • 多个例程直接写 ibcode1，用字符形式标识当前谱带/子带，再调用 gethist_03(ib) 切换历史谱线表。
  • 错误处理分两类：大多数数值问题设置 ierr/varerr 并返回，由调用方 errors 处理；MZMC121 的非法子带索引使用 abort_physic。
  • 例程会对 c110/c121/vc110/vc121/taustar* 和若干局部矩阵清零后重算；调用顺序依赖 nlte_tcool_mod.F 的初始化。

核心逻辑

  1. MZESC110 在 CTS 高度网格上用 co2y_cts/nty_cts/ty_cts/py_cts 建立柱量和混合比，读取 110 历史表，逐层积分得到 tauinf，最后调用 mzescape_normaliz_02 得到 taustar11_cts。
  2. MZTUD110 把 GCM 主网格的 CO2 柱量按振动温度修正，读取 110 历史表，逐层/逐路径计算 tau(in,ir) 与 tauinf(in)；若最大透过率超过 1，返回错误 42。
  3. MZCUD110 对 tau/tauinf 做差分，乘以 pi*deltanu 和垂直差分算子，写出 c110 与 vc110。
  4. MZMC121 对 121 热带的三个子带 ib=2,3,4 逐一调用 MZTUD121，再按频率比例和温度指数因子合成 c121/vc121。
  5. MZESC121 逐一调用 MZESC121sub 计算三个 121 子带的 escape probability，再按频率比例合成并调用 mzescape_normaliz。
  6. MZTVC121 逐一调用 MZTVC121sub 计算三个子带边界向量，再按频率比例合成 vc121。

伪代码

nlte_tcool_mod calls nlte_calc routines:

for 110 band:
  MZESC110:
    build CTS CO2 column and mixing-ratio arrays
    get history table 1
    integrate layer-to-space optical depths
    normalize taustar11_cts

  MZTUD110:
    interpolate vibration temperature to GCM grid
    correct CO2 column and get history table 1
    for each source layer in:
      integrate optical paths to lower layers and to space
      use interstrhunt/intershphunt + we_clean for equivalent widths
    MZCUD110 converts tau differences into c110 and vc110

for 121 hot band:
  MZMC121:
    for ib in 2, 3, 4:
      MZTUD121 computes one sub-band matrix and vector
      scale by frequency/temperature factors
      accumulate into c121 and vc121

  MZESC121:
    for ib in 2, 3, 4:
      MZESC121sub computes one sub-band taustar
    scale and normalize into taustar12

  MZTVC121:
    for ib in 2, 3, 4:
      MZTVC121sub computes one sub-band boundary vector
    scale and accumulate into vc121

参与的主题流程

主题 参与方式
NLTE 15 微米 CO2 冷却 生成 nlte_tcool_mod.F 所需的 110/121 带 escape probability、Curtis matrix 和边界向量。
辐射矩阵求解 c110/c121/vc110/vc121 进入后续 vibrational population 和 cooling-rate 矩阵计算。
数值错误传播 通过 ierr/varerr 把非正 CO2 柱量、非正压力、透过率大于 1、负 vc 等问题传给 errors。

写法特点

  • 固定格式 Fortran，旧 1-D 文件名保留在注释中，顶层 module 包装未启用。
  • 110 和 121 代码结构高度相似：先积分光学路径得到 tau/tauinf，再转成矩阵或 escape probability。
  • MZMC121、MZESC121 和 MZTVC121 都把 ib=2,3,4 三个热带子带按频率比例合成；MZMC121 还含温度指数缩放因子。
  • 多处用 ff=1.0d10 防止过小数值，并用 beta=1.8d5 把 box 柱量传给 we_clean。

复现要点

  • 调用本文件前，nlte_setup.F 必须已经填好 nlte_commons_h 的 CO2/温度/高度/谱线表路径与历史表缓存。
  • nlte_aux.F::rhist_03 读入的历史谱线表必须覆盖 ib=1..4，否则 gethist_03(ib) 恢复的表不完整。
  • co2y 顶层非正、压力非正、混合比非正或 tau > 1 都会触发 ierr/varerr，不能忽略返回值。
  • 121 热带只按源码支持 ib=2,3,4；扩展谱带时需要同时修改合成例程和历史表索引。
  • MZTVC121sub 遇到负 vc 会用上一层值替代，但仍在末尾统计 minvc；复现时要保留这个数值保护顺序。

待确认

  • ierr 代码 33、42、43、45、46、51、52、83 等的完整语义需要结合 nlte_tcool_mod.F::errors 继续整理。
  • beta=1.8d5、ff=1.0d10、2.d5*deltaz 等常数的物理单位来源需要旧 IAA/CSIC 文档确认。
  • 121 热带三个子带频率缩放因子的物理推导本页按源码记录，未外推解释。
