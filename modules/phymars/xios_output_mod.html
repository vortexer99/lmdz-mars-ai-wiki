文件定位

xios_output_mod.F90 是 LMDZ.MARS 物理包到 XIOS 输出库的 Fortran 封装层。它在 CPP_XIOS 编译期开关下提供 XIOS 上下文初始化、时间步推进、0D/1D/2D/3D 字段发送、字段活跃性查询和上下文 finalize；无 CPP_XIOS 时，源码中实际可见的模块内容只剩 time_it 与 context_id 等静态定义，XIOS 例程不会进入编译。

运行时该模块与 deftank/xios/*.xml 配置配合：context_id="LMDZ" 对应 context_pcm_physics.xml，initialize_xios_output 设置垂直轴、土壤轴、坡面轴、水平 domain、用户自定义日历和起始时间，然后调用 wxios_closedef 关闭定义阶段；后续 update_xios_timestep 每个物理时间步推进 XIOS calendar，send_xios_field 泛型接口把具体字段发送给 XIOS。

共享状态与副作用

  • time_it 是模块私有 SAVE 状态，由 OpenMP master 在 update_xios_timestep 中递增。
  • 多数 XIOS API 调用包在 !$OMP MASTER 中，调用前后常用 !$OMP BARRIER 同步。
  • initialize_xios_output 会关闭 XIOS 定义阶段；之后再改 XML 定义或动态添加轴/field 不属于该例程覆盖范围。
  • histwrite2d_xios 和 histwrite3d_xios 会把物理网格字段通过 Gather_omp 与 grid1Dto2D_mpi 转为当前 MPI 子域的二维/三维数组再发送。
  • xios_is_active_field 的结果由 master 查询后用 bcast_omp 广播；但当前 write_output_mod 中活跃性检查被注释，实际发送路径将 is_active 强制为 .true.。

核心逻辑

  1. CPP_XIOS 编译时定义 send_xios_field 泛型接口，按实参秩分派到 0D/1D/2D/3D 发送例程。
  2. initialize_xios_output 在 master 线程设置所有垂直/土壤/坡面轴属性。
  3. 根据 grid_type 调用 wxios_domain_param_unstructured("dom_glo") 或 wxios_domain_param("dom_glo") 初始化水平 domain。
  4. master 定义 user-defined calendar：XIOS 时间步为 1 秒，day_length=nint(daysec/dtphys)，月份长度硬编码为 12 个 Mars 月长度。
  5. 设置 time_origin=xios_date(1,1,1,0,0,0) 和同样的 start_date，然后 wxios_closedef。
  6. 每个 XIOS 时间步调用 update_xios_timestep：先 set_xios_context，再递增 time_it 并 xios_update_calendar(time_it)。
  7. 写字段时，0D/1D 直接 xios_send_field；2D/3D 先校验第一维等于 klon，再 gather/recast 后发送。
  8. 结束时调用 finalize_xios_output 执行 xios_context_finalize。

伪代码

initialize_xios_output(day, timeofday, dtphys, daysec, presnivs, pseudoalt, mlayer):
  master:
    set XIOS axes presnivs, altitude, interlayer, soil_layers, subslope
  barrier
  if grid_type == unstructured:
    wxios_domain_param_unstructured("dom_glo")
  else:
    wxios_domain_param("dom_glo")
  master:
    define user calendar using day_length = nint(daysec / dtphys)
    set time_origin and start_date
    wxios_closedef()
  barrier

update_xios_timestep():
  set_xios_context()
  master:
    time_it += 1
    xios_update_calendar(time_it)

send_xios_field(field_name, field):
  if field is 0D or 1D:
    master sends directly
  if field is 2D or 3D:
    require first dimension == klon
    Gather_omp(field)
    master grid1Dto2D_mpi(...)
    master xios_send_field(...)

参与的主题流程

主题 参与方式
XIOS 输出 提供 Fortran 到 XIOS runtime 的初始化、时间推进、字段发送和关闭接口。
通用输出封装 write_output_mod 在 CPP_XIOS 下调用 send_xios_field。
物理主循环 physiq 首次调用时初始化 XIOS，每步更新 calendar，末尾发送控制表和轴相关字段。
配置系统 与 xios-config 中的 XML context、field_def 和 file_def 配套。

写法特点

  • 所有实际 XIOS 例程都在 #ifdef CPP_XIOS 内；编译配置决定该模块是否提供可调用输出接口。
  • context_id 硬编码为 "LMDZ"，必须与 XML context id 保持一致。
  • day 和 timeofday 当前没有参与 start_date 计算；源码注释保留了“减去 dtphys”的说明，但实际 start_date=time_origin。
  • histwrite1d_xios 的 dimens 参数未使用，1D 实参直接发送给 XIOS。
  • 2D/3D 字段发送前只检查第一维 klon，不在本例程内验证 XML field 的 grid_ref 是否匹配。

复现要点

  • 必须使用 CPP_XIOS 编译，并保证 XIOS XML 中存在 context id LMDZ。
  • initialize_xios_output 必须在任何 send_xios_field 之前执行，且必须完成 wxios_closedef。
  • 每个实际物理输出步需要先调用 update_xios_timestep，否则 XIOS calendar 与字段时间轴会错位。
  • dtphys 和 daysec 若不是整数，源码只打印 warning；day_length 仍取 nint(daysec/dtphys)。
  • 2D/3D 发送字段第一维必须等于 dimphy:klon，否则 abort_physic 中止。
  • 非结构网格走 wxios_domain_param_unstructured，与结构化经纬 domain 的 XML/后处理假设不同。

待确认

  • day、timeofday 在接口中传入但当前未用于 start_date；是否为历史遗留或待恢复逻辑需结合 XIOS 时间轴测试确认。
  • regular_lonlat_mod、若干 MPI 子域变量在当前实代码中主要出现在注释或调试路径，是否仍需导入待确认。
  • write_output_mod 当前注释掉 xios_is_active_field 检查并强制发送；若恢复活跃性过滤，XIOS 输出量和性能都会改变。
