文件定位

nlte_commons_h.F90 定义真实 Fortran module nlte_commons_h，把旧 1-D NLTE 15 微米 CO2 参数化中的多个 common block 合并成一组 save module 变量。它本身不包含 contains、subroutine 或 function；作用是为 nlte_setup.F、nlte_aux、nlte_calc 和 nlte_tcool_mod.F 共享谱线常数、查表数据、主网格/CTS 网格大气剖面、Curtis matrices、escape probability、振动温度、碰撞率和历史谱线缓存。

源文件几乎所有变量都声明为 save 并配套 !$OMP THREADPRIVATE(...)。因此它不是普通只读参数头文件，而是 NLTE 计算链的每线程共享状态仓库：上游初始化例程写入表和剖面，辅助/矩阵例程读写工作数组，下游 cooling-time 例程再读取结果矩阵和加热率。

共享状态与副作用

  • 所有主要变量均为 save module 状态，且每组变量后都有 !$OMP THREADPRIVATE 指令；OpenMP 运行时每个线程保留自己的副本。复现时不能假设一个线程写入后其他线程自动可见。
  • nlte_setup.F 写入谱线常数、预计算表、hisfile 和碰撞率；nlte_aux.F::rhist_03 再根据 hisfile 填充 *_stored 历史表缓存。
  • nlte_tcool_mod.F 的初始化/准备路径写入主网格、CTS 网格、边界索引、mmean_cts/cpnew_cts 等大气状态；同一文件后续读取这些状态并求解各带冷却。
  • nlte_calc 读取 elow/deltanu、大气剖面和历史谱线表工作数组，并写出 c110/vc110/c121/vc121/taustar12/taustar11_cts 等矩阵输入。
  • nlte_aux 的 initial、gethist_03、rhist_03 和插值例程会直接清零、恢复或读取 eqw/aa/cc/dd、thist/no/sk1/xls1/xld1、mr/mr_cts 等工作变量。

核心逻辑

  1. 编译期从 nlte_paramdef_h 取得所有数组维度，nlte_commons_h 据此声明旧 common block 对应的 module 变量。
  2. nlte_setup.F 在初始化阶段读取或计算谱线常数、查表数据、历史表文件名和碰撞率，并写入本模块。
  3. nlte_tcool_mod.F 把 GCM 输入剖面转换为 NLTE 主网格、CTS 网格和高分辨率插值网格状态，写入本模块的 zl/t/pl/...、zy/ty/...、zl_cts/t_cts/...。
  4. nlte_aux.F 读取历史谱线表到 *_stored，再按当前谱带恢复为 thist/no/sk1/xls1/xld1 工作数组，并提供插值/等效线宽工具。
  5. nlte_calc.F 用共享剖面、谱线表和工作数组构造 Curtis matrices、边界向量和 escape probability，并写回 c*、vc*、taustar*。
  6. nlte_tcool_mod.F 读取这些矩阵和速率系数，计算 NLTE 振动温度、各带冷却率和最终冷却贡献。

伪代码

compile-time:
  import NLTE dimensions from nlte_paramdef_h
  allocate saved, threadprivate module variables

setup phase:
  nlte_setup writes spectral constants, tabulation arrays and rates
  nlte_setup sets hisfile for each history table
  nlte_aux.rhist_03 reads history tables into *_stored caches

profile preparation:
  nlte_tcool_mod writes main and CTS atmospheric profiles
  nlte_tcool_mod sets grid spacing and boundary indices

matrix construction:
  nlte_aux.gethist_03 copies one stored table into current work arrays
  nlte_calc reads profiles and line tables
  nlte_calc writes Curtis matrices, boundary vectors and taustar arrays

cooling solve:
  nlte_tcool_mod reads rates, matrices, taustar and profiles
  nlte_tcool_mod updates vt*, hr*, el* and sl* result arrays

参与的主题流程

主题 参与方式
NLTE 15 微米 CO2 冷却 保存从初始化、查表、矩阵构造到 cooling-time 求解之间传递的全部共享状态。
辐射计算 c*、vc*、taustar*、hr* 等数组进入 CO2 15 微米非 LTE 冷却贡献计算。
并行/OpenMP 运行 THREADPRIVATE 使 NLTE 状态按线程隔离，初始化和计算必须在每线程语义下保持一致。

写法特点

  • 文件保留旧 common /.../ 注释，下面用 module 变量替代，这使旧 1-D 模型命名仍可追溯。
  • 本文件没有 public/private 显式限制；下游通过 use nlte_commons_h, only: ... 选择需要的共享变量。
  • 同一物理流程的数组分散在多个旧 common block 分组中，例如谱线常数、矩阵、主网格、CTS 网格、碰撞率和历史谱线缓存，需要按上下游读写关系理解。
  • real 与 real*8 混用：大气剖面多为 real，谱线、矩阵、速率和历史表多为 real*8。复现或接口迁移时不能统一假设为双精度。

复现要点

  • 必须先完成 nlte_setup.F 的表和速率初始化，再进入 nlte_tcool_mod.F / nlte_calc.F 计算；否则本模块中的 save 数组可能仍未初始化。
  • 历史谱线表必须通过 rhist_03 写入 *_stored，再由 gethist_03 恢复当前谱带工作数组；直接调用矩阵例程会依赖这些缓存状态。
  • nbox_max、nhist、nb 等维度来自 nlte_paramdef_h，输入表尺寸超过这些上限会在下游读表或积分阶段触发错误。
  • OpenMP 复现时要确认每个线程都拥有正确初始化后的 THREADPRIVATE 副本；这类状态不会自动跨线程共享。
  • 主网格与 CTS 网格分别使用 nl/nzy 和 nl_cts/nzy_cts 维度，不能把 taustar11_cts、*_cts 剖面和主网格数组混用。

待确认

  • input_cza 的完整状态枚举和分支语义需要结合 nlte_tcool_mod.F 后续页面整理。
  • k19*、k20*、k21*、k33* 等碰撞率系数的物理来源需结合 nlte_setup.F 和旧 IAA/CSIC 文档确认。
  • real 主剖面与 real*8 谱线/矩阵数组混合精度是否有历史误差控制依据，源文件本身未解释。
