文件定位

nlte_aux.F 是 CO2 15 微米 NLTE 冷却快速方案的外部辅助例程集合。文件开头保留了 MODULE nlte_aux_mod / CONTAINS / END MODULE nlte_aux_mod 的注释行，但当前源码并没有启用 module 包装；下游通过外部函数和子程序名直接调用这些例程。

本文件合并了旧 1-D NLTE 模型的若干文件片段，包括 Planck 函数、振动温度表初始化、温度/高度/多维查表插值、escape probability 归一化、LU 分解和矩阵/向量工具，以及历史谱线表 hisfile 的读取和广播。它服务于 nlte_setup.F 的表读取、nlte_calc.F 的 NLTE 辐射查表和等效线宽计算，以及 nlte_tcool_mod.F 的冷却时间尺度矩阵求解。

共享状态与副作用

  • initial、leetvt、gethist_03、rhist_03 等例程直接读写 nlte_commons_h 中的共享数组。
  • rhist_03 使用固定 Fortran unit 3 打开 hisfile，读取后关闭；非 MESOSCALE 编译路径只让 is_master 进程做文件 I/O，再用 bcast 广播数组。
  • 多个插值例程在输入超出表范围时会写 stdout 警告或调用 abort_physic。intershphunt 对接近边界的温度做 0.01 K 容忍钳制，但真正越过查表边界仍可能终止。
  • we_clean 对 Voigt 组合项为负的情况调用 abort_physic，说明谱线数据或输入范围不再物理一致。
  • ludcmp_dp 在发现奇异矩阵时调用 abort_physic("ludcmp_dp","singular matrix!",1)。

核心逻辑

  1. nlte_setup.F 初始化 NLTE 表文件名后，多次调用 rhist_03(ihist) 读取不同历史谱线表。非 MESOSCALE 并行路径由主进程读文件并广播缓存。
  2. nlte_calc.F 进入具体 NLTE 冷却计算前调用 gethist_03(ihist)，把某个历史表缓存恢复为当前 thist/no/sk1/xls1/xld1 工作数组。
  3. intzhunt*、interstrhunt、intershphunt 和多组 interhunt* 按高度、温度和其他表坐标查找包围区间，做线性或多维线性插值。
  4. we_clean 用 Lorentz、Doppler 和 Voigt 近似把吸收路径量与线宽合成为等效宽度；若组合项不可开方则终止。
  5. mztf_correccion 和 mzescape_normaliz* 修正柱量和 escape probability 光学厚度，供 nlte_calc.F 的多谱带/多同位素计算继续使用。
  6. nlte_tcool_mod.F 使用 planckdp 计算边界/层内 Planck 项，并用 LUdec 解冷却时间尺度相关的线性系统；suaviza 对某些剖面做平滑。

伪代码

setup phase:
  for each NLTE history table:
    rhist_03(ihist)
      if not MESOSCALE:
        master opens hisfile on unit 3
      read mm_stored, nbox_stored, thist_stored, no/sk1/xls1/xld1
      if nbox_stored > nbox_max:
        abort
      if not MESOSCALE:
        broadcast cached arrays

calculation phase:
  gethist_03(ihist)
    copy cached history table into current working arrays

  for each target layer/box:
    locate height and temperature intervals with hunt/huntdp
    interpolate state, line strength, Lorentz width and Doppler width
    compute equivalent width with we_clean
    normalize escape probability arrays

  for cooling-time linear systems:
    build coefficient matrices
    LUdec calls ludcmp_dp + lubksb_dp
    return solved vectors/matrices to nlte_tcool_mod

参与的主题流程

主题 参与方式
NLTE 15 微米 CO2 冷却 为 nlte_calc.F 提供历史谱线表读取、温度/高度插值、等效线宽和 escape probability 工具。
辐射计算 nlte_tcool_mod.F 使用 planckdp 和 LUdec 构造/求解冷却时间尺度中的辐射矩阵。
并行运行 rhist_03 在非 MESOSCALE 路径中使用主进程读表和广播，避免每个进程重复读文件。

写法特点

  • 文件保留固定格式 Fortran 和旧 1-D 模型来源注释，代码中混合 c 注释、! 注释、@/& 续行和预处理条件。
  • 顶层 module 包装被注释掉，因此其他文件以 external 或隐式外部子程序方式引用本文件例程。
  • 查表例程很多，命名以 hunt / interhunt 组合维度和精度，复现时需要按调用方实参确认具体坐标含义。
  • rhist_03 的文件 I/O 依赖 nlte_commons_h::hisfile 已由上游设置；本文件不负责构造路径。

复现要点

  • 复现 NLTE cooling 前必须先通过 rhist_03 读入对应历史表，否则 gethist_03 只能复制未初始化或旧缓存。
  • nbox_stored(ihist) 不能超过 nlte_paramdef_h::nbox_max。
  • 温度历史表插值对极小边界误差有容忍，但超出查表范围仍可能终止；输入温度网格需要与表范围一致。
  • we_clean 的 y、pl、xalsa、xalda 不能导致 Voigt 组合项为负；否则说明谱线参数或路径状态超出适用范围。
  • 非 MESOSCALE 并行复现时要保证 mod_phys_lmdz_para 和 mod_phys_lmdz_transfert_para 的广播语义可用。

待确认

  • hisfile 的实际文件名和数据来源由 nlte_setup.F / 运行时数据目录决定，本页只确认 rhist_03 的读取格式。
  • mztf_correccion、mzescape_normaliz* 中各硬编码谱带/同位素修正的物理来源需要结合 IAA/CSIC 旧模型文档确认。
  • 多组 interhunt* 例程的每个坐标变量物理含义需要在具体调用点逐一对应，不能仅凭例程名推断。
