文件定位

phys_state_var_init_mod.F90 是 phymars 物理状态数组和基础常数的生命周期编排模块。它不计算具体物理 tendency，而是在物理初始化入口里按固定顺序释放旧数组、重新分配共享模块数组，并写入第一批物理常数和时间控制状态。

源码注释说明这些工作过去由 inifis 完成。当前 3D GCM 路径中，iniphysiq_mod.F90 在 inigeomphy 之后进入 OpenMP 并行区，调用 phys_state_var_init，随后才调用 ini_fillgeom 填充几何三角缓存，再调用 conf_phys 读取物理配置。1D init_testphys1d_mod.F90 也复用同一个入口，在它之后执行 ini_fillgeom、conf_phys 和 initracer。

复现时要把本文件看作“分配和初始化顺序契约”：很多下游模块假定自己的数组已经按 ngrid/nlayer/nq/nslope 分配好，但几何值、restart 字段和运行开关仍由后续入口继续填充。

共享状态与副作用

  • 非 MESOSCALE 构建下，只有 is_master 进程直接打开 startfi.nc 查询 nslope 维度；随后 bcast(nslope) 把结果同步给所有进程。
  • MESOSCALE 构建下不读取 startfi.nc 的坡面维度，而是打印 subgrid slopes temporarily switched off in mesoscale, nslope=1 并强制 nslope=1。
  • 例程会连续调用大量 end_* 后接 ini_*，其副作用是释放旧分配并按新维度重建共享状态。复现初始化顺序时不能跳过这些释放调用，因为下游 ini_* 入口未必都能安全处理已分配数组。
  • 本文件只调用 ini_comgeomfi_h(ngrid) 分配 sinlat/coslat/sinlon/coslon 等缓存；实际几何值由调用方随后调用 ini_fillgeom 填入。
  • 源码直接调用 abort_physic，但本文件未显式 use abort_physic 所在模块；复现编译行为时应以当前构建系统的外部接口解析为准。

核心逻辑

  1. 把目标文件名固定为 startfi.nc。
  2. 非 MESOSCALE 下由 master 尝试打开 startfi.nc。文件不存在或没有 nslope 维度时设置 nslope=1；若 nslope 维度存在但无法读取长度，则调用 abort_physic 中止；读取成功时令 nslope=nslope_read。
  3. 非 MESOSCALE 下广播 nslope；MESOSCALE 下直接强制 nslope=1。
  4. 释放并初始化 tracer_mod，再初始化输出控制。
  5. 调用 init_comcstfi_h(prad,pcpp,pg,pr) 写入基础物理常数。
  6. 调用 init_time(day_ini,day_end,hour_ini,pdaysec,ptimestep) 写入物理日历和时间步状态。
  7. 按源码顺序释放/分配坡面、季节、地表、几何、土壤、辐射、长波、混合物属性、湍流、尘埃、化学、水云、非地形重力波、坡面分布、古气候和 tracer 质量修正相关模块数组。

伪代码

phys_state_var_init(ngrid,nlayer,nq, day_ini,day_end,hour_ini, pdaysec,ptimestep, prad,pg,pr,pcpp)
  filename = "startfi.nc"

  if not MESOSCALE:
    if is_master:
      if startfi.nc cannot be opened:
        nslope = 1
      else if dimension "nslope" is absent:
        nslope = 1
      else if dimension length cannot be read:
        abort_physic("phys_state_var_init", "nslope present but not readable", 1)
      else:
        nslope = nslope_read
    bcast(nslope)
  else:
    print mesoscale slope-disabled message
    nslope = 1

  end_tracer_mod()
  ini_tracer_mod()
  init_print_control()
  init_comcstfi_h(prad, pcpp, pg, pr)
  init_time(day_ini, day_end, hour_ini, pdaysec, ptimestep)

  for each shared-state module in source order:
    call its end_* routine
    call its ini_* routine with ngrid, nlayer, nq, and/or nslope

参与的主题流程

主题 参与方式
物理初始化 在 3D iniphysiq 和 1D init_testphys1d 中分配 phymars 共享状态，是 conf_phys 和 physiq 前的状态准备层
物理 restart 兼容 先从 startfi.nc 的 nslope 维度决定坡面相关数组大小，使后续 phyetat0_mod 读坡面字段时维度匹配
次网格坡面 初始化 slope_mod、surfdat_h、comsoil_h、dimradmars_mod、comslope_mod 和 paleoclimate_mod 的坡面维度相关数组
水云和 CO2/尘埃相关流程 分配 watercloud_mod、dust_param_mod、dust_rad_adjust_mod、compute_dtau_mod 等共享缓存，具体物理值由后续配置、restart 或物理步填充
非地形重力波 分配随机 GW 和 eddy-mix GW 的跨步记忆数组，restart 读取由 phyetat0_mod 负责

写法特点

  • 本例程是一个长初始化序列，源码顺序就是运行契约。新增模块若有共享数组，一般需要在这里补上对应 end_* / ini_* 调用。
  • nslope 在任何坡面相关数组分配前确定，避免 surfdat_h、comsoil_h、dimradmars_mod、comslope_mod 和 paleoclimate_mod 维度不一致。
  • 3D OpenMP 路径中本例程位于 !$OMP PARALLEL COPYIN(day_ini,day_end,hour_ini) 内；被初始化模块若含 threadprivate 状态，复现时需要保留调用上下文。

复现要点

  • 若 startfi.nc 存在且含 nslope 维度，坡面相关数组会按该长度分配；若文件缺失或无 nslope 维度，则默认只有一个坡面。
  • 若 startfi.nc 中存在 nslope 维度但 NetCDF 无法读取长度，初始化会中止，而不是回退到 nslope=1。
  • 单独调用 phys_state_var_init 只分配 comgeomfi_h 的几何缓存，不会填入经纬度三角函数；3D 和 1D 调用方都在之后调用 ini_fillgeom。
  • 1D 无 restart 路径后续可能通过 nslope 配置重新释放并分配 surfdat_h、comsoil_h、dimradmars_mod 和 paleoclimate_mod 的 slope-var 数组；不要把 phys_state_var_init 的初始 nslope 当作 1D 最终坡面维度的唯一来源。
  • ini_tracer_mod 与 1D 后续 initracer 不是同一层入口：本文件负责 tracer 模块初始化，1D 入口后续还会执行更具体的 tracer 初始场处理。

待确认

  • abort_physic 在本文件中未显式通过 use 导入；需以当前编译单元和接口声明方式确认是否依赖外部过程解析。
