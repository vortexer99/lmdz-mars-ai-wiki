文件定位

conf_phys.F 定义 conf_phys_mod 模块（第 1 行）和 conf_phys(ngrid,nlayer,nq) 子程序（第 7 行），是 Mars PCM 物理参数化的运行时配置入口。它不计算物理 tendency；它的职责是在物理初始化阶段读取 run.def/callphys.def 风格的配置键，写入 callkeys_mod、dust_param_mod、dimradmars_mod、datafile_mod、time_phylmdz_mod 等共享状态，并在同一个入口内执行大量依赖/互斥校验。

从源码结构看，配置流程分三段：非 MESOSCALE 分支先读取动力-物理共享调度键（第 101-129 行），随后检查并读取 callphys.def（第 133-1291 行），最后给长波辐射设置默认值并可选读取 radia.def 覆盖（第 1304-1381 行）。因此，复现实验时不能只保存 callphys.def；还要记录源码默认值、是否存在 radia.def、以及编译宏 MESOSCALE 对默认值和 bcast 路径的影响。

共享状态与副作用

  • conf_phys 会写入大量 SAVE/THREADPRIVATE 模块变量，尤其是 callkeys_mod 中的运行时开关。OpenMP 复现时要确认初始化流程在每个线程上的传播路径与源码一致。
  • callphys.def 缺失直接终止；radia.def 缺失不是错误，源码使用第 1308-1318 行的默认长波设置。
  • 它会打开、关闭 callphys.def 和 radia.def，并在非 MESOSCALE 路径广播文件状态和值。
  • 它在 naerkind 确定后调用 ini_scatterers(ngrid,nlayer)，这意味着辐射气溶胶数组大小取决于本入口读到的配置，而不是后续辐射模块自行决定。
  • 它含大量 abort_physic 配置守卫。复现失败时，很多错误不是数值发散，而是配置组合在初始化阶段被拒绝。

核心逻辑

  1. 进入 conf_phys 后，在非 MESOSCALE 分支读取物理输出和调用频率。废弃键 ecritphy 若存在会立即终止；outputs_per_sol 还必须整除 steps_per_sol。
  2. OpenMP master 检查 callphys.def 是否存在，非 MESOSCALE 时广播 ierr。不存在则在第 1289-1291 行终止。
  3. 对每个配置项先写源码默认值，再调用 getin_p。例如 iaervar=8、callrad=.true.、calldifv=.true.、callsoil=.true.、dustbin=0、water=.false.、co2clouds=.false.、callthermos=.false.。
  4. 读取并校验主物理开关：辐射/NLTE/NIR、垂直扩散、热羽流、干对流、Yamada4/ATKE、CO2 凝结、土壤、地形/非地形重力波、尘埃注入、古气候、水循环、CO2 云、吸附和热层。
  5. 尘埃逻辑先设置 dustbin/active/doubleq/submicron/lifting/freedust/dustscaling_mode，再校验组合。例如 active 需要 doubleq，submicron 需要双矩尘埃，freedust 需要 dustbin > 0，dustinjection=1 需要 lifting 和 freedust。
  6. dustiropacity 只接受 tes 或 mcs。tes 写 dustrefir=9.3E-6，mcs 写 21.6E-6；若启用 reff_driven_IRtoVIS_scenario，源码要求使用 tes 参考波长。
  7. 水/CO2 云逻辑读 sedimentation/activice/water/hdo/activeco2ice/CLFvarying/co2clouds/...，然后执行依赖检查：例如 activice 需要 water，activeco2ice 需要 co2clouds，microphys 需要 water 和 doubleq，scavenging 需要 microphys 且 dustbin >= 1。
  8. naerkind 读取后先检查 activice 至少需要两个 scatterer，再调用 ini_scatterers。随后按 doubleq/active、rdstorm、topflows、water/activice、co2clouds/activeco2ice、submicron/active 的组合填充 name_iaer。
  9. 高层大气逻辑读取 callthermos 及其子开关。若 callthermos=.false.，则 callconduct/calleuv/callmolvis/callmoldiff/thermochem 任一为真都会终止；最后还要求 photochem 和 callthermos 均依赖 water。
  10. 主配置完成后，设置长波默认值：callemis=.true., ilwd=1, ilwn=1, ilwb=1, lwxn_linear=.true., lwxn_ncouche=3, lwxn_alphan=0.4, semi=0。若 radia.def 存在，则按固定顺序读取并覆盖这些值。

伪代码

subroutine conf_phys(ngrid, nlayer, nq):
    if not MESOSCALE:
        reject obsolete ecritphy
        read outputs_per_sol, slow_diagfi, iphysiq, ecritstart
        require outputs_per_sol divides steps_per_sol

    check callphys.def exists; broadcast result when parallel
    if missing: abort

    for each callphys key:
        assign source default
        getin_p(key, variable)
        print selected value
        enforce local dependency and incompatibility rules

    read naerkind
    require active water ice has enough scatterers
    ini_scatterers(ngrid, nlayer)
    fill name_iaer according to dust, water, CO2, storm, topflow flags

    require thermosphere child switches depend on callthermos
    require photochem and callthermos depend on water

    assign longwave defaults
    if radia.def exists:
        read fixed-order longwave overrides
        broadcast overrides when parallel
end subroutine

写法特点

  • 固定格式 Fortran，很多字符串和 getin_p 调用跨行续写；用行号审查时要同时看续行。
  • 大多数默认值直接写在 getin_p 前一两行，因此“默认值”是源码事实，不是 callphys.def 文件事实。
  • 该文件把配置读取、默认值、互斥检查和少量初始化动作放在一个长子程序中；新增开关时容易同时影响多个主题。
  • radia.def 读取是固定顺序文本读取，不按键名读取；文件格式不匹配会错位。

复现要点

  • 保存 callphys.def 的同时记录源码版本，因为默认值和依赖检查写死在 conf_phys.F。
  • 检查运行目录是否存在 radia.def。存在时长波设置不再完全由源码默认值控制。
  • 记录 MESOSCALE 编译状态；例如 callstats、callslope 默认值和并行广播路径都受它影响。
  • 记录 naerkind 与 name_iaer 的派生结果。下游辐射路径依赖气溶胶名称顺序，不只是依赖开关真假。
  • 若启用微物理、清除、CO2 云、热层或尘埃注入，必须复核本文件中的依赖规则，否则运行会在初始化阶段终止。

待确认

  • radia.def 中 callg2d 被读取和广播，但源码注释称它是 obsolete unused flag；其历史用途需结合旧辐射路径确认。
  • semi 在 callkeys_mod 中的物理含义仍需结合长波半隐式路径进一步确认。
  • abort_physic 未在本文件显式 use，应来自上层接口或外部链接环境；本文只记录调用位置和错误语义。
