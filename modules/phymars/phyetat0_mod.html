文件定位

phyetat0_mod.F90 是物理初始状态读取和兜底初始化模块。它在 startphy_file=.true. 时打开 startfi.nc，通过 iostart 读取地表、土壤、坡面、云分数、热羽流、非地形重力波、tracer 表面储量、古气候和质量修正相关状态；在 startphy_file=.false. 时走无 startfi.nc 的学术/理想化初始化路径，给反照率、发射率、土壤地热通量和少量共享状态设置缺省值。

本文件不是物理倾向计算器，而是把 tabfi、iostart、soil_settings 和多个共享状态模块串起来，决定一次运行从哪一个物理初始场和 restart 记忆状态开始。

共享状态与副作用

  • tab_cntrl_mod 使用 SAVE，并声明为 !$OMP THREADPRIVATE(tab_cntrl_mod)；多线程运行时每个线程持有独立控制表副本。
  • phyetat0 在 startphy_file=.true. 时会打开并关闭 startfi.nc，大量读取 NetCDF 字段；缺少必需字段时调用 abort_physic 终止。
  • 函数会写入多个 module 级状态，而不只填调用方实参，包括 surfdat_h、comslope_mod、paleoclimate_mod、非地形重力波模块、尘埃模块和 tracer 质量修正模块。
  • no-startfi 路径不会调用 soil_settings，而是把 flux_geo(:,:)=0.；同时要求 nslope==1，否则中止。
  • 多时间轴 startfi.nc 通过 timestart 选择 indextime；缺省 -9999 使用最后一个 Time。

核心逻辑

  1. 根据 startphy_file 决定是否打开 startfi.nc，随后调用 tabfi。有 startfi 时从文件读控制表；无 startfi 时用空文件标识走理想化初始化。
  2. 如果 start 文件存在坡面定义，读取 def_slope 和 subslope_dist，归一化坡面分布并确定 major_slope；若无 startfi 且 nslope!=1，直接中止。
  3. 读取必需地表字段：phisfi、albedodat、ZMEA、ZSTD、ZSIG、ZGAM、ZTHE、tsurf、emis、q2 等。缺字段时按源码规则中止或给缺省值。
  4. 若存在 Time 维度，读取时间轴并用 timestart 选择 indextime；随后初始化 tab_cntrl_mod。
  5. 读取尘埃、云分数、热羽流、地表温度/反照率/发射率、粗糙度、湍流能量和非地形重力波记忆场。
  6. 若 call_mass_fixer_dyn 为真，读取 8 个主要大气成分的 mass_predyn_* 标量，只有字段存在且值大于 0 时才设置对应 found_startfi_*。
  7. 按 tracer 名称读取 qsurf：h2o_vap 读取 h2o_ice 地表 tracer，HDO 分支把 hdo_vap 映射到 hdo_ice，co2 优先读 co2ice，其他缺失非 CO2 tracer 清零。
  8. 读取 watercap 和 watercaptag。没有 watercap 时把负的 h2o_ice 地表储量转移到 watercap 后清零；没有 watercaptag 时设置 icelocationmode=4。
  9. 有 startfi 时调用 soil_settings 读取土壤状态；古气候字段存在时读取 h2oice_depth、coef_ssdif、co2ice_depth、perennial_co2ice，缺失时填源码缺省值。
  10. 关闭 startfi.nc，返回已填充的物理初始状态。

写法特点

  • 本文件是大型初始化例程，许多字段的缺省值和错误处理是 restart 格式契约的一部分；复现时不能只看变量名，需要逐字段确认 found 分支。
  • get_field 的 indextime 参数只在存在时间轴或相应字段按时间保存时有意义；timestart=-9999 表示选择最后一个可用时间。
  • tab_cntrl_mod 只填了固定槽位：1=nbp_lon、2=nbp_lat-1、3=nbp_lev、4=idayref、5=rad、6=omeg、7=g、8=mugaz、9=rcp、10=daysec、11=dtphys、27=hour_ini。

复现要点

  • startfi.nc 至少需要包含 phisfi、albedodat、ZMEA、ZSTD、ZSIG、ZGAM、ZTHE、tsurf、emis、q2；若定义了坡面字段，还需要可读的 subslope_dist。
  • hmons、summit、base 缺失时只有 rdstorm=.true. 才中止；否则三者清零。
  • totcloudfrac 分支按源码字面行为记录：CLFvarying 为真时尝试读取 totcloudfrac，但随后仍把 totcloudfrac(:)=CLFfixval。这看起来可能覆盖文件值，复现时应按源码执行并等待开发者确认意图。
  • 非地形 GW 读取块中 dv_nonoro_gwd 出现两次读取；df_eddymix_flx 的字段名字符串带尾随空格 "df_eddymix_flx "，这可能影响 restart 字段匹配。
  • mass_predyn_* 只有在 startphy_file .and. call_mass_fixer_dyn 时读取，且字段值必须大于 0 才视为有效。
  • watercap 缺失时，负的 h2o_ice 地表储量会被解释为永久水冰储库并转移到 watercap；负的 hdo_ice 地表储量会直接清零。
  • paleoclimate 字段缺省值包括 h2oice_depth=-999.、co2ice_depth=-999.、coef_ssdif=4.e-4；若最后一个纬度为南极，perennial_co2ice(ngrid,:)=10*1.6e3。
  • 无 startfi.nc 路径要求 nslope==1，并通过 surfalbedo_without_startfi、surfemis_without_startfi 提供地表反照率和发射率。

待确认

  • totcloudfrac 在 CLFvarying 分支中读入后又被 CLFfixval 覆盖，需确认是有意固定云分数还是遗漏了 else。
  • dv_nonoro_gwd 的重复读取块是否为拷贝残留。
  • get_field("df_eddymix_flx ",...) 字段名包含尾随空格，需确认 restart 文件中实际变量名是否也包含空格或 NetCDF 层是否会裁剪。
  • 局部变量 igcm_co2_tmp 在古气候块中被查找但后续未见实际使用，需确认是否为遗留逻辑。
