文件定位

comsoil_h 是 Mars GCM 土壤子系统的核心共享状态模块。它定义了土壤层数常数 nsoilmx=57、层深度几何、热惯量、热扩散求解器系数数组、地下 tracer（水汽/冰/吸附水）以及相关控制开关。所有涉及土壤热传导、水循环地下过程、初始场读写和诊断输出的文件都依赖此模块。它本身不包含物理计算逻辑，仅提供数据声明、参数常量和分配/释放子程序。

依赖的模块

无。comsoil_h 是纯声明模块，不 use 任何其他模块。

共享状态与副作用

comsoil_h 是纯粹的共享状态容器。所有变量均为 save 属性，部分标记 allocatable。所有可分配数组标记 !$OMP THREADPRIVATE 以支持 OpenMP 并行。

关键副作用：

  • ini_comsoil_h / ini_comsoil_h_slope_var 会分配数组；重复调用会导致内存泄漏（无已分配检查）。
  • end_comsoil_h / end_comsoil_h_slope_var 会释放数组；含 allocated() 保护。
  • volcapa 的值来自 start 文件 control(35)（在 tabfi.F 或 soil_settings.F 中设置），不同模拟可能不同。

核心逻辑

comsoil_h 不含物理计算逻辑，其结构为：

  1. 参数声明（行 6, 37–41）：nsoilmx=57 定义土壤层数；nqsoil=3 定义地下 tracer 数；igcm_h2o_vap_soil/ice/ads 定义 tracer 索引；porosity_reg=0.45 定义表土孔隙率。

  2. 土壤层几何（行 8–9）：layer(nsoilmx) 和 mlayer(0:nsoilmx-1) 分别存储层界面深度和层中点深度，由 soil_settings.F 计算。

  3. 热性质（行 10–12）：inertiedat（当前气候参考值）和 inertiesoil（含坡面变化的实际值）控制热扩散速率；volcapa 是体积热容量。

  4. 热扩散求解器数组（行 19–27）：tsoil 是土壤温度；mthermdiff/thermdiff 是扩散系数；coefq/coefd/alph/beta/mu 是隐式三对角求解系数，在 soil.F 的 firstcall 中计算。

  5. 地下 tracer（行 32–43）：qsoil(ngrid,nsoilmx,3,nslope) 存储三个 tracer（水汽、冰、吸附水）；adsorption_soil/choice_ads/ads_const_D/ads_massive_ice 控制吸附行为。

  6. 分配/释放（行 49–128）：四个子程序分别处理全量和坡面子集的分配与释放。ini_comsoil_h 在 phys_state_var_init 中调用；ini_comsoil_h_slope_var 在需要重新分配坡面维度时调用。

伪代码

module comsoil_h:
    constant nsoilmx = 57              # 土壤层数
    constant nqsoil = 3                # 地下 tracer 数
    constant igcm_h2o_vap_soil = 1     # tracer 索引
    constant igcm_h2o_ice_soil = 2
    constant igcm_h2o_vap_ads = 3
    constant porosity_reg = 0.45       # 表土孔隙率

    save layer(nsoilmx)                # 层深度
    save mlayer(0:nsoilmx-1)           # 层中点深度
    save inertiedat(ngrid,nsoilmx)     # 参考热惯量
    save inertiesoil(ngrid,nsoilmx,nslope)  # 实际热惯量
    save volcapa                       # 体积热容量
    save tsoil(ngrid,nsoilmx,nslope)   # 土壤温度
    save mthermdiff, thermdiff         # 热扩散系数
    save coefq, coefd, alph, beta, mu  # 求解系数
    save flux_geo(ngrid,nslope)        # 地热通量
    save qsoil(ngrid,nsoilmx,nqsoil,nslope)  # 地下 tracer
    save adsorption_soil, choice_ads, ads_const_D, ads_massive_ice  # 控制开关

    subroutine ini_comsoil_h(ngrid, nslope):
        allocate all soil arrays

    subroutine end_comsoil_h:
        deallocate all soil arrays (with allocated() guard)

    subroutine ini_comsoil_h_slope_var(ngrid, nslope):
        allocate slope-dependent subset only

    subroutine end_comsoil_h_slope_var:
        deallocate slope-dependent subset (with allocated() guard)

参与的主题流程

主题 参与方式
water-cycle 提供地下 tracer（qsoil）和吸附控制开关，是 soilwater/vdifc 的数据基础
radiation 间接：土壤热惯量影响地表温度，进而影响辐射

写法特点

  • 纯声明模块：不含物理计算逻辑，仅数据声明 + 分配/释放。
  • !$OMP THREADPRIVATE：所有 save 变量均标记线程私有，支持 OpenMP 并行。
  • 坡面子集分配：ini_comsoil_h_slope_var 允许在不重新分配 layer/mlayer/inertiedat/volcapa/coefq 的情况下重新分配坡面相关数组，用于 nslope 变化时。
  • volcapa 的特殊初始化路径：注释说明它从 start 文件 control(35) 读取（在 physdem 中），或通过 tabfi 设置，或在 soil_settings.F 中初始化。
  • 固定格式：部分使用大写关键字（MODULE/SUBROUTINE/END），混合 F77/F90 风格。

复现要点

  • nsoilmx=57 是硬编码常数，改变需要重新编译整个 GCM。
  • porosity_reg=0.45 是表土孔隙率，用于 soilwater 吸附计算。
  • choice_ads=3 表示无吸附（见 soilwater.F90）。
  • volcapa 的值取决于 start 文件，不同初始条件可能不同。
  • layer 和 mlayer 的具体数值由 soil_settings.F 计算，取决于 nsoilmx。

待确认

  • ini_comsoil_h 无 allocated() 保护，重复调用是否会导致内存泄漏（推断：是，但调用方 phys_state_var_init 通常只调用一次）。
  • mu 变量的用途未在本文件中注释，推断用于热扩散求解器（待确认：需查 soil.F）。
