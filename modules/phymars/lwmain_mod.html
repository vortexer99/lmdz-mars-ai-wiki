MODULE lwmain_mod

文件定位

lwmain_mod.F 是 LTE 长波热红外辐射主程序模块。它由 callradite_mod 在每个辐射子域内调用，接收已经准备好的层压厚、界面压力/温度、层温、气溶胶 IR 光学性质、地表发射率和地表 CO2 冰量；随后按固定顺序组织长波链：

lwb -> lwu -> (lwxd/lwxn/lwxb, 按 ilw* 频率刷新) -> lwflux -> lwi

本文件本身不实现 Planck 多项式、CO2 透过率、交换系数或散射通量细节；这些细节分别在下游文件页说明。lwmain 的关键职责是把这些步骤按正确状态顺序串联起来，决定何时刷新 yomlw_h::xi 交换率缓存，并把 lwflux 给出的显式冷却率替换为 lwi 的半隐式修正冷却率。

本文件只在 1:nlaylte 的 LTE 有效层内工作。nlaylte 由 nlthermeq 写入 yomlw_h，callradite 负责只把这部分层的输入复制进子域工作数组，并在返回后把更高层 dtlw/dtsw 置零。

定义的符号

符号 类型 行号 作用

本文件没有 SAVE 变量、没有 THREADPRIVATE 声明、没有 common block。持久共享状态由被调用模块和 yomlw_h 承载。

共享状态与副作用

  • 本文件不保存本地状态，不读写文件，不调用诊断输出 API。
  • lwxd/lwxn/lwxb 会按 icount 和 ilwd/ilwn/ilwb 的频率门控写入 yomlw_h 的 xi/xi_ground/xi_emis。如果本步没有触发某一类刷新，后续 lwflux/lwi 会继续使用上一次缓存的对应交换率。
  • lwflux 和 lwi 读取 yomlw_h::xi/xi_ground/gcp。gcp 由 callradite 首次调用时写为 g/cpp。
  • lwmain 会把 co2ice > 20.e-3 的列设置为 emis_gaz=1.，只影响传给交换系数例程的地表发射率；传给 lwflux/lwdiff 的仍是原始 emis。
  • 文件尾部有 xi 范围检查和新旧冷却率打印模板，但均被注释，不产生运行时诊断。

初始化 / firstcall / restart / diagnostic 行为

  • 本文件没有 firstcall 分支。旧版中 xi 初始化已移到 yomlw_h 的 ini_yomlw_h。
  • 本文件不读写 restart；它依赖 phys_state_var_init_mod 初始化 yomlw_h，依赖 nlthermeq 更新 nlaylte，依赖 callradite 提供 gcp 和子域输入。
  • 交换系数不是每次必定全量刷新：lwxd/lwxn/lwxb 分别受 ilwd/ilwn/ilwb 控制。icount=1 时三者都会因 mod(icount-1,ilw*)==0 触发。
  • 诊断分支均是注释代码；真正输出由上游 physiq/callradite 的通量和 tendency 数组管理。

核心流程

1. CO2 冰覆盖列的气体发射率

行 88-95 构造 emis_gaz：

if co2ice(jl) > 20.e-3:
  emis_gaz(jl) = 1.
else:
  emis_gaz(jl) = emis(jl)

这个派生量只传给 lwxd/lwxb 的地面反射/边界交换路径。lwflux 仍接收原始 emis，因此地表通量/带外扩散仍按调用方给定发射率处理。

2. Planck 函数和半层差分

行 104-105 调用 lwb。输出包括：

  • bsurf/btop：地表和大气顶谱带 Planck 函数。
  • blev/blay：层界和层中心谱带 Planck 函数。
  • dblay：层中心 Planck 温度导数，后续给 lwi。
  • dbsublay：每层拆成上下两个半层的 Planck 差分，后续给 lwdiff。

这些量必须在 lwflux/lwi 之前准备，因为交换率最终要乘 Planck 函数差，半隐式线性化要用温度导数。

3. CO2 吸收量与气溶胶 IR 总光学性质

行 112-117 调用 lwu。它把 dp/plev/tlay 转成 CO2 的 co2_u/co2_up 累积吸收路径，把多类气溶胶合成为：

tautotal(k,j,ir)
omegtotal(k,j,ir)
gtotal(k,j,ir)

同时为 CO2 带生成气溶胶吸收透过率 aer_t。这些数组分别供交换系数例程和 lwflux/lwdiff 使用。

4. 按频率刷新交换率缓存

行 128-155 是 lwmain 最重要的状态门控段：

if mod(icount-1, ilwd) == 0: call lwxd(...)
if mod(icount-1, ilwn) == 0: call lwxn(...)
if mod(icount-1, ilwb) == 0: call lwxb(...)

三类刷新覆盖不同 xi 区域：

  • lwxd：非相邻真实层之间的远距离交换。
  • lwxn：相邻真实层交换和底层 xi_ground。
  • lwxb：地面/空间边界交换。

lwflux 和 lwi 不关心本步是否刷新过，只读取 yomlw_h 当前缓存。因此复现某一步的长波结果时，必须知道前序步是否已经按这些频率更新过交换率。

5. 显式通量、净辐射收支和冷却率

行 160-167 调 lwflux。该例程合并两部分：

  • CO2 15 微米带内：读取 xi，乘 bsurf/blay/btop 的 Planck 函数差，得到净交换率贡献。
  • CO2 带外 IR：调用 lwdiff，用 tautotal/omegtotal/gtotal 和 dbsublay 计算红外散射扩散通量。

输出 netrad 和显式 coolrate = gcp * netrad / dp，同时写 fluxground 与 fluxtop。

6. 半隐式冷却率修正

行 188-201 调 lwi 并覆盖 coolrate：

call lwi(..., netrad, dblay, dp, newcoolrate)
for jkk = 1:nlaylte:
  coolrate(:,jkk) = newcoolrate(:,jkk)

这意味着 lwmain 的最终 coolrate 输出不是 lwflux 的显式冷却率，而是 lwi 根据 semi、dtphys*iradia、xi 和 dblay 求出的半隐式修正结果。netrad 不被 lwi 改写，仍表示 lwflux 计算的显式净辐射收支。

伪代码

lwmain:
  for each column:
    if co2ice > 20e-3:
      emis_gaz = 1
    else:
      emis_gaz = emis

  call lwb to compute Planck functions and gradients
  call lwu to compute CO2/aerosol absorber amounts and IR optical properties

  if mod(icount-1, ilwd) == 0:
    call lwxd to update distant-layer xi

  if mod(icount-1, ilwn) == 0:
    call lwxn to update neighbouring-layer xi and xi_ground

  if mod(icount-1, ilwb) == 0:
    call lwxb to update ground/space boundary xi

  call lwflux:
    read xi/xi_ground
    combine CO2 net exchange and non-CO2-band diffusion
    output explicit coolrate, fluxground, fluxtop, netrad

  call lwi:
    read netrad, dblay, dp, xi and timestep controls
    output newcoolrate

  for each LTE layer:
    coolrate = newcoolrate

参与的主题流程

主题 参与方式
辐射 LTE 长波主程序；输出长波冷却率、地表下行长波、顶层 OLR 和层净辐射收支。
气溶胶辐射 消费 aerosol 与 QIRsQREF3d/omegaIR3d/gIR3d，经 lwu/lwdiff 影响长波吸收、散射通量和冷却率。
CO2 循环 / 地表 CO2 冰 co2ice > 20.e-3 时把气体交换系数路径中的 emis_gaz 设为 1；CO2 15 微米带吸收路径由 lwu/lwtt 处理。
LTE/NLTE 边界 只覆盖 nlaylte 内的 LTE 长波层；高层 NLTE 冷却和 LTE/NLTE 混合由 physiq 其他路径处理。
地表能量收支 fluxground 是地表下行长波通量；fluxtop 是大气顶出射长波通量，供诊断和能量收支使用。

写法特点

  • 固定格式 Fortran 模块，subroutine lwmain 名称为小写，但调用方使用大小写不敏感的 lwmain。
  • 本文件是调度层，真正数值公式分散在 7 个被调用例程中；复现时不能只重写 lwmain，必须保留被调用例程的状态顺序。
  • kflev 用于接口数组维度，实际物理循环主要由共享 nlaylte 控制。
  • 交换率缓存有时间复用：ilwd/ilwn/ilwb 允许某些步沿用旧 xi，以降低长波计算成本。
  • coolrate 会被写两次：lwflux 写显式值，lwi 计算后由 lwmain 覆盖为半隐式值。

复现要点

  • 调用前必须完成 dimradmars_mod、yomlw_h 和气溶胶光学表初始化；callradite 必须已经设置 gcp=g/cpp。
  • nlaylte 必须已由 nlthermeq 更新，并且 kflev/nflev 数组维度要覆盖 nlaylte+1 层界。
  • 第一次辐射调用或任意需要重建交换率的调用必须让 icount 与 ilwd/ilwn/ilwb 触发对应刷新；否则 lwflux/lwi 会读取旧 xi。
  • co2ice 阈值是 20.e-3 kg.m-2；超过该值只改变 emis_gaz，不是全局改写 emis。
  • lwu 合成气溶胶光学性质时没有在本层做零保护，复现无气溶胶或极薄光学厚度情形要按源码检查除零风险。
  • 若只想得到显式长波冷却率，不能直接使用 lwmain 最终 coolrate，因为它已经被 lwi 覆盖；需要在 lwflux 调用之后、lwi 覆盖之前截取。

待确认

  • yomlw_h::xi 在本文件中只用于注释掉的检查块；当前 use yomlw_h, only: nlaylte, xi 中的 xi 是否仍有必要保留待源码维护者确认。
  • 局部变量 jk/ja 只出现在注释掉的检查块或未使用路径中；是否为历史调试残留待确认。
  • emis_gaz=1. 的 CO2 冰阈值物理依据不在本文件内说明；本页只记录源码字面行为。
  • netrad 在半隐式修正后不随 coolrate 更新，若下游同时诊断 netrad 和 coolrate，两者对应的是显式收支与半隐式倾向的组合。
