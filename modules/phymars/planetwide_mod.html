路径

libf\phymars

文件定位

planetwide_mod.F90 定义 planetwide_mod 模块，封装在整颗行星物理网格上做最大值、最小值和求和的工具接口。串行构建直接调用 Fortran intrinsic；CPP_PARA 并行构建先把本地物理列数组 gather 到全局物理网格，再由 master 进程做规约并用 bcast 广播标量结果。该模块不保存业务状态，主要为诊断、热羽流、山顶地形、tracer 质量修正和高层逃逸通量提供“全行星”意义上的规约值。

输入

输入 来源 类型 / 维度 含义
values(:) 调用方 integer 或 real 1D 数组 本地物理网格列上的待规约值；第一维对应本进程物理列。
values(:,:) 调用方 integer 或 real 2D 数组 本地物理网格列与第二维类别、层或 tracer 维度组成的待规约值。

输出

输出 去向 类型 / 维度 含义
values_max 调用方 integer 或 real scalar planetwide_maxval_* 返回全行星最大值。
values_max 调用方 integer 或 real scalar planetwide_minval_* 返回全行星最小值；形参名仍沿用 values_max，复现时不要据名称误判操作。
values_sum 调用方 integer 或 real scalar planetwide_sumval_* 返回全行星总和。

共享状态与副作用

  • 模块本身没有 SAVE 业务变量，也不读写文件。
  • CPP_PARA 分支会执行跨进程通信：本地数组先 gather 到 master，再把结果 bcast 回所有进程。
  • 整个模块主体包在 #ifndef MESOSCALE 内；MESOSCALE 构建下这些接口不会由本文件提供。

核心逻辑

  1. 每个泛型接口按实参类型和维度分派到 12 个具体例程之一。
  2. 串行路径直接对传入的 values 调用 maxval、minval 或 sum。
  3. 并行路径声明 values_glo(klon_glo[,size(values,2)])，调用 gather(values,values_glo)。
  4. 仅当 is_master 为真时，对 values_glo 做 intrinsic 规约。
  5. 调用 bcast 把标量结果发回所有进程，使调用方在任一进程上拿到一致结果。

伪代码

planetwide_operation(values, result):
    if CPP_PARA:
        values_glo = gather(values)
        if is_master:
            result = operation(values_glo)
        bcast(result)
    else:
        result = operation(values)

参与的主题流程

主题 参与方式
并行诊断 为 physiq 质量积分、山顶地形极值和逃逸通量提供跨进程一致的全行星规约。
热羽流 把局地热羽流层数需求转成全行星最大层数，用于后续数组范围和循环控制。
tracer 质量守恒 tracer_mass_fixer_dyn_mod 用全行星质量和构造修正因子，减小动力学传输后的质量漂移。
高层大气逃逸 moldiff_red 和 moldiff_MPF 都对本地 H/H2/D 逃逸通量数组求总通量，输出全行星逃逸量。

写法特点

  • 三组泛型接口覆盖 integer/real 与 1D/2D 数组，避免调用方显式选择具体例程。
  • 并行规约没有使用 MPI collective 名称，而是通过 LMDZ 物理并行层的 gather 和 bcast 封装。
  • 2D 例程只把第一维扩展到 klon_glo，第二维保留 size(values,2)；调用方需要保证第一维是本地物理列。

复现要点

  • 并行 GCM 中不要把这些调用替换成本地 maxval/minval/sum，否则只会得到当前进程子域的规约值。
  • 输入数组第一维应对应本地物理列；若传入其他布局，gather 的全局拼接语义会失真。
  • integer 求和结果仍是 integer scalar，极端大数组或大权重下需要注意溢出风险。
  • MESOSCALE 构建下本模块接口被预处理排除，调用方需要走相应 mesoscale 分支或避免依赖这些接口。

待确认

  • 各具体例程导入 dimphy::klon，但源码中未直接引用；是否为旧版数组接口残留或预处理分支依赖，需要后续全仓构建配置确认。
