文件定位

iostart.F90 定义 iostart 模块，是 LMDZ.MARS 物理重启文件（startfi.nc 和 restartfi.nc）的 NetCDF I/O 核心模块。它提供读取初始场（get_field/get_var）和写入重启场（put_field/put_var）的统一接口，支持 1D/2D/3D/4D 数组的读写，并处理并行环境下的数据分发和重排。

本模块是物理初始化（phyetat0）和重启输出（phyredem）的核心依赖，被 soil_settings.F、phyetat0_mod.F90、phyredem.F90 等文件调用。

共享状态与副作用

  • 模块级 SAVE 变量：
    • nid_start：startfi.nc 文件标识符
    • nid_restart：restartfi.nc 文件标识符
    • idim0~idim11：restartfi.nc 维度标识符
    • timeindex：当前时间索引
    • first_time1d：1D 模式首次调用标记
  • 这些变量由主进程维护，通过 bcast 广播到所有进程。
  • open_restartphy 会创建或打开 restartfi.nc 文件，并定义维度。
  • put_field_rgen 和 put_var_rgen 会修改 restartfi.nc 文件内容。

核心逻辑

  1. 文件打开：

    • open_startphy：主进程打开 startfi.nc，广播文件标识符
    • open_restartphy：首次调用时创建 restartfi.nc 并定义维度，后续调用直接打开

  2. 字段读取 (get_field)：

    • 主进程从 startfi.nc 读取全局数据
    • 按 ind_cell_glo 重排列顺序
    • 通过 scatter 分发到各进程

  3. 字段写入 (put_field)：

    • 各进程通过 gather 收集数据到主进程
    • 主进程按 ind_cell_glo 重排列顺序
    • 根据字段维度（1D/2D/3D）和时间依赖性选择 NetCDF 维度组合
    • 写入 restartfi.nc

  4. 变量读取 (get_var)：

    • 主进程从 startfi.nc 读取变量
    • 通过 bcast 广播到所有进程

  5. 变量写入 (put_var)：

    • 主进程将变量写入 restartfi.nc
    • 特殊处理 Time 变量（追加模式）

伪代码

iostart:
    nid_start, nid_restart: NetCDF file IDs
    idim0..idim11: dimension IDs for restartfi.nc
    timeindex: current time index

    open_startphy(filename):
        if is_master:
            NF90_OPEN(filename, nid_start)
        bcast(nid_start)

    get_field(field_name, field):
        if is_master:
            NF90_GET_VAR(nid_start, field_name, field_glo)
        reorder field_glo by ind_cell_glo
        scatter(field_glo, field)

    open_restartphy(filename):
        if is_master:
            if first_time:
                NF90_CREATE(filename, nid_restart)
                define dimensions (scalar, index, physical_points, ...)
            else:
                NF90_OPEN(filename, nid_restart)

    put_field(field_name, field):
        gather(field, field_glo)
        if is_master:
            reorder field_glo by ind_cell_glo
            NF90_REDEF(nid_restart)
            NF90_DEF_VAR(field_name, dimensions)
            NF90_PUT_VAR(field_glo)

    close_startphy():
        if is_master:
            NF90_CLOSE(nid_start)

    close_restartphy():
        if is_master:
            NF90_CLOSE(nid_restart)

参与的主题流程

主题 参与方式
物理初始化 phyetat0 调用 open_startphy 和 get_field/get_var 读取初始场
物理重启输出 phyredem 调用 open_restartphy 和 put_field/put_var 写入重启场
土壤初始化 soil_settings 调用 get_field 读取土壤参数
并行计算 所有 I/O 操作由主进程执行，通过 bcast/scatter/gather 分发数据

写法特点

  • .F90 自由格式模块，使用 Fortran 90 的 INTERFACE 和 MODULE PROCEDURE 实现泛型接口。
  • 使用 PRIVATE/PUBLIC 控制可见性，模块级变量默认私有。
  • 使用 SAVE 属性保存文件标识符和维度 ID。
  • 使用 #ifdef NC_DOUBLE 预处理指令选择 NetCDF 数据类型（NF90_FLOAT 或 NF90_DOUBLE）。
  • 特殊处理 1D 模式（klon_glo==1）的 restartfi.nc 创建。
  • 特殊处理 tsoil/inertiesoil 字段的 4D 写入（含坡面维度）。
  • timeindex 由 put_var_rgen 中的 Time 变量写入时自动更新。

复现要点

  • 必须确保 startfi.nc 文件存在且包含所需字段。
  • 并行环境下，所有 I/O 由主进程执行，数据通过 MPI 通信分发。
  • ind_cell_glo 必须正确初始化，用于重排列顺序。
  • restartfi.nc 的维度定义必须与写入的数据匹配。
  • timeindex 在每次写入 Time 变量时自动递增。

待确认

  • first_time1d 的特殊处理逻辑（行 480-484）是否正确处理了 1D 模式的重启。
  • put_field_rgen 中 tsoil/inertiesoil 的 4D 写入（行 900-905）是否正确处理了坡面维度。
  • #ifdef NC_DOUBLE 的默认行为（NF90_FLOAT）是否适用于所有编译环境。
