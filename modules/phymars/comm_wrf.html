文件定位

comm_wrf.F90 定义 comm_wrf，是 phymars 的 MESOSCALE 编译分支使用的 WRF 风格输出共享缓存模块。源码注释说明它用于输出那些尚未放入其他共享模块的诊断场。模块本身只声明、分配和释放数组；实际写入发生在 physiq_mod.F 的 #ifdef MESOSCALE 分支中。

复现上，这个模块不是物理过程入口，也不改变物理倾向。它把 physiq 已经算出的短波/长波加热率、短波地表向下通量、尘埃/水冰光学与半径、水汽/水冰柱量等复制到 comm_* 数组，供外部 WRF/mesoscale 输出接口读取。

共享状态与副作用

  • 所有 comm_* 数组都是 REAL,SAVE,ALLOCATABLE 模块变量。
  • 源码行 20-22 用 OpenMP THREADPRIVATE 声明全部 comm_* 数组。
  • allocate_comm_wrf 对每个数组使用 .not.allocated(...) 保护，因此重复调用不会重复分配；但若网格维度改变而数组仍已分配，本例程不会自动重分配。
  • deallocate_comm_wrf 没有 allocated(...) 保护；若数组未分配时调用，Fortran 运行时可能报错。
  • 全源码搜索只发现 physiq_mod.F 行 3260 调用 allocate_comm_wrf；未发现 deallocate_comm_wrf 的外部调用。
  • 本文件不直接写文件、不调用 write_output，也不执行物理计算。

核心逻辑

  1. physiq_mod.F 行 107-110 在 #ifdef MESOSCALE 分支中 use comm_wrf。
  2. physiq_mod.F 行 3254 进入 MESOSCALE 输出段，行 3260 调用 allocate_comm_wrf(ngrid,nlayer)。
  3. allocate_comm_wrf 逐个检查 11 个 comm_* 数组，未分配时按 ngrid 或 ngrid,nlayer 分配。
  4. physiq_mod.F 行 3263-3297 把已计算的物理诊断量复制到 comm_* 数组。
  5. 若 igcm_dust_mass 不为 0，comm_QSURFDUST 取 qsurf(:,igcm_dust_mass,1)；否则全设为 0。
  6. comm_MTOT 和 comm_ICETOT 分别取 mtot、icetot 并乘 1.e6/rho_ice。
  7. 若 igcm_h2o_ice 不为 0，comm_VMR_ICE = 1.e6*zq(:,:,igcm_h2o_ice)*mmean/mmol(igcm_h2o_ice)；否则全设为 0。
  8. 若 activice 为真，comm_TAU_ICE 取 taucloudtes；否则取 tauTES。

伪代码

module comm_wrf:
    declare WRF-style diagnostic arrays as allocatable SAVE
    mark all arrays THREADPRIVATE

allocate_comm_wrf(ngrid, nlayer):
    for each 2-D diagnostic:
        if not allocated, allocate(ngrid, nlayer)
    for each 1-D diagnostic:
        if not allocated, allocate(ngrid)

physiq MESOSCALE output block:
    call allocate_comm_wrf(ngrid, nlayer)
    copy shortwave/longwave heating rates
    copy surface downward shortwave flux
    copy dust optical depth and dust radius
    if dust mass tracer exists:
        copy surface dust mass
    else:
        write zero surface dust mass
    convert water vapor and water ice column masses by 1e6/rho_ice
    if water ice tracer exists:
        compute water ice VMR in ppm
    else:
        write zero VMR
    select active/inactive water ice optical depth source
    copy water ice particle radius

参与的主题流程

主题 参与方式
MESOSCALE / WRF 输出 在 #ifdef MESOSCALE 分支中为 WRF 风格 registry/state 输出缓存诊断数组
辐射诊断 缓存 zdtsw/zdtlw 加热率和 fluxsurf_dn_sw_tot 地表短波通量
尘埃诊断 缓存 tau_pref_gcm、rdust 和可选 qsurf(:,igcm_dust_mass,1)
水循环诊断 缓存 mtot/icetot 的微米等效厚度、VMR_ICE、TAU_ICE 和 rice

写法特点

  • 模块本体是自由格式 .F90，但使用 LMD 共享状态风格：SAVE,ALLOCATABLE 加 THREADPRIVATE。
  • physiq_mod.F 中对应输出注释采用 WRF Registry 风格，例如 state real HR_SW ikj misc ... "K/s"。
  • comm_* 数组名是输出接口名的 Fortran 缓存形式；实际输出变量名由 physiq_mod.F 注释给出。
  • 本模块没有 implicit none 放在模块层，只在两个子程序内部声明。

复现要点

  • 这些数组只在 MESOSCALE 预处理分支中由 physiq_mod.F 填充；普通 GCM 非 MESOSCALE 编译不会执行该输出缓存段。
  • allocate_comm_wrf 不会在维度变化时重分配已存在数组；复现变网格或多实例场景时要先确认释放/重新分配策略。
  • deallocate_comm_wrf 没有分配状态保护，不能在数组未分配时直接调用。
  • comm_ICETOT 的 WRF 注释写成 kg m-2，但赋值和 comm_MTOT 一样乘了 1.e6/rho_ice；页面按源码赋值标为 pmic，并把注释不一致作为复现风险。
  • 因为数组是 THREADPRIVATE，并行输出时需要确认写入线程与读取输出接口的线程一致或有明确同步。

待确认

  • 外部 WRF/mesoscale 输出接口如何读取这些 comm_* 数组不在 LMDZ.MARS/libf 搜索结果中体现；本页只记录 phymars 侧写入证据。
  • deallocate_comm_wrf 未在当前源码树中被调用，是否由外部框架或链接单元调用需结合完整 MESOSCALE 构建确认。
  • comm_ICETOT 的注释单位与赋值换算不一致，需由 WRF 输出约定确认最终展示单位。
