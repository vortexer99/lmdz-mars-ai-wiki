dyn1d/parallel_lmdz.F90

路径

LMDZ.MARS\libf\phymars\dyn1d\parallel_lmdz.F90

文件定位

dyn1d/parallel_lmdz.F90 在当前源码树中不是独立实现，而是一行相对路径：

../../../../LMDZ.COMMON/libf/dyn3dpar/parallel_lmdz.F90

它把 1D testphys1d 构建路径中的 parallel_lmdz 解析到公共并行动力运行时模块。目标模块负责 MPI/OpenMP 运行时状态、纬向带分配、halo 交换、broadcast/gather 和并行结束清理。当前 dyn1d 目录中直接使用该模块的源码点是 testphys1d.F90，只导入并调用 init_parallel。

初始化流程

init_parallel 的源码流程如下：

  1. 按编译宏设置运行模式：定义 CPP_MPI 时 using_mpi=.TRUE.，否则 .FALSE.；定义 CPP_OMP 时 using_OMP=.TRUE.，否则 .FALSE.。
  2. 调用 InitVampir 初始化性能计时钩子。
  3. 若 using_mpi，用 mod_const_mpi 提供的 COMM_LMDZ 调 MPI_COMM_SIZE 和 MPI_COMM_RANK；否则强制 mpi_size=1、mpi_rank=0。
  4. 如果 lunout 不是标准输入输出单元 5/6，非 root rank 打开 lmdz.out_XXXX 作为文本输出文件。
  5. 分配 jj_begin_para/jj_end_para/jj_nb_para(0:mpi_size-1)。
  6. 将 jjm+1 个纬向带尽量均分到所有 rank：每个 rank 先取整数商，前 MOD(jjm+1,mpi_size) 个 rank 多得一带。
  7. 若某 rank 的 jj_nb_para(i)<=1，打印错误；MPI 分支下调用 MPI_ABORT(COMM_LMDZ,-1,ierr)。
  8. 由累计纬向带得到每个 rank 的 jj_begin_para(i) 和 jj_end_para(i)，再写入当前 rank 的 jj_begin/jj_end/jj_nb。
  9. 计算扁平水平范围：ij_begin=(jj_begin-1)*iip1+1，ij_end=jj_end*iip1。
  10. 设置极点标志：mpi_rank==0 为北极 rank，mpi_rank==mpi_size-1 为南极 rank。
  11. OpenMP 分支中取 omp_size/omp_rank，默认 omp_chunk=ceil((llm+1)/omp_size)，再用 getin('omp_chunk',omp_chunk) 允许配置覆盖；非 OpenMP 时设 omp_size=1、omp_rank=0。

通信和分布算法

SetDistrib

SetDistrib(jj_Nb_New) 接收一个从 0 到 MPI_Size-1 的纬向带数量数组，直接覆写 jj_Nb_Para，然后从 rank 0 起逐段累加生成新的 jj_begin_para/jj_end_para。最后按当前 mpi_rank 重算 jj_begin/jj_end/jj_nb/ij_begin/ij_end。

复现时要注意：这个例程不重新分配 jj_*_para，调用前必须已经执行过 init_parallel，且 jj_Nb_New 的大小必须与当前 MPI_Size 一致。

exchange_hallo

exchange_hallo(Field,ij,ll,up,down) 只在 using_mpi 为真时工作：

  1. 先执行全局 barrier，再调用 VTb(VThallo) 开始 halo 计时。
  2. 默认允许向上/向下发送和接收。
  3. 若当前 rank 是北极，禁止向上发送和接收；若是南极，禁止向下发送和接收。
  4. 若 up==0，禁止向下发送和向上接收；若 down==0，禁止向上发送和向下接收。
  5. 每个启用的方向分配对应 buffer，Pack_Data 把 row*iip1*ll 个元素打包。
  6. 用 MPI_ISSEND 把北侧/南侧边界发送给 mpi_rank-1 或 mpi_rank+1，用 MPI_IRECV 接收相邻 rank 的边界。
  7. MPI_WAITALL 等待所有请求完成。
  8. 用 Unpack_Data 把收到的 buffer 写回 Field(ij_begin-up*iip1,1) 或 Field(ij_end+1,1) 位置。
  9. 调 VTe(VThallo) 结束计时，再执行一次 barrier。

Pack_Data 和 Unpack_Data 的循环次序是外层垂直层 l=1..ll，内层水平点 i=1..row*iip1，因此通信 buffer 以“每层连续水平行块”的顺序排列。

Gather_Field、AllGather_Field 和 Broadcast_Field

Gather_Field(Field,ij,ll,rank) 支持两种水平数组长度：

  • ij==ip1jmp1：包含 jjm+1 纬向行的全水平网格。
  • ij==ip1jm：少一个纬向行，源码用 min(jj_end,jjm) 避免南边界越界。

每个 rank 先把本地纬向带打包到 Buffer_send。root rank 根据所有 jj_begin_para/jj_end_para 构造 Recv_count 和 displ，调 MPI_GATHERV 后再逐 rank Unpack_Data 回完整 Field。非 root rank 为避免 debug 模式未分配实参报警，会分配一个长度为 1 的 dummy Buffer_Recv。

AllGather_Field 是 Gather_Field(...,0) 加 MPI_BCAST(Field,ij*ll,...)；Broadcast_Field 则直接从指定 rank 广播整个 Field。

结束流程

Finalize_parallel 先释放 jj_begin_para/jj_end_para/jj_nb_para。随后按编译宏和 type_ocean 选择关闭路径：

  • type_ocean == 'couple' 且有 CPP_XIOS：调用 wxios_close()。
  • type_ocean == 'couple' 且无 XIOS 但有 CPP_COUPLE：调用 prism_terminate_proto(ierr)；若返回值不是 PRISM_Ok，调用 abort_gcm。
  • 非 couple 且有 CPP_XIOS：调用 wxios_close()。
  • 非 couple 且有 CPP_MPI：若 using_mpi，调用 MPI_FINALIZE(ierr)。

dyn1d 使用路径

testphys1d.F90 在程序开头导入：

use mod_const_mpi,       only: init_const_mpi
use parallel_lmdz,       only: init_parallel

随后在主流程早期顺序调用：

call init_const_mpi
call init_parallel

因此 1D 主程序依赖 mod_const_mpi 先建立 COMM_LMDZ，再由 parallel_lmdz::init_parallel 建立 rank/线程状态。当前 dyn1d 源码未发现直接调用 Finalize_parallel、exchange_hallo、Gather_Field、AllGather_Field 或 Broadcast_Field 的位置；这些例程来自公共并行模块，供完整动力/物理并行路径或被链接的公共代码使用。

复现要点

  1. 当前 dyn1d 文件本身只有一行路径；实际行为以 LMDZ.COMMON/libf/dyn3dpar/parallel_lmdz.F90 为准。
  2. init_parallel 假设 mod_const_mpi 已经初始化了 COMM_LMDZ；在 testphys1d 中源码顺序确实是先 init_const_mpi 再 init_parallel。
  3. MPI 纬向分配要求每个 rank 至少两条纬向带，否则源码会 abort；这对小网格/多进程组合是硬约束。
  4. 非 MPI 编译时，using_mpi=.FALSE.，mpi_size=1，mpi_rank=0，通信例程主体不会执行。
  5. exchange_hallo 的 up/down 为 halo 行数；为 0 时源码会禁用对应方向的发送/接收，不是“交换 0 行后继续通信”。
  6. jjb_u/jje_u/... 和 ijb_u/ije_u/... 在本目标文件内没有赋值逻辑；它们是为了公共接口兼容保留的 dummy 状态。

待确认

  • 当前 1D 构建默认是否启用 CPP_MPI、CPP_OMP 或 CPP_XIOS，需要结合 makefile/编译脚本确认；本页只按源码条件分支记录。
  • Windows 工作树把 dyn1d 入口呈现为一行路径文本；原始类 Unix 工作树中它可能是符号链接。
  • Finalize_parallel 在 testphys1d 运行结束路径是否被其他公共代码调用，当前 dyn1d 直接源码未发现调用点。
