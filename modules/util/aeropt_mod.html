路径

所属目录/模块

util

文件定位

aeropt_mod.F90 是 util/aeroptical.F90 使用的气溶胶光学性质表模块。它不直接读写 GCM NetCDF 诊断场，而是读取外部 ASCII 光学性质表，把波长轴、粒径轴、消光效率 Qext 和单次散射反照率 omeg 缓存在模块变量中，并为 aeroptical 提供按目标波长和有效半径插值的接口。

该模块是离线气溶胶光学厚度计算链的支撑层：aeroptical 负责读取 GCM 变量和写 NetCDF 输出，aeropt_mod 负责光学表读取、查表插值、日志文件和缓存释放。

副作用

  • read_optpropfile 会分配并覆盖模块级缓存 wvl/radiusdyn/Qext/omeg；调用前若已有分配，本例程没有先释放。
  • 光学表文件不存在、维度行或数据行读取失败、参考波长越界时直接 stop。
  • 有效半径越界不会停止，而是向 aeroptlogfileID 对应日志写 warning，并把 Qext_val/omeg_val 设为 missval。
  • create_logfile 使用 STATUS='replace'，同名日志会被覆盖。

核心逻辑

读取光学性质表

  1. 用 INQUIRE(FILE=datadir//'/'//optpropfile) 检查文件是否存在。
  2. 打开 formatted ASCII 文件。
  3. 第一轮扫描跳过以 # 或空格开头的行，读取两个非注释数据项：
    • 第 1 项：nwvl；
    • 第 2 项：nsize。
  4. 分配 wvl(nwvl)、radiusdyn(nsize)、Qext(nwvl,nsize)、omeg(nwvl,nsize)。
  5. 第二轮继续跳过注释/空行，依次读取：
    • wvl(:)；
    • radiusdyn(:)；
    • nsize 行 Qext(:,isize)；
    • nsize 行 omeg(:,isize)。
  6. 关闭文件并打印波长、半径范围。

插值

  1. 在 wvl 中找到包围 wvl_val 的 iwvl1/iwvl2。若 wvl_val 超出表范围，打印错误并 stop。
  2. 在 radiusdyn 中找到包围 reff_val 的 isize1/isize2。
  3. 若 reff_val==missval，直接输出缺测。
  4. 若 reff_val 超出半径表范围，写日志 warning，输出缺测。
  5. 否则对 Qext 和 omeg 分别做两步线性插值：
    • 先在固定 iwvl1 和 iwvl2 上沿有效半径插值；
    • 再沿波长插值到 wvl_val。

伪代码

read_optpropfile(datadir, optpropfile):
  require datadir/optpropfile exists
  open formatted file
  skip comments/blank-leading lines
  read nwvl, nsize
  allocate wvl, radiusdyn, Qext, omeg
  skip comments/blank-leading lines
  read wvl(:)
  read radiusdyn(:)
  for each size: read Qext(:, size)
  for each size: read omeg(:, size)
  close file

interp_wvl_reff(wvl_val, reff_val, missval):
  find bracketing wavelength indices, stop if wavelength out of range
  if reff_val is missing: return missval
  find bracketing radius indices
  if radius out of range: log warning and return missval
  interpolate Qext and omeg in radius, then wavelength

参与的主题流程

主题 参与方式
util 气溶胶诊断链 为 aeroptical.F90 提供光学表读取和查表插值。
辐射/气溶胶后处理 将外部光学性质表转成每个格点有效半径对应的 Qext/omeg，供离线不透明度计算。

复现要点

  • ASCII 光学表必须按源码读取顺序组织；注释行只能用 # 开头，源码也跳过首字符为空格的行。
  • wvl 与 radiusdyn 必须按升序排列；插值查找逻辑假定数组递增。
  • wvl_val 必须在表范围内，否则程序停止；reff_val 越界只写日志并输出缺测。
  • 调用 interp_wvl_reff 前必须已经调用 read_optpropfile，否则模块数组未分配。
  • 处理多个 aerosol 时应在切换表前调用 end_aeropt_mod 或确保上一表缓存已释放。

待确认

  • read_optpropfile 内声明 iwvl 但未使用，属于无害冗余还是历史残留需确认。
  • use netcdf 在本文件中未见 NetCDF API 调用，是否仍需要该依赖需确认。
  • 注释称 create_logfile 的 Purpose 为 “Deallocate module variables”，与实际创建日志不一致。
  • 如果重复调用 read_optpropfile 而未先释放已分配数组，Fortran 对已分配 allocatable 再 ALLOCATE 的行为会导致运行时错误；调用方当前在 aerosol 循环末尾调用 end_aeropt_mod，但异常路径不会释放。

复现风险

  • 光学表格式稍有变化会导致读取错误或数据错位。
  • 半径越界不会中止，只会把该点光学性质设为缺测；后续 NetCDF 输出中可能出现空间缺口。
  • 日志文件使用固定 unit 范围，由调用方设置 aeroptlogfileID=100+iaer，多处并行或嵌套调用需避免 unit 冲突。
