solzenangle.F90

路径

LMDZ.MARS\util\solzenangle.F90

所属目录/模块

util

文件定位

solzenangle.F90 是一个离线 NetCDF 后处理程序，用于把 diagfi.nc、concat.nc 或 stats.nc 类输出中的 3D/4D 变量插值到指定太阳天顶角对应的本地真太阳时。它常用于抽取晨侧或昏侧的掩星/晨昏线条件，例如 sza=90 度时的地表终止线。

程序不改变经纬度网格或垂直坐标。核心流程是先按日期、经度、纬度和目标太阳天顶角反解本地时 LTsolzenangle(lon,lat,Time)，再沿输入文件的时间维对用户选择的变量做一维线性插值，输出 *_MO.nc 或 *_EV.nc。

solzenangle.def 示例依次给出 diagfi.nc、tsurf/ps/temp、空行、morning、90 和 0。其中注释说明 90 度代表地表终止线，>90 度可代表高空终止线条件。

输出

输出 去向 类型/维度 单位 含义
*_MO.nc 磁盘 NetCDF 文件 - morning 分支输出，输入文件去掉 .nc 后追加 _MO.nc。
*_EV.nc 磁盘 NetCDF 文件 - evening 分支输出，输入文件去掉 .nc 后追加 _EV.nc。
latitude / longitude / altitude 输出 NetCDF 1D 输入或固定属性 原坐标复制。
Time 输出 NetCDF unlimited 源码属性写成 calendar-like 字符串 非 stats 分支写整数 sol；stats 分支写 intsol*24。
GCM_layers / aps / bps / phisinit 输出 NetCDF 可选 - 从输入复制的辅助坐标和地形字段。
用户选择变量 输出 NetCDF 3D (lon,lat,Time) 或 4D (lon,lat,alt,Time) 输入 units 插值到目标 SZA 对应本地时后的变量。
LTsolzenangle 输出 NetCDF (lon,lat,Time) hours 每个经纬度、sol 上目标 SZA 对应的本地真太阳时；无解时为 1E+20。

副作用

  • 使用 NF_CLOBBER 和 NF_64BIT_OFFSET 创建输出，同名 _MO.nc 或 _EV.nc 会被覆盖。
  • 标准输入顺序严格依赖 solzenangle.def；交互输入与重定向输入必须保持同一顺序。
  • 所选变量整块读入内存，再整块写出；大文件会按 lon*lat*alt*time 放大内存压力。
  • 缺少必要维度/变量、时间轴为 Solar longitude、planetside 或 sza 非法、NetCDF 读写失败时会 stop。
  • 若目标 SZA 在极昼/极夜条件下无对应本地时，LTsolzenangle 和相应插值输出会写 missing。

核心逻辑

  1. 变量筛选：程序枚举输入文件中的所有变量，排除 Time/controle/rlonu/latitude/longitude/altitude/rlatv/aps/bps/ap/bp/soildepth/cu/cv/aire/phisinit 这 16 个字段；用户输入 all 时取其余变量，否则逐行读取变量名。
  2. 坐标和时间读取：要求存在 latitude、longitude、altitude、Time；controle 和 GCM_layers 可选。Time 的 long_name 或 title 若等于 Solar longitude，程序停止，因为后续插值需要 sol 类时间轴。
  3. SZA 参数检查：planetside 只能是 morning 或 evening；sza 必须在 [0,180) 度。
  4. 确定季节日期：
    • 有 controle 时用 int(int(ctl(4))+int(time(1)+ctl(27))) 推导 startsol。
    • 无 controle 时用 int(time(1))。
    • 若 timelen==12、time(1)==2、time(last)==24，识别为 stats 文件，把时间小时除以 24，nsol=1，并从 stdin 读取平均 Ls。
    • 其他文件按 nsol=int(time(last))-int(time(first)) 生成每个输出 sol。
  5. 反解目标本地时：对每个输出 sol、经度和纬度：
    • 晨侧先用 tmpsol=intsol+(6-lon/15)/24+startsol 估计用于计算 Ls 的日期。
    • 昏侧用 tmpsol=intsol+(18-lon/15)/24+startsol。
    • 非 stats 分支调用 sol2ls(tmpsol,tmpLs)。
    • 调用 LTsolzenangle(lat,tmpLs,planetside,sza,miss_lt,tmpLT) 得到本地真太阳时小时。
  6. 创建输出和复制辅助字段：initiate 写基本维度与坐标；init2 在可用时复制 aps/bps/phisinit。
  7. 逐变量时间插值：
    • 只处理 ndim>=3 的变量，ndim==3 按 (lon,lat,time)，ndim==4 按 (lon,lat,alt,time)。
    • 每个经度的输入本地时轴为 lt_gcm=time+lon/360。
    • 目标 LTsolzenangle 小时被换成 lt_outc=intsol+LT/24。
    • 单 sol 输入会在尾部追加周期点；多 sol 输入若目标点越界则把 lt_out 和 lt_outc 置 missing。
    • interpolf 对每条时间序列线性插值。
  8. 写输出变量和属性：变量保留输入 long_name/title 与 units；输入没有 missing_value 属性时，源码把 missing 设为 1E+20 并写回输出。
  9. 写 LTsolzenangle 和全局注释：输出目标 SZA 对应的本地时变量，并在全局 comment 中说明可由 LTsolzenangle、经度和 Time 组合推回经度 0 度的 sol。

太阳天顶角反解公式

LTsolzenangle 使用固定火星倾角 obliquity=25.1919 度，先计算太阳赤纬：

declin = asin(sin(Ls) * sin(obliquity))

然后由源码注释中的太阳天顶角方程反解时角：

cos(sza) = sin(lat)*sin(declin)
         + cos(lat)*cos(declin)*cos(2*pi*(localtime/24 - 0.5))

程序实际计算：

tmpcos = cos(sza)/(cos(lat)*cos(declin)) - tan(lat)*tan(declin)

若 tmpcos 超出 [-1,1]，视为极昼/极夜下无对应本地时，返回 missing。否则晨侧使用：

lt = 24 * (0.5 - acos(tmpcos)/(2*pi))

昏侧使用：

lt = 24 * (0.5 + acos(tmpcos)/(2*pi))

参与的主题流程

主题 参与方式
util 后处理链 常接在 concatnc、zrecast 或 localtime 类工具附近，用于把模型输出抽样到晨/昏侧指定 SZA。
观测/掩星比较链 sza=90 可模拟晨昏线或太阳掩星几何条件，输出的 LTsolzenangle 可检查每个格点实际抽样本地时。
太阳几何 与 solarlong 的主模式轨道换算不同，本文件内置 sol2ls 参数并直接用 SZA 几何公式反解本地时。

复现要点

  • 输入时间轴必须是 sol 类时间，不可先把 Time 轴替换为 Solar longitude。
  • solzenangle.def 中变量列表后必须保留空行，随后才是 morning/evening、sza 和参考时间。
  • 晨侧/昏侧只影响目标 SZA 的反解分支和输出后缀；morning 输出 _MO.nc，evening 输出 _EV.nc。
  • 非 stats 输入的输出时间数由 int(time(last))-int(time(first)) 决定；时间覆盖不足时会影响输出 sol 数。
  • 单 sol 输入会把首个时间点复制为周期尾点参与插值；多 sol 输入不会周期补齐，越界目标点会写 missing。
  • 目标 SZA 在高纬季节可能无晨侧或昏侧解，LTsolzenangle 会保留 missing，相关变量也会写 missing。

待确认

  • startsol = int(int(ctl(4))+int(time(1)+ctl(27))) 中 ctl(4) 与 ctl(27) 的语义需结合具体输出文件控制表确认。
  • stats 分支把输出 Time 写为 intsol*24，而 Time 属性仍使用 calendar-like 字符串；单位语义需按下游工具实际读取方式确认。
  • all 分支用 Nnotprocessed+1:nbvarfile 选择变量，隐含排除字段在变量表前部连续的假设；不同 NetCDF 变量顺序可能需要实测。
  • LTsolzenangle 在 cos(lat)==0 或 cos(declin)==0 时直接停止；极点和极端季节输入是否常见需结合实际网格确认。

复现风险

  • _MO.nc / _EV.nc 会覆盖同名文件。
  • 大 4D 变量整块读写，内存占用可能很高。
  • 文件名生成假定输入后缀为 .nc，通过 file(1:len_trim(file)-3) 截断。
  • 内置 sol2ls 常数与主模式 planete_h/solarlong 参数不完全共享，精确季节对齐需核验。
