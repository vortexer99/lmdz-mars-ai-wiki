gencol.F90

路径

LMDZ.MARS\util\gencol.F90

所属目录/模块

util

文件定位

gencol.F90 是 LMD Mars GCM 输出的离线柱含量计算工具。它读取 diagfi.nc、concat.nc、stats.nc 一类 NetCDF 文件中的四维 tracer 变量，根据变量单位判断输入是质量混合比、体积混合比还是数浓度，再沿 hybrid 垂直层积分出二维水平场随时间变化的柱质量，输出文件名为 input_col.nc。

程序只选择维度数为 4 且单位属于 kg/kg、kg.kg-1、mol/mol、mol.mol-1、cm-3 或 m-3 的变量。输出变量名为原变量名追加 _col，单位固定写为 kg/m2。

定义的符号

符号 类型 行号 作用
genCol program 1 主程序：读取输入文件、筛选 tracer、读取 hybrid 坐标和辅助场，计算并写出柱含量。
initiate internal subroutine 800 创建输出 NetCDF 文件，定义并写入 latitude/longitude/altitude/Time 坐标。
def_var internal subroutine 923 在输出文件中定义变量并写入 title、units 属性。
getmm internal subroutine 977 从变量名中解析物种名，并映射到分子量表；供 VMR 和数浓度转换使用。

依赖

输入

输入 来源 类型/维度 单位 含义
输入文件名 stdin / gencol.def NetCDF 文件名 - diagfi.nc、concat.nc、stats.nc 等。
变量列表 stdin / gencol.def all 或逐行变量名，空行结束 - 要积分的 4D tracer；all 会自动选择全部合格 tracer。
latitude / longitude / altitude / Time 输入 NetCDF 1D 随文件 输出坐标的来源。
ap / bp 输入 NetCDF，或 diagfi.nc / diagfi1.nc altlength+1 Pa / 无量纲 hybrid 层界面坐标，用于每层空气质量。
aps / bps 输入 NetCDF，或 diagfi.nc / diagfi1.nc altlength Pa / 无量纲 hybrid 层中心坐标，用于必要时计算压力。
ps 输入 NetCDF (longitude,latitude,Time) Pa 地表压力；所有积分分支必需。
rho 输入 NetCDF (longitude,latitude,altitude,Time) kg m-3 数浓度转质量必需，也可用于反推局地平均分子量。
mmean 输入 NetCDF (longitude,latitude,altitude,Time) 源码注释按 g mol-1 处理 VMR 转质量时的局地平均分子量；读不到时尝试计算。
temp / pressure 输入 NetCDF (longitude,latitude,altitude,Time) K / Pa 当 mmean 不可读且 rho 存在时，用理想气体关系反推 locmm。
tracer 变量 输入 NetCDF 4D kg/kg、mol/mol、cm-3 等 被积分的 tracer。

输出

输出 去向 类型/维度 单位 含义
input_col.nc 磁盘 NetCDF 文件 - 由 infile(1:len_trim(infile)-3)//"_col.nc" 生成，会覆盖同名文件。
latitude / longitude / altitude / Time 输出 NetCDF 1D degrees / km / days since... 从输入坐标复制或按 initiate 的固定属性写出。
<var>_col 输出 NetCDF (longitude,latitude,Time) kg/m2 每个选择 tracer 的柱质量。

副作用

  • 使用 NF_CLOBBER 创建输出，同名 _col.nc 会被覆盖。
  • 输入文件和变量列表完全由标准输入提供，批处理时应保存实际 gencol.def。
  • tracer、rho、temp、press、locmm 都按完整 (lon,lat,alt,time) 分配。
  • 缺少 ps、必要 hybrid 坐标、未知分子量、浓度分支缺 rho、NetCDF 读写失败时直接 stop。

核心逻辑

  1. 打开输入文件，遍历所有变量，打印维度为 4 且单位为 MMR、VMR 或数浓度的变量。
  2. 用户输入 all 或逐行列出变量；程序再次检查变量存在、维度为 4、单位可识别。
  3. 读取 latitude、longitude、Time、altitude。
  4. 优先从输入文件读取 ap/bp 和 aps/bps；若没有，则依次尝试当前目录的 diagfi.nc 和 diagfi1.nc。
  5. 读取必需的 ps，可选读取 rho。
  6. 优先读取 mmean。若读取失败且已有 rho，尝试用 temp 与 pressure 计算 locmm = rgp*rho*temp/pressure；若 pressure 不存在，则用 aps+bps*ps 回推；若仍不可行，则使用 cstmm=rgp/Rmean。
  7. 创建 *_col.nc 并写入坐标。
  8. 对每个选择 tracer 读取完整四维数组，按单位进入 MMR、VMR 或数浓度分支。

每层空气质量增量为：

incMass = ((ap(k)-ap(k+1)) + (bp(k)-bp(k+1))*ps(i,j,t)) / g

其中 g=3.72。MMR 分支累加 incMass*tracer。VMR 分支先用 getmm 得到物种分子量 tmpmm，常数平均分子量时累加 incMass*tracer*tmpmm/cstmm，局地平均分子量时累加 incMass*tracer*tmpmm/locmm*1.e3。数浓度分支要求 rho 和 getmm 成功，累加 incMass/rho*tracer*tmpmm/nav，nav=6.023d23；输入为 cm-3 时最终再乘 1.d6。

getmm 物种表

getmm 假设变量名格式为 *_name_* 或 *_name，用第一个和第二个下划线之间的字符串作为物种名。它覆盖 CO2/CO/O/O2/O3/H/H2/OH/HO2/H2O2/N2/CH4/Ar/N/NO/NO2、热层离子、水汽和冰、HDO 同位素等名称，最后把 g mol-1 转成 kg mol-1。

复现时要注意变量名必须让解析出的 varname 与表中字符串完全相等；否则 VMR 和数浓度分支会停止。

参与的主题流程

主题 参与方式
util 后处理链 可接在 concatnc、zrecast 或原始 diagfi/stats 输出之后，把 tracer 场转换为柱质量诊断。
气溶胶与化学诊断 对质量混合比、体积混合比或数浓度 tracer 统一输出 kg/m2 柱量，便于与光学厚度、柱含量或观测产品比较。

复现要点

  • 输入文件名必须以 .nc 结尾；输出名通过去掉最后 3 个字符再拼 _col.nc 得到。
  • 输入文件必须有 latitude、longitude、altitude、Time 和 ps。
  • ap/bp 长度按 altlength+1 使用，aps/bps 长度按 altlength 使用；若输入缺失，需要当前目录存在匹配的 diagfi.nc 或 diagfi1.nc。
  • 只有四维变量会被处理；二维/三维 tracer 即使单位匹配也不会进入候选列表。
  • 浓度变量必须同时有 rho，否则程序停止。
  • VMR 或浓度变量名要能被 getmm 解析出已知物种名。

待确认

  • 源码中用 ps(ilon,ialt,it) 计算回退压力，第三个索引位置按 ps(lon,lat,time) 声明看疑似应为 ps(ilon,ilat,it)；复现时按源码字面行为执行，需开发者确认意图。
  • 输出 altitude 的单位在 initiate 中固定写为 km，没有读取输入 altitude 的原始单位。
  • getmm 物种名比较大小写敏感，表中 Ar_N2 为混合大小写；实际变量名若使用 ar_n2 将无法匹配。
  • locmm 的单位处理依赖源码注释“mmean is stored in g/mol”，不同输入文件若单位不同需人工确认。

复现风险

  • _col.nc 会被 NF_CLOBBER 覆盖。
  • 大网格和长时间序列会一次性分配多个四维数组，内存占用较高。
  • 输入缺少 rho/temp/pressure/mmean 时 VMR 转换会退到常数平均分子量，物理误差取决于大气组成变化。
  • getmm 不识别新 tracer 名称时程序直接停止，需要扩展源码中的分子量表。
