Mars 运行参与度：必经 到 接口兼容。control_mod, logic_mod, temps_mod, comconst_mod 是 Mars GCM、MARS dynphy_lonlat/phymars/nogcm.F90、newstart/start2archive/testphys1d 共享的状态层；com_io_dyn_mod 在动力 history/dynav 输出打开时条件经过；conf_dat_m 是 Mars utility 和插值链可复用的数据方向标准化 helper。

模块职责

模块 状态类别 主要职责
control_mod SAVE 全局控制状态 运行长度、时间步频率、物理调用频率、输出频率、Mars ecritstart/timestart、planet_type、tracer/守恒/输出开关。
logic_mod 全局逻辑门控 Matsuno/leapfrog 当前阶段、是否调用 physics/dissipation/guide、是否读 start、垂直坐标类型和物理类型。
temps_mod 全局时间状态 itaufin/itau_dyn/itau_phy、起止日、参考日期、当前动力步长 dt、calendar 和 Mars hour_ini；history/dynav 输出状态深挖见 dynamic-output-state-holders。
comconst_mod 全局常量和维度 im/jm/lllm、dtvr/dtphys/dtdiss/daysec、行星半径/重力/气体常数/热容/自转、耗散和 sponge 参数。
com_io_dyn_mod 动力输出句柄 dyn_hist*.nc 文件名常量和 IOIPSL history id，由 inithist/initdynav 设置，由 writehist/writedynav 消费；file id 生命周期见 dynamic-output-state-holders。
conf_dat_m 无持久状态 helper conf_dat2d/conf_dat3d 把输入 lon/lat/pressure 数据调整为 COMMON 期望方向和单位。

写入和初始化顺序

  1. conf_gcm.F90 是 COMMON 6.3 gcm.F90 和 MARS dynphy_lonlat/phymars/nogcm.F90 共同消费的主配置写入点。它 USE control_mod, logic_mod, temps_mod, comconst_mod，通过 getin 设置 planet_type, calend, nday, ndynstep, starttime, timestart, ecritstart, day_step, iperiod, periodav, ok_guide, read_start, iflag_phys, iphysiq 等状态。
  2. COMMON gcm.F90 和 MARS dynphy_lonlat/phymars/nogcm.F90 在入口处调用 conf_gcm(99,.TRUE.)，随后检查 iphysiq 与 iperiod、day_step 与 iperiod/iphysiq 的整除关系。
  3. COMMON gcm / MARS dynphy_lonlat/phymars/nogcm 用 daysec/day_step 计算 zdtvr，必要时覆盖 dtvr；然后调用 iniconst。
  4. iniconst.F90 使用 control_mod 和 comconst_mod，把 dtphys=iphysiq*dtvr，并按 planet_type 选择垂直离散默认路径。
  5. COMMON leapfrog.F 和 MARS phymars/leapfrog_nogcm.F 用 nday/day_step/less1day/fractday/ndynstep 推出 itaufin，再在时间循环中维护 logic_mod 的 forward/leapf/apphys/apdiss 和 temps_mod%dt。
  6. dynetat0.F90 从 start.nc 的 controle 和 Time 恢复 daysec/dtvr/rad/g/omeg/itau_dyn 等状态；dynredem.F90 写回这些控制量，并在 Mars 路径写 Time。

Mars 主流程中的读写关系

COMMON 3D GCM 和 MARS nogcm

COMMON 6.3 gcm.F90 与 MARS dynphy_lonlat/phymars/nogcm.F90 共享以下状态层；后者不是 COMMON 6.3 源文件：

conf_gcm
  -> control_mod / logic_mod / temps_mod / comconst_mod
  -> day_step/iperiod/iphysiq consistency checks
  -> dtvr = daysec / day_step
  -> iniconst
  -> iniphysiq when iflag_phys enables physics
  -> COMMON leapfrog or MARS dynphy_lonlat/phymars/leapfrog_nogcm

Mars 运行中 planet_type=="mars" 会影响 leapfrog 的日期计算、ecritstart restart 写出、dynetat0/dynredem 的 controle 偏移和 Time 变量处理。MARS dynphy_lonlat/phymars/nogcm.F90 变体保留同样的状态层，但大部分动力 tendency 被关掉，主要用作 Mars lon-lat 物理驱动。

Mars utility

MARS phymars utility 直接消费这些 COMMON 状态：

Mars 文件 COMMON 状态
newstart.F control_mod: day_step/iphysiq/anneeref/planet_type, comconst_mod: lllm/daysec/dtphys/dtvr, temps_mod: day_ini/hour_ini。
start2archive.F control_mod: planet_type, comconst_mod: daysec/dtphys/rad/g/r/cpp, temps_mod: day_ini/hour_ini。
lect_start_archive.F comconst_mod: kappa/g/pi。
readhead_NC.F comconst_mod: im/jm/lllm/daysec/dtvr, temps_mod: day_ini。
ini_archive.F comconst_mod: daysec/dtvr/rad/omeg/g/cpp/kappa/pi, logic_mod: fxyhypb/ysinus。
callphysiq_mod.F90 control_mod: planet_type，用于检查物理包和行星类型是否一致。
testphys1d/init_testphys1d_mod.F90 logic_mod: hybrid，并通过 getin 设置 1D 运行的 day/time/vertical 相关配置。

这些消费者说明 control_mod/comconst_mod/temps_mod/logic_mod 不是单纯 3D 动力内部状态；它们也是 Mars 初始化、archive 转换和 1D 测试链的接口兼容层。

conf_dat_m 的数据方向契约

conf_dat2d 和 conf_dat3d 不保存状态。它们接受输入坐标和场数组，并原地修改输出坐标/数据：

  • 经度接受弧度 [-pi,pi] 或 [0,2*pi]，也接受角度 [-180,180] 或 [0,360]；角度会转换成弧度。
  • [0,2*pi] 或 [0,360] 经度会旋转到 [-pi,pi]。
  • 纬度若从南到北排列，会翻转成北到南。
  • conf_dat3d 若垂直坐标最大值小于 1200，把压力从 hPa 量级乘以 100 转为 Pa；若压力从低到高排列，会翻转为地面到模型顶。
  • interbar=.TRUE. 时，经纬坐标改成相邻点中点，用于 barycentric interpolation。

这个 helper 适合作为 Mars utility/interpolation 组页的支撑证据；它本身不是状态模块。

副作用和风险

  • 这些模块大多没有显式初始化默认值，除 conf_gcm 或 Mars defrun_new 写入外，直接读取会得到未定义或旧运行状态。
  • control_mod 变量带 SAVE，在长进程或工具程序复用时必须确认入口是否重新写入全部关键字段。
  • logic_mod 的 forward/leapf/apphys/apdiss 是时间循环临时状态，不应在 leapfrog* 外当作配置值解释。
  • temps_mod%dt 在 Matsuno 和 leapfrog 步之间变化：forward 阶段为 dtvr，leapfrog 阶段通常为 2*dtvr。
  • com_io_dyn_mod 的 history id 只有在对应 inithist/initdynav 调用后才有效。
  • conf_dat_m 遇到异常坐标只 WRITE(6,*) 报告，没有 abort；调用方需要自己判断输入是否合法。

复现顺序

要复现 Mars COMMON 状态层，最低顺序是：

  1. 通过 conf_gcm 或 Mars defrun_new 读取 run.def/gcm.def 风格配置，写入 control_mod, logic_mod, temps_mod。
  2. 确认 planet_type=="mars" 或目标 utility 所需的兼容行星类型。
  3. 由 COMMON gcm 或 MARS dynphy_lonlat/phymars/nogcm.F90 校验 day_step, iperiod, iphysiq，并设 dtvr=daysec/day_step。
  4. 调 iniconst 写入 dtphys、维度派生量和垂直离散相关状态。
  5. 如从 start/restart 继续运行，调用 dynetat0 读取 controle 与 Time，刷新 itau_dyn/daysec/dtvr 等状态。
  6. 进入 leapfrog* 后只把 logic_mod 的阶段门控解释为当前时间步状态。
