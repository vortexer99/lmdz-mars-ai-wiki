iniconst / conf_planete 行星常数初始化链

源码：

  • LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3d_common\iniconst.F90
  • LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3d_common\conf_planete.F90
  • LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3d_common\comconst_mod.F90

状态：TASK-0183 完成。本文只覆盖 COMMON 6.3 的行星常数、时间步和垂直离散入口；Mars nogcm.F90/newstart.F/tabfi.F 只作为 MARS 侧消费者和边界上下文，不视为 COMMON 6.3 源文件。

结论

COMMON 6.3 有两条行星常数进入动力核的路径：

  1. 有 start.nc 的主流程：dynetat0("start.nc") 从 controle/tab_cntrl 读取 rad/omeg/g/cpp/kappa/daysec/dtvr/pa/preff，写入 comconst_mod 和 comvert_mod；随后 gcm 用 daysec/day_step 重算 dtvr，再调用 iniconst 派生 dtphys/r/pi/unsim 并建立垂直坐标。
  2. 无 start.nc 或理想化场路径：iniacademic 调 conf_planete，后者用 getin 读取行星半径、重力、气体常数、热容、自转和日长等常数；随后 iniacademic 计算 dtvr=daysec/day_step 并调用 iniconst。

因此，conf_planete 不是常规 Mars restart 运行的唯一入口；它是理想化/academic 初始化路径的常数来源。常规 Mars/GCM restart 路径首先信任 start.nc 的 controle 元数据，再由 run.def 的 day_step/iphysiq/nsplit_phys 决定动力步长和 physics 步长。

conf_planete 默认值和配置键

conf_planete 先给出 Earth-like 默认值，再允许 run.def/getin 覆盖。默认值本身不代表 Mars production 值；Mars 值通常来自 start.nc/startfi.nc 的 controle 数组或 Mars physics 默认。

dynetat0 的 restart 常数路径

gcm.F90 在读取 run.def/conf_gcm、初始化 tracer 后，如果 read_start 为真，会调用 dynetat0("start.nc",...)。dynetat0 打开 start.nc，读取 controle 数组；当 planet_type 为 generic 或 mars 时，将文件识别为 planeto-like start，使用 idecal=4：

tab_cntrl(idecal+1)  -> rad
tab_cntrl(idecal+2)  -> omeg
tab_cntrl(idecal+3)  -> g
tab_cntrl(idecal+4)  -> cpp
tab_cntrl(idecal+5)  -> kappa
tab_cntrl(idecal+6)  -> daysec
tab_cntrl(idecal+7)  -> dtvr
tab_cntrl(idecal+13) -> pa
tab_cntrl(idecal+14) -> preff

dynredem 写 restart 时使用同一组偏移把这些值写回 controle。这意味着 start/restart 文件是 Mars/Generic restart 运行的常数快照，不只是场变量容器。

iniconst 派生逻辑

iniconst 本身不读取 rad/g/omeg/daysec，也不调用 conf_planete。它假定这些值已经来自 dynetat0 或 conf_planete/iniacademic，然后做以下派生：

  1. 从 dimensions.h/paramet.h 写入维度状态：im=iim, jm=jjm, lllm=llm, imp1=iim, jmp1=jjm+1, lllmm1=llm-1, lllmp1=llm+1。
  2. 用当前 dtvr 和 control_mod: iphysiq 计算 dtphys=iphysiq*dtvr。
  3. 计算 unsim=1./iim 和 pi=2.*asin(1.)。
  4. 用 r=cpp*kappa 派生气体常数；若 cpp/kappa 来自 restart 或 conf_planete 覆盖，这里只做一致的乘积。
  5. 按 planet_type 设置垂直离散默认：earth 默认 disvert_type=1，其他行星默认 disvert_type=2。
  6. 允许 getin('disvert_type',...) 覆盖默认值。
  7. 设置 pressure_exner = disvert_type == 1，再允许 getin('pressure_exner',...) 覆盖。
  8. disvert_type==1 调 disvert()；disvert_type==2 调 disvert_noterre；其他值调用 abort_gcm。

这也是 Mars/Generic 默认进入 disvert_noterre 的原因：默认分支来自 planet_type，但最终仍可被 run.def 覆盖。

主流程顺序

COMMON 串行 gcm.F90 和并行 gcm.F 的顺序一致：

  1. parse_args() 处理 runtime option。
  2. conf_gcm(99,.TRUE.) 读取 run.def，其中 day_step 默认 240，iphysiq/iperiod/nsplit_phys 等也在这里确定。
  3. 若 read_start，调用 dynetat0("start.nc",...)，常数来自 restart controle。
  4. 若 .not.read_start，调用 iniacademic，常数来自 conf_planete/getin。
  5. 入口检查 day_step 必须能整除 iperiod 和 iphysiq。
  6. 计算 zdtvr=daysec/REAL(day_step)，若旧 dtvr 不同则打印警告，再执行 dtvr=zdtvr。
  7. 物理开启的常规路径 iflag_phys==1 下，设置 dtvr=zdtvr 并调用 iniconst、inigeom、inifilr。
  8. 调用 inidissip，耗散时间步 dtdiss 后续由 dissip_period*dtvr 得到。
  9. 若 physics 开启，iniphysiq(... daysec, day_ini, dtphys/nsplit_phys, ..., rad,g,r,cpp,iflag_phys) 把 COMMON 常数传给 MARS physics 初始化入口。

Mars 侧 LMDZ.MARS\libf\dynphy_lonlat\phymars\nogcm.F90 也复用同一套 COMMON 初始化接口：它计算 zdtvr=daysec/REAL(day_step)，在 iflag_phys==1 时调用 iniconst/inigeom/inifilr，然后把 daysec, dtphys/nsplit_phys, rad, g, r, cpp 传给 iniphysiq。这说明 Mars nogcm 是 COMMON 状态的消费者，不是 COMMON 6.3 的源文件。

Mars utility 边界

Mars newstart.F 和 tabfi.F 说明了另一个方向的常数同步：

  • newstart.F 从动力 controle 中读取 kappa 等 COMMON 动力常数。
  • newstart.F 调 Mars physics tabfi，从 start_archive.nc 或 startfi.nc 的 controle 读取 p_rad/p_omeg/p_g/p_mugaz/p_daysec/time。
  • tabfi.F 在 nid==0 时给 Mars 默认值：rad=3397200., daysec=88775., omeg=4*asin(1.)/daysec, g=3.72, mugaz=43.49, rcp=.256793。
  • tabfi.F 在读取文件时将 tab_cntrl(tab0+5..11) 映射为 rad/omeg/g/mugaz/rcp/daysec/dtphys，并把 p_omeg/p_g/p_mugaz/p_daysec/p_rad 作为输出参数返回。

因此 Mars utility 和 physics 的 mugaz/rcp 状态属于 MARS physics 层；COMMON 动力层对应的是 cpp/kappa/r。两者数值要通过 start/startfi/archive 文件和初始化调用保持一致，但模块名和职责不同。

复现检查清单

复现 Mars/Generic 常数初始化时按这个顺序检查：

  1. run.def 中的 planet_type, read_start, day_step, iphysiq, nsplit_phys, disvert_type, pressure_exner。
  2. 若 read_start=y，检查 start.nc:controle 的 planeto-like 偏移：rad/omeg/g/cpp/kappa/daysec/dtvr/pa/preff。
  3. 若无 start，检查 conf_planete 支持的 getin 键，尤其 radius/g/molmass/kappa/cpp/daysec/daylen/year_day/omeg。
  4. 确认 gcm 或 Mars nogcm 是否把 dtvr 重设为 daysec/day_step。
  5. 确认 iniconst 后 dtphys=iphysiq*dtvr，而 iniphysiq 收到的是 dtphys/nsplit_phys。
  6. 对 Mars utility，另查 startfi.nc/start_archive.nc:controle 和 tabfi 的 rad/omeg/g/mugaz/rcp/daysec/dtphys。

风险和待确认

  • molmass 在 COMMON 注释中写作 molar mass，comconst_mod 注释标为 (g/mol)，而 conf_planete 的 cpp=8.3145/molmass/kappa*1000. 体现了单位换算。本文只记录源码行为，不替源码统一单位命名。
  • conf_planete 的默认值是 Earth-like；Mars production 运行不应仅凭这些默认值判断火星常数。
  • 具体算例若同时修改 run.def、start.nc 和 startfi.nc，需要以实际运行文件核对动力层 cpp/kappa/r 与 Mars physics 层 mugaz/rcp 是否一致。
