leapfrog_p 状态契约

页面状态：TASK-0126 完成。 源码范围：

LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3dpar\gcm.F
LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3dpar\leapfrog_p.F

Mars 运行参与度：条件经过。Mars 并行 GCM 构建经过 dyn3dpar/gcm.F -> leapfrog_p；串行 Mars GCM 和 MARS phymars/nogcm.F90 不经过本并行主循环。COMMON 6.3 已删除 COMMON nogcm，本页只讨论 dyn3dpar 并行入口。

入口调用点

dyn3dpar/gcm.F 在完成 conf_gcm、MPI/OMP 初始化、band/halo 初始化、tracer 分配、dynetat0 或 iniacademic、iniconst/inigeom/inidissip、iniphysiq、history/restart 初始化后进入 OpenMP parallel 区：

gcm.F 556: c$OMP PARALLEL DEFAULT(SHARED)
gcm.F 567: CALL leapfrog_p(ucov,vcov,teta,ps,masse,phis,q,time_0)
gcm.F 569: c$OMP END PARALLEL

源码 557-564 行保留了 temps_mod 和 logic_mod 的 OpenMP COPYIN 注释，说明这些状态曾经或应当作为 threadprivate 复制对象处理；当前实际 parallel directive 是 DEFAULT(SHARED)，没有启用这些 COPYIN 行。

入口实参

leapfrog_p 的入口形状在源码 7-8 行：

SUBROUTINE leapfrog_p(ucov,vcov,teta,ps,masse,phis,q,time_0)
实参 维度 读写 含义
ucov ip1jmp1,llm inout U 网格纬向 covariant wind。
vcov ip1jm,llm inout V 网格经向 covariant wind。
teta ip1jmp1,llm inout 位温。
ps ip1jmp1 inout 地表压力，Pa。
masse ip1jmp1,llm inout 层空气质量，由 p/pressure 同步更新。
phis ip1jmp1 inout 地表位势。
q ip1jmp1,llm,nqtot inout advected tracer；第三维来自 infotrac:nqtot。
time_0 scalar in 源码注释写 “not used”，但 Mars restart 分支实际用 time_0-floor(time_0) 修正 restart 时间。

这些数组仍使用全局 COMMON 维度形状，而不是显式局部数组形状。并行局部范围由 parallel_lmdz 的 ij_begin/ij_end、jj_* 分布和 SetDistrib 当前状态解释。

入口工作数组

leapfrog_p 大量工作数组是 SAVE 状态，跨调用保留；这与 OpenMP parallel 区共同存在时要特别小心。

数组 维度 作用
p, pks, pk, pkf ip1jmp1,llmp1 或 ip1jmp1,llm 层界 pressure、地表和层中 Exner。
phi, w, temp, tsurpk ip1jmp1,llm 位势、垂直速度、温度和 cp(T) 相关缓存。
pbaru, pbarv, flxw ip1jmp1/ip1jm,llm caldyn 给 tracer 和输出使用的质量通量。
ucovm1/vcovm1/tetam1/psm1/massem1 与状态数组同形 上一时间层状态，供 leapfrog/integrd 使用。
du/dv/dteta/dp/dq 与状态数组同形；dq allocatable 动力 tendency。
dufi/dvfi/dtetafi/dpfi/dqfi 与状态数组同形；dqfi allocatable physics tendency。
dudis/dvdis/dtetadis wind/theta 形状 dissipation tendency。
dvfi_tmp/dufi_tmp/dtetafi_tmp/dpfi_tmp/dqfi_tmp iip1 局部经向切片；dqfi_tmp allocatable physics band 切换前后暂存本 rank 非极点切片。
ecin/ecin0/dtetaecdt/vcont/ucont/vnat/unat 与风/温度形状 耗散守恒修正和诊断。

dq, dqfi, dqfi_tmp 在 262-272 行只由 OpenMP master 在线程第一次调用时分配，然后 barrier 同步。它们的第三维固定为当前 nqtot，因此 infotrac_init 必须在 gcm.F 调用 leapfrog_p 前完成。

初始化契约

进入主循环前，leapfrog_p 先建立运行终止步和初始 thermodynamic state：

  1. 237-250 行由 nday/day_step/less1day/fractday/ndynstep 计算 itaufin 和 itaufinp1。
  2. 252-254 行设 itau=0，并由 iflag_phys 推出局部 physics 布尔值；iflag_phys==0 或 2 时 physics 关闭。
  3. 262-272 行按 nqtot 分配 tracer tendency 数组。
  4. 278-296 行 master 清零 dissipation、physics、top-bound、tracer 和 mass-flux tendency。
  5. 298-303 行由 pression(ap,bp,ps,p) 和 pressure_exner 选择 exner_hyb 或 exner_milieu 初始化 p/pks/pk/pkf。

这个初始化发生在并行区内部，但多处由 c$OMP MASTER 包裹。复现并行问题时要同时检查 master-only 初始化、barrier 和后续各线程看到的共享状态。

并行边界状态

虽然入口实参是全局形状，运行时只应操作当前分布对应的局部范围：

  • gcm.F 在进入 leapfrog_p 前已 SetDistrib(jj_Nb_Caldyn)。
  • leapfrog_p 内在首次调整和阶段切换时多次 SetDistrib，例如 596 行回到 jj_nb_caldyn，1001/1042/1075 行在 physics 前后切换 jj_nb_Physic 与 jj_nb_Physic_bis。
  • 输出和 restart 前会 Gather_Field，如 1691-1701、1764-1773 行，把字段收集到 root/rank 0 写出。
  • physics tendency 在 calfis_p 后通过 Register_Hallo/SendRequest/WaitRequest 合并边界，再用 _tmp 数组叠加回本 rank 切片。

因此 leapfrog_p 的状态契约不是“全局数组直接全局写”，而是“全局形状数组 + 当前 band/halo 约定 + root gather 写出”。

Mars 边界

Mars 并行 GCM 经过本页路径时，关注点是 planet_type=="mars" 下的日期和 restart 时间：

  • leapfrog_p 的 physics 日期仍按 Mars/generic 分支构造 jD_cur/jH_cur。
  • Mars restart 分支没有统一使用 JH_cur。中途 restart 在 1822-1830 行使用 REAL(itau)/REAL(day_step)+time_0-floor(time_0) 或再减 day_end-day_ini；最终分支在 2047-2053 行使用 REAL(itau)/REAL(day_step) 或再减 day_end-day_ini。JH_cur 是非 Mars fallback 分支。
  • 这也是为什么 time_0 不能简单按源码入口注释理解为 “not used”。

复现检查表

  1. 确认 dyn3dpar/gcm.F 已完成 infotrac_init 和 q 分配，nqtot 与 traceur.def 一致。
  2. 确认进入 leapfrog_p 前当前分布是 jj_Nb_Caldyn。
  3. 检查 dq/dqfi/dqfi_tmp 是否首次调用时按当前 nqtot 分配；长进程复用时尤其要小心 SAVE firstcall。
  4. 对 pressure/masse 问题，先查 ps -> pression -> p -> massdair_p 的刷新顺序。
  5. 对 physics tendency 问题，检查 SetDistrib(jj_nb_Physic)、calfis_p、Register_Hallo 和 _tmp 回填。
  6. 对输出或 restart 问题，确认 Gather_Field 已在 root 写出前执行，且 Mars restart 时间分支按 time_0 修正。

待确认

  • 源码注释称 time_0 ! not used，但 Mars restart 分支实际使用 time_0-floor(time_0)；注释与代码不一致。
  • gcm.F 中 logic_mod/temps_mod 的 OpenMP COPYIN 行被注释，当前 shared/threadprivate 语义需结合编译选项和模块声明确认。
  • SAVE 工作数组和 allocatable tracer tendency 在同一进程内多次调用 leapfrog_p 时是否需要重分配，源码未显式处理 nqtot 变化。
