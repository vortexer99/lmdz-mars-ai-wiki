6.3 关键变化

范围 6.3 当前事实 Wiki 影响
构建入口 makegcm 已删除；Mars 构建应围绕 makelmdz_fcm。 构建页和任务口径不再释放 makegcm 路径任务。
FCM 配置 bld.cfg 不再包含 src::evolution %EVOLUTION，配置名使用 %{ARCH}%{SUFF_DIR}。 evolution/PEM 从范围排除变为“6.3 源码不存在”。
运行参数 dyn3d/gcm.F90、dyn3dpar/gcm.F 新增 program_options:parse_args()。 主入口页、符号索引和任务表需要新增 runtime option/build provenance 任务。
生成版本模块 makelmdz_fcm 生成 libf/misc/version_control.F90。 --version、编译命令、用户、主机、当前目录和 VCS 状态需要单独成页或组页。
COMMON nogcm libf/dyn3d/nogcm.F90 和 libf/dyn3d/leapfrog_nogcm.F 已删除。 删除/退役 COMMON nogcm 页面和任务；Mars 专用 nogcm 不在 COMMON 6.3 正文展开。
restart 时间 leapfrog/leapfrog_p 写 restart 时不再用旧 start_time 主线；Mars 分支使用 itau/day_step/time_0/day_end 组合，非 Mars fallback 使用 JH_cur。 时间推进、restart、dynredem 页面必须按分支口径返工。
restart 元数据 dynredem* 新增 controle_descriptor、descriptor、description_size、put_char1 等。 restart I/O 页需要按 6.3 重写 NetCDF 元数据契约。
MPI/OMP transfer transfer 模块新增 5D/6D broadcast，OMP buffer 逻辑修正。 mpi-data-transfer、omp-data-transfer 组页和 hard transfer 任务需按 6.3 更新。
物理 getin ioipsl_getin_p_mod 改用 is_master。 getin/XIOS/history 边界页需集中说明 master abstraction。
parallel_lmdz omp_chunk 会避开某些线程无工作导致的 MPI_Waitall 错配。 并行主题和 parallel_lmdz 页面需加入 hybrid run 修复。
geometry 输出面积 新增 cell_area_for_lonlat_outputs 和 init_geometry_cell_area_for_outputs。 geometry/inigeomphy 输出面积页需要拆出 physics area 与 lon-lat output area。
misc 新工具 新增 job.F90、program_options.F90、readTracFiles_mod.f90、strings_mod.f90。 misc 总览、tracer、runtime option、job auto-exit 需要新增任务。

立即退役的 COMMON 旧页面

  • modules/dyn3d/nogcm.md：目标 COMMON 源文件在 6.3 已删除。
  • routines/leapfrog-nogcm-differences.md：目标 COMMON 源文件在 6.3 已删除。

这些内容不应继续被根入口、文件索引、符号索引或覆盖矩阵作为当前 COMMON 页面引用。
