实用版 wiki 验收清单

页面状态：复验版，按功能面判断，不按任务数量单独判断。 源码范围：LMDZ.COMMON-6.3\LMDZ.COMMON，仅覆盖 Mars run 会经过或必须保持接口兼容的 COMMON 路径。 统计时间：2026-06-24 复扫。

验收原则

实用版 wiki 的目标不是把 registry 里的每个任务清零，而是让读者能沿 Mars 运行链复现和定位问题：

编译命令 -> arch/env/path -> FCM source groups -> 可执行文件名
        -> run.def/start 文件 -> gcm/newstart/testphys1d
        -> 动力初始化/时间推进 -> 物理接口
        -> restart/history/getin/I/O -> Mars 工具链

因此任务数量只作为覆盖密度指标。真正的验收依据是：主流程是否串起来、关键模块是否能定位、Mars 不经过的路径是否被排除、剩余待确认项是否被正确分诊为运行、外部库、设计意图或物理/数值问题。

当前总判断

结论：实用版已达标。

原因：

  • 主流程、构建配置、Mars/COMMON 映射、start/restart/archive 边界、串并行动力骨架、物理接口骨架和小工具组页已经可用。
  • 构建配置闭环已完成：源码路径、IOIPSL、arch/env、deftank 运行环境、target 输出命名都能互相追踪。
  • history/XIOS/getin 输出与配置读取职责边界已经补齐，能区分动力 history、Mars physics diagfi/stats、XIOS helper 和 getin/getin_p。
  • 登记表内可完成的 hard 父页已收口；剩余 blocked 项为非 Mars 范围或 6.3 删除项，剩余 待确认 属于运行验证、配置样例、外部库、设计意图或物理/数值解释。

最小完成条件

实用版当前已经满足以下最小条件：

  1. 已完成 history/XIOS/getin 输出和配置读取职责边界收口页。
  2. 入口页、file-index、coverage matrix 能从每个功能面跳到对应页面。
  3. 非 Mars 路径仍为 blocked，不因任务数量目标重新打开。
  4. hard 父页已收口；剩余 待确认 已集中到 pending-confirmation-triage。

可后置项目

这些任务可以提高完整度，但不应阻止实用版验收：

  • 单个短小 helper 的逐文件页，已有组页覆盖时可后置。
  • 非 Mars 路径：Venus/Titan newstart/archive/restart、evolution/PEM。
  • 需要真实运行样例、外部库行为或维护者设计意图的 待确认 项。
  • 平台专有 scheduler 指令细节，源码无法完全证明，需要真实集群文档或运行环境确认。

复验流程

每次声称“实用版完成”前，按以下顺序检查：

  1. 从 index 的阅读路径进入，确认主流程、构建、I/O、映射表都能到达。
  2. 从 source-coverage-matrix 查看是否还存在 未达标 功能面。
  3. 从 pending-confirmation-triage 查看剩余待确认项是否仍属于运行/外部/设计意图类。
  4. 从 file-index 随机反查 makelmdz_fcm、gcm.F90、calfis.F、dynredem*、getin、write_field 等关键范围。
  5. 抽查 registry：不能出现已 done 但无页面/报告的任务；不能把 blocked 非 Mars 任务重新算作缺口。
