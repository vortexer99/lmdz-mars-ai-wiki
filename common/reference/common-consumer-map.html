COMMON 使用方映射表

页面状态：初版，面向 COMMON 文件反查 MARS 使用点。
源码范围：LMDZ.COMMON-6.3\LMDZ.COMMON 与 LMDZ.MARS。 用途：写 COMMON 文件页时，快速定位 MARS 侧的直接消费者、路径转发和相关复现链路。

使用方式

  • 直接转发 表示 MARS 文件内容直接指向 COMMON 文件。
  • USE/CALL 表示 MARS 源码中搜索到模块或例程使用点。
  • 同名变体 表示 MARS 与 COMMON 中其他行星目录存在同名流程文件，不能自动视为相同实现。

优先级建议

已完成的横向契约页：

  • Mars dyn1d COMMON 直接转发契约：覆盖 phymars/dyn1d 14 个一行转发文件和 1D/testphys1d 接口兼容边界。
优先级 先写 COMMON 页面 原因
P0 infotrac.F90, control_mod.F90, comconst_mod.F90, comvert_mod.F90, logic_mod.F90 MARS phymars 和 dyn1d 都直接依赖，且影响运行配置和复现。
P1 disvert_noterre.F, inigeom.F, defrun.F, leapfrog/nogcm 相关页 垂直离散、初始化和时间推进链需要这些状态模块支撑。
P2 parallel_lmdz.F90, mod_const_mpi.F90, mod_hallo.F90, bands.F90 MARS 1D 也通过 COMMON 并行抽象保持接口兼容。
P3 mod_interface_dyn_phys.F90, inigeomphy_mod.F90, calfis.F/calfis_p.F 动力-物理接口是 MARS 与 COMMON 的关键边界。
P4 write_field, NetCDF95/ioipsl wrappers 输出和 restart 主题页需要这些底层证据。

维护规则

  • 每完成一个 COMMON 文件页，在本表对应行的“是否需要专页”列补实际页面状态。
  • 发现新的 MARS USE/CALL 使用点时，追加到“COMMON 文件到 MARS 使用方”表。
  • 发现构建系统层面的依赖时，单独标注为 build-provider，不要和源码 USE 混写。
  • phyvenus、phytitan.old、evolution/PEM 等 非 Mars 路径 不作为 consumer-map 后续任务释放对象。
