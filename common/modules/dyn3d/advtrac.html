dyn3d/advtrac.F90

页面状态：TASK-0164 完成。

源码路径：LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3d\advtrac.F90

Mars 运行参与度：条件经过。串行 Mars 3D 动力时间步在 leapfrog -> caladvtrac -> advtrac 链中进入本文件；只有累计步数 iadvtr == iapp_tracvl 时才真正执行 tracer advection。当前 6.3 infotrac_init 标准白名单只允许 iadv=0/10/14，因此 Mars 标准可达的串行输送主要是 Van Leer vlsplt 和第 1 个 tracer 的饱和水汽 vlspltqs 路径。PPM、Prather、advn、pentes_ini 分支保留在源码中，但标准初始化通常会在进入本文件前阻断对应 iadv。

文件角色

advtrac.F90 是串行动力 tracer advection 调度器，不是具体数值格式内核。它负责：

  • 累计多个动力步的水平质量通量 pbaru/pbarv。
  • 在一轮 tracer 输送的第一个动力步保存空气质量 massem。
  • 到第 iapp_tracvl 步时调用 groupe 生成 pole-grouped 通量和垂直通量 wg。
  • 计算 flxw、DP/P 风险诊断和每日 CFL 诊断。
  • 对 iq=1,nqperes 的父 tracer 按 iadv(iq) 选择具体输送格式。
  • 在 ok_iso_verif 时执行输送前后 isotope 一致性检查。
  • 完成实际输送后把内部累计计数 iadvtr 复位为 0。

并行构建不使用本文件；对应源码是 dyn3dpar/advtrac_p.F90 中被 leapfrog_p 调用的 advtrac_p，其当前可达数值调度集中在 vlspltgen_p。

参数契约

SUBROUTINE advtrac(pbaru,pbarv,p,masse,q,iapptrac,teta,flxw,pk)
参数 Intent 维度 说明
pbaru IN (ip1jmp1,llm) 当前动力步 U 点水平质量通量。
pbarv IN (ip1jm,llm) 当前动力步 V 点水平质量通量。
p IN (ip1jmp1,llmp1) 层界压力；饱和路径 vlspltqs 使用。
masse IN (ip1jmp1,llm) 当前空气质量；首个累计步复制到 massem。
q INOUT (ip1jmp1,llm,nqtot) tracer mixing ratio/state；具体内核原地更新。
iapptrac OUT scalar 当前累计步号，等于内部 iadvtr。
teta IN (ip1jmp1,llm) 位温；饱和路径使用。
flxw OUT (ip1jmp1,llm) 实际输送轮次的垂直质量通量诊断，wg/REAL(iapp_tracvl)。
pk IN (ip1jmp1,llm) Exner 相关状态；饱和路径使用。

注意：当 iadvtr 尚未达到 iapp_tracvl 时，本例程不会设置 flxw 新值，也不会更新 q；它只累计 pbaruc/pbarvc 并返回 iapptrac。

持久状态和工作数组

名称 属性 维度 作用
iadvtr SAVE, DATA 0 scalar 当前 tracer advection 累计步数；达到 iapp_tracvl 后复位。
pbaruc, pbarvc SAVE U/V 通量维度 累计多个动力步的水平质量通量。

cflxmax/cflymax/cflzmax SAVE (llm) 每日最大 CFL 诊断缓存。

pbaruc/pbarvc 只在 iadvtr==0 时清零；如果外部调用序列被打断，下一次调用会继续累积上一次未完成轮次的通量。

控制流

1. 初始化和累计

if iadvtr == 0:
  pbaruc = 0
  pbarvc = 0

for each level:
  pbaruc += pbaru
  pbarvc += pbarv

if iadvtr == 0:
  massem = masse

iadvtr = iadvtr + 1
iapptrac = iadvtr

massem 保存的是本轮输送开始时的空气质量，之后即使后续动力步的 masse 变化，本轮 tracer advection 仍以 massem 作为输送前质量基准。

2. iapp_tracvl 门控

if iadvtr != iapp_tracvl:
  return

只有门控满足时才进入实际输送。iapp_tracvl 由 conf_gcm 设置，默认等于 iperiod，也可由 getin("iapp_tracvl") 覆盖。

3. 通量整理与诊断

达到门控时：

groupe(massem,pbaruc,pbarvc) -> pbarug,pbarvg,wg
flxw = wg / REAL(iapp_tracvl)

随后源码计算每层 DP/P 估计：

zdp = pbarug(i-1)-pbarug(i) - pbarvg(j-1)+pbarvg(j) + wg(l+1)-wg(l)
zdp = zdp * dtvr / massem
if max(abs(zdpmin),abs(zdpmax)) > 0.5:
  print WARNING DP/P

CFL 诊断分别按通量正负选择上游质量：

CFL 正通量分母 负通量分母
cflx masse(ij,l) masse(ij+1,l)
cfly masse(ij,l) masse(ij+iip1,l)
cflz masse(ij,l) masse(ij,l-1)

countcfl 每次实际输送增加 iapp_tracvl；当 countcfl==day_step 时按垂直层写出 L, CFL[xyz]max，随后 countcfl=0。

4. 按 iadv 分派

实际输送前先调用 massbar(massem,massebx,masseby)，然后在 ok_iso_verif 条件下检查 q。主循环只遍历 iq=1,nqperes，即由 infotrac 判定为直接由空气输送的父 tracer。

PPM 保留分支

PPM 分支在单文件内占用较长代码段，复现时要注意它不是直接调用 ppm3d 就结束：

adaptdt(iadv,dtbon,n,pbarug,massem)
检查垂直 CFLmaxz
interpre(q,qppm,wg,fluxwppm,massem,apppm,bpppm,massebx,masseby,pbarug,pbarvg,unatppm,vnatppm,psppm)
for indice = 1..n:
  ppm3d(..., IORD,JORD,KORD by iadv)
interpost(q,qppm)

interpre/interpost 负责 LMDZ 与 PPM3d 的数组、垂直层序和通量格式转换；只测试 ppm3d 内核不能代表 advtrac 的 PPM 分支。当前 6.3 初始化通常阻断 11/16/17/18，所以这段主要用于历史兼容或本地补丁审查。

Mars 边界

  • Mars 串行动力若启用 tracer，会通过 leapfrog -> caladvtrac -> advtrac 条件经过本文件。
  • 本文件本身不读取 planet_type；Mars/Earth 后处理差异在调用者 caladvtrac 中处理。
  • 本文件直接消费 infotrac 的 nqtot/nqperes/iadv，因此 traceur.def 与 infotrac_init 是复现入口。
  • 标准 Mars 6.3 可达输送格式应先按 iadv=10/14/0 判断；旧格式分支存在不等于可达。
  • 并行 Mars 构建应读 advtrac_p 和 vlspltgen_p，不要把本页的串行局部数组和循环范围套到并行 band/halo 上。

复现检查点

  1. 必须连续模拟从 iadvtr=0 到 iapp_tracvl 的多次调用；只截取最后一步会漏掉累计通量。
  2. massem 来自本轮第一个累计步，不是实际输送步的当前 masse。
  3. flxw 是 wg/iapp_tracvl，不是原始 wg。
  4. nqperes 只覆盖父 tracer；子 tracer 的更新由具体内核的 family 逻辑处理。
  5. ok_iso_verif 检查不改变正常数值路径，但会引入额外输出和 abort 风险。
  6. massbar 即使在标准 iadv=10/14 路径中也会被调用；它主要服务保留 PPM 分支。
  7. CFL 诊断使用当前实参 masse，而通量 grouping 使用 massem；复现时不要混淆。
  8. iadvtr 是 SAVE 状态，单元测试之间需要重新加载程序或显式设计调用序列。
