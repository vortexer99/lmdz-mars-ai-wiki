文件定位

gcm.F90 定义串行 PROGRAM gcm，负责完整串行 GCM 的启动序列：检查 run.def，读取运行配置，初始化 tracer、初始状态、动力常量、几何、耗散、物理接口、restart/history 输出，最后调用 leapfrog 进入时间推进。

Mars 运行参与度

条件经过：COMMON 串行完整 GCM 入口。Mars 源码树中主要可见 phymars/nogcm.F90 同类入口；本文件是否作为 Mars 生产入口需结合构建配置确认。

共享状态与副作用

程序写入或依赖 control_mod、temps_mod、logic_mod、comconst_mod、infotrac、comgeom.h/comgeom2.h 中的全局状态；会打开或写出 restart.nc、history 文件、XIOS 状态；会因配置不一致调用 abort_gcm 或 STOP。

核心逻辑

  1. 初始化本地状态和默认输出文件名。
  2. 用 INQUIRE 确认 run.def 存在；缺失则 abort。
  3. 调用 conf_gcm(99,.TRUE.) 读取控制参数，并检查 iphysiq 是 iperiod 的倍数。
  4. 禁止串行 use_filtre_fft，按编译宏初始化 XIOS。
  5. 初始化 Cp(T)、日历和 tracer，分配 q。
  6. 如果 read_start，可先用 iniacademic 补 SW/Newtonian 常量，再用 dynetat0("start.nc",...) 读初态；否则直接 iniacademic。
  7. 检查 day_step 与 iperiod/iphysiq 的整除关系，重算 dtvr。
  8. 处理 raz_date 和 restart 日期差异，设置 Julian day。
  9. 若 iflag_phys==1，调用 iniconst、inigeom、inifilr。
  10. 调用 inidissip、计算 day_end，初始化物理、restart 和 history 输出。
  11. 设置 offline 存储频率，调用 leapfrog。

伪代码

require run.def
conf_gcm()
reject invalid periods and serial FFT filter
initialize optional XIOS, Cp(T), calendar, tracers
allocate q from nqtot
if read_start:
    maybe initialize academic constants
    read start.nc through dynetat0
    maybe read relax.nc
else:
    build academic initial state
validate day_step, dates, and dtvr
if physics enabled:
    initialize constants, geometry, filter, physics
initialize dissipation, restart, history outputs
leapfrog(state, time_0)

参与的主题流程

主题 参与方式
dynamics-time-integration 完整串行入口，最终调度 leapfrog。
restart-io 读取 start.nc，写 restart.nc，初始化 history。
dynphys-interface 传入动力几何和常量调用 iniphysiq。

写法特点

该文件是自由格式 Fortran 90，但仍通过 include 使用固定格式时代的公共头文件。大量控制流由预处理宏 CPP_IOIPSL、CPP_XIOS、CPP_PHYS 决定。

复现要点

  • ecritstart>0 且 planet_type=="mars" 时，初始 dynredem0 使用 day_ini，否则使用 day_end。
  • nday<0 表示运行 -nday 个动力步，ndynstep>0 又可覆盖结束日计算。
  • read_start=.false. 会走 iniacademic，这会改变几何/常量初始化来源。
