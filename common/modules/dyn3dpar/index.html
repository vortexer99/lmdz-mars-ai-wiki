dyn3dpar 总览

路径

LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3dpar

页面状态：TASK-0005 hard-index 完成。当前目录有 87 个源码文件。

职责

dyn3dpar 是 COMMON 并行动力目录。它在串行 dyn3d 主链的基础上加入 MPI/OpenMP 初始化、纬向 decomposition、stage-specific band 分布、halo/swap 通信、root/domain I/O 和并行动力-物理接口约束。Mars 串行 3D 或 Mars nogcm 类路径不经过本目录；Mars 并行动力构建、并行入口验证或并行物理接口测试会经过这里。

最短阅读路径：

  1. gcm：并行入口，说明 conf_gcm -> init_const_mpi -> init_parallel -> bands/halo -> leapfrog_p。
  2. conf_gcm：并行配置读取，覆盖 adjust/use_filtre_fft/use_mpi_alloc 和并行 sponge。
  3. parallel_lmdz、bands、mod_hallo：理解 decomposition、band 和通信。
  4. leapfrog_p：并行时间推进、band 切换、physics、耗散、输出和 restart。
  5. 串行/并行动力对照主题页：从串行公式对照并行边界。

并行基础契约

dyn3dpar 的可复现性主要依赖三个基础契约：

  1. parallel_lmdz 给出当前 rank 的纬向范围、极点所有权和局部一维索引。
  2. bands 为 caldyn、Van Leer、dissip、physics 分别提供 stage-specific 分布；leapfrog_p 运行中会频繁 SetDistrib。
  3. mod_hallo 负责跨 rank halo 和重分布请求；复杂字段必须先注册，再发送/等待/解包。

因此排查并行数值差异时，不应先假设公式错误。更稳的顺序是：确认当前 stage 的 band、确认 ij_begin/ij_end 和极点所有者、确认 halo 请求宽度和时机，再进入具体算子。

Mars 范围边界

  • Mars 串行 3D 正常入口可不经过本目录；本目录是 Mars 并行动力条件路径。
  • Mars physics 的 MPI/OMP 列分布还依赖 phy_common/mod_phys_lmdz_para、parallel-physics-grid-distribution 和 transfer 页面。
  • Venus/Titan 专属分支、evolution/PEM 和 COMMON 6.3 已删除的 dyn3d/nogcm.F90 不属于本目录页面范围。

复现清单

  1. 记录编译目标是串行还是并行；并行目标再记录 MPI ranks、OpenMP threads 和 omp_chunk。
  2. 读取 run.def 中 adjust/use_filtre_fft/use_mpi_alloc/offline/ngroup 等并行相关键。
  3. 检查 Bands_*prc.dat 是否存在；若不存在，确认 bands 的默认分布和 WriteBands 输出。
  4. 对某个 _p 算子定位问题前，先确认当前 SetDistrib 使用的是 caldyn、Van Leer、dissip 还是 physics band。
  5. 对极点或边界行差异，先查 pole_nord/pole_sud、exchange_hallo 和 mod_hallo 请求。
  6. 对 output/restart 差异，查 root rank、IOIPSL domain、局部切片和 gather 路径。
  7. 对 physics tendency 差异，同时查 calfis_p 动力侧映射和 phy_common transfer。
