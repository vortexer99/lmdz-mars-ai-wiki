文件定位

conf_gcm.F90 定义 SUBROUTINE conf_gcm(tapedef, etatinit)，是并行动力入口的运行配置读取中心。它通过 getin 从 run.def 等配置源读取日期、时间步、physics/tracer、restart、输出、耗散、sponge、zoom 和并行专用开关，然后直接写入 control_mod、logic_mod、comconst_mod、serre_mod、sponge_mod_p、mod_filtre_fft、mod_hallo 等模块全局状态。

与串行 dyn3d/conf_gcm 相比，本文件主体同构，但额外处理 OpenMP adjust 限制、并行 FFT filter 开关、MPI 全局 buffer allocation 开关，并使用并行 sponge 模块 sponge_mod_p。串并行差异的横向对照见 conf_gcm 串并行配置差异。

Mars 运行参与度

条件经过：Mars 并行动力构建或并行入口测试经过 `dyn3dpar/gcm.F -> conf_gcm(99,.TRUE.)`。串行 Mars 3D 正常入口使用 `dyn3d/conf_gcm.F90`；MARS dyn1d 不是本文件直接入口。

并行特有行为

行为 源码位置 影响
adjust OpenMP 限制 104-117 CPP_OMP 下进入 OpenMP parallel 区检查线程数；若线程数大于 1 且 adjust=.true.，调用 abort_gcm。
offline .AND. adjust warning 537-544 不停止运行，但提示 defstoke.nc/fluxstoke.nc/fluxstokev.nc 不会创建，只保留 phystoke.nc。
use_filtre_fft 590-596 并行版只读入 mod_filtre_fft:use_filtre_fft；不像串行版直接拒绝 true。源码注释说明此时 grossismx 尚未定义，zoom/FFT 兼容检查被注释。
use_mpi_alloc 607-615 写入 mod_hallo:use_mpi_alloc，由 halo 通信 buffer allocation 消费；细节见 mod_hallo。
sponge_mod_p 20, 414-440 使用并行 sponge 状态模块，配置键与串行 sponge 语义相同。

conf_gcm 自身不建立 MPI decomposition。并行分布链在 dyn3dpar/gcm 中发生：conf_gcm 之后才调用 init_const_mpi -> init_parallel -> Read_Distrib -> set_bands -> SetDistrib -> Init_Mod_hallo。因此排查并行配置时要先看本页的 getin 写入，再看 gcm.F 的 MPI/bands/halo 初始化。

Zoom 分支

etatinit 控制 zoom 处理方式：

分支 行号 行为
.not.etatinit 687-871 从 run.def 读 clon/clat/grossismx/grossismy/fxyhypb/dzoomx/dzoomy/taux/tauy/ysinus 到局部变量，并与当前 serre_mod/logic_mod 状态比较；差异超过 0.001 或布尔不一致时 STOP。
etatinit 873-970 直接初始化 serre_mod/logic_mod zoom 状态；默认 grossism*=1、fxyhypb=.TRUE.、dzoom*=0.2、taux/tauy=3、ysinus=.TRUE.，并派生 alphax=1-1/grossismx、alphay=1-1/grossismy。

Mars utility 的 defrun_new 另有 Mars 专用门控，特别是 ysinus=.false.、无 zoom 强制 fxyhypb=.false.、有 zoom 要求双曲映射；几何消费链见 inigeom zoom/map 几何契约。本页只记录 COMMON 并行 conf_gcm.F90 的源码行为，不把 Mars defrun_new 的默认值混入本文件。

核心流程

open lmdz.out if lunout is not 5/6
read adjust and validate OpenMP compatibility
read run length, date, restart, timestep, output and tracer controls
read dissipation, vertical profile, top-bound sponge and sponge_mod_p controls
read planet-specific defaults and physics/guide/start switches
read parallel-only use_filtre_fft and use_mpi_alloc
derive vert_prof_dissip default from planet_type/ok_strato
if etatinit:
    initialize zoom state in serre_mod/logic_mod
else:
    compare run.def zoom keys against already-loaded state
print effective configuration summary

串并行差异要点

  • 串行页 dyn3d/conf_gcm 记录相同主配置管线；并行页额外覆盖 adjust、use_filtre_fft、use_mpi_alloc 和 sponge_mod_p。
  • 串行版会拒绝 use_filtre_fft=.true.；并行版只读取该开关，留给并行滤波链消费。
  • adjust 在串行版不是关键路径；并行版在 OpenMP 多线程时直接 abort。
  • history 初始化不是本文件完成的：并行入口 gcm.F 后续只在 mpi_rank==0 建立 history/dynav 文件。
  • ngroup 是共享配置，不是并行专属；它约束极区经度分组，要求 iim 与 2**ngroup 兼容。

风险和待确认

  • tapedef 参数仍保留在签名中，但当前源码主要通过 getin；是否仍有外部旧式单元语义需结合更早版本或构建配置确认。
  • use_filtre_fft 与 zoom 的兼容检查在并行版被注释，源码注释称此时 grossismx 未定义；实际并行 FFT filter 与 zoom 同用时应通过运行测试或滤波页进一步确认。
  • use_mpi_alloc 的性能收益和平台兼容性不由本文件判断；这里只确认配置写入 mod_hallo。
  • adjust 的数值/物理意图不在本文件附近解释；本页只记录 OpenMP 约束和 offline warning。
  • 并行 Mars 是否为某个生产脚本默认入口需结合 makelmdz_fcm target 和实验脚本确认；源码层面只确认条件可达。
