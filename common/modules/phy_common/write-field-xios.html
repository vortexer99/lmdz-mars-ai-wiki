xios_writefield 流程

  1. generic xios_writefield 根据 rank 调用 2D 或 3D 版本。
  2. 检查字段是否已登记在 FieldName/FieldIt 表中；首次字段会创建 context_lmdz_<name>。
  3. root 读取 xios_writefield_nlon/nlat，广播到物理并行域。
  4. 初始化 XIOS context、D360 calendar、start/time origin 和 timestep。
  5. 根据 grid_type 选择 regular lon-lat domain 或 unstructured domaingroup。
  6. 对 3D 字段额外创建 vertical axis。
  7. 每次写出前 Gather_omp，regular grid 还调用 grid1Dto2D_mpi，最后 xios_send_field。

write_field_phy 和 misc/write_field

  • write_field_phy 是物理列到旧 WriteField 的桥：Gather(Field,New_Field) 后只在 root 把 1D 物理列转成全局 2D 网格。
  • misc/write_field.F90 管理字段表：若名字不存在则 CreateNewField，然后按 1D/2D/3D 维度写变量。
  • 文件名和变量名由字段名派生，int2str 支撑多字段编号。

复现要点

  • xios_writefield 的 field/context 是按变量名懒创建；第一次调用和后续调用逻辑不同。
  • unstructured grid 且 output_native_grid 为假时，field 的 domain_ref 会切到 regular_domain，让 XIOS 做插值输出。
  • wxios_domain_param 使用物理并行 gather/bcast 收集经纬度和 mask，再写入 XIOS domain。

待确认

  • 本地 COMMON 6.3 与 LMDZ.MARS 源码未见 xios_writefield 直接调用点；外部 physics 包或用户 patch 是否调用需另行确认。
