dyn3d_common 总览

路径

LMDZ.COMMON-6.3\LMDZ.COMMON\libf\dyn3d_common

职责

dyn3d_common 包含串行和并行动力共享的核心状态、初始化、网格几何、restart、垂直坐标、差分算子、输送内核和 I/O helper。当前目录有 123 个文件。

Mars 运行参与度：必经 到 接口兼容 混合。control_mod、comconst_mod、comvert_mod、infotrac 等由 MARS 直接转发或 USE；dynetat0、inigeom 被 MARS phymars/nogcm.F90、newstart.F、start2archive.F 等调用链使用。

典型流程

  1. conf_gcm 设置 control_mod、logic_mod 和耗散/网格控制变量。
  2. infotrac_init 读取 tracer 元数据，入口据此分配 tracer 数组。
  3. dynetat0 或 iniacademic 生成初始 ucov/vcov/teta/q/masse/ps/phis/time_0。
  4. iniconst、inigeom、inifilr 准备常量、几何和滤波数据。
  5. leapfrog/leapfrog_nogcm 调用 pressure、mass、operator、advection、diagnostic 和 restart helper 推进时间。

复现要点

  • 维度来自 dimensions.h 和 paramet.h，几何数组主要在 comgeom2.h 中共享。
  • dynetat0 读入的 start 控制数组会写入 comconst_mod、comvert_mod、temps_mod、serre_mod 等状态。
  • inigeom 会按 fxyhypb/ysinus 选择不同网格映射分支，复现时必须同时记录 zoom 参数。

待确认

  • 各 adv* 和 ppm3d/prather 文件在 Mars 具体 tracer 配置下的启用条件需由后续输送任务细化。
  • defrun.F 是否仍被某些旧工具链调用，当前源码内未发现 COMMON/MARS 主链调用。
