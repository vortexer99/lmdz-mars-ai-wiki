文件定位

dynetat0.F90 定义 SUBROUTINE dynetat0，从动力 restart/start NetCDF 文件读取控制数组、网格、几何、动力状态、tracer、气柱质量和地表压力。它同时根据 planet_type 区分 Earth-like、Titan 和 planeto-like/Mars start 文件布局，并处理 timestart 选择多时间片。

Mars 运行参与度

必经/条件经过：COMMON 6.3 `gcm` 和 MARS `phymars/nogcm.F90` 在 read_start 路径调用 `dynetat0("start.nc",...)`；MARS newstart/start2archive 也调用 dynetat0 读取起始状态。

共享状态与副作用

例程写入动力常量、垂直坐标、时间、能量和 zoom 相关 module 全局状态；读取失败或维度不符会 abort；内部读取函数会检查经度周期端点，不一致时打印 warning 并把最后经度点改为第一个经度点。

核心逻辑

  1. 打开 NetCDF start 文件，读取 controle 数组。
  2. 根据 planet_type 选择 Earth-like 或 planeto-like 布局：Mars/generic 使用 idecal=4，Earth/Titan 使用 idecal=5。
  3. 从 controle 解码网格维度、日期、行星常量、能量、垂直坐标和 zoom 参数。
  4. 对 Mars，把 hour_ini 从控制数组第 29 项读取；其他行星设为 0。
  5. 检查 im/jm/llm 与编译维度 iim/jjm/lllm 一致。
  6. 读取经纬度、cu/cv/aire/phisinit。
  7. 读取 temps 或回退读取 Time，根据 timestart 选择最后时间片或匹配时间片。
  8. Mars 路径把 day_ini/hour_ini/time0 重组为整数日加小数日。
  9. 读取 vcov/ucov/teta。
  10. 对每个 tracer，按 tname(iq) 读取；缺失时置零，Earth isotope 路径可做理论初始化。
  11. 读取 masse 和 ps，关闭文件。
  12. 非 Mars 路径将 time0 的整数部分累加到 day_ini。

伪代码

open start NetCDF
read controle
if planet_type is mars or generic:
    decode planeto layout
else:
    decode earth-like layout
write constants, dates, energy, vertical, zoom state
check file dimensions against compiled dimensions
read longitude/latitude and geometry fields
read temps or Time axis
select indextime from timestart or last value
adjust Mars day_ini/hour_ini/time0
read vcov, ucov, teta
for each tracer name:
    read tracer at selected time or initialize missing tracer
read masse and ps
close file

参与的主题流程

主题 参与方式
restart-io 读取动力 start.nc 初态，是 dynredem 写出 restart 的反向路径。
dynamics-time-integration 提供进入 leapfrog 的初始状态。
Mars start handling Mars 使用 planeto-like 控制数组布局，并处理 timestart 和 hour_ini。

写法特点

Fortran 90 子程序内含多个内部读取 helper。读取 helper 同时做周期端点修正，这意味着输出数组可能不是 NetCDF 原值的逐字节拷贝。

复现要点

  • Mars/generic 的 controle 解码偏移与 Earth-like 不同，不能直接按 Earth start 文件布局解析 Mars start。
  • timestart<0 默认选最后一个时间片；非负时必须在时间轴中找到误差小于 0.01 的时间。
  • 缺失 tracer 不一定 abort：会置零；Earth isotope 情况还可能按父 tracer 做简化初始化。
  • 经度周期端点不一致时，代码会强制 iip1 点等于第 1 点。

待确认

  • controle 数组各项单位除源码上下文外未全部显式声明，部分单位需结合写出端 dynredem 确认。
  • Mars start 文件中 hour_ini=tab_cntrl(29) 的历史约定需和 MARS start 生成器对照。
