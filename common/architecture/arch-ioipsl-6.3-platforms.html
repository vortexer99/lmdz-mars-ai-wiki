arch and IOIPSL 6.3 platform updates

页面状态：6.3 返工完成。 源码基线：LMDZ.COMMON-6.3\LMDZ.COMMON。 Mars 运行参与度：必经构建层。

新增 arch 平台

6.3 新增平台覆盖 ADASTRA、BAO_YGG、MESOIPSL、MESU、ROMEO 以及 gfortran parallel 配置：

arch-ADASTRA-gnu.{env,fcm,path}
arch-ADASTRA-ifort.{env,fcm,path}
arch-BAO_YGG.{env,fcm,path}
arch-MESOIPSL-gnu.{env,fcm,path}
arch-MESU-gnu.{env,fcm,path}
arch-X64_ROMEO_INTEL.{env,fcm,path}
arch-gfortran-para.{fcm,path}
arch-ifort_MESOIPSL.{env,fcm,path}

gfortran / NetCDF / IOIPSL 路径变化

arch-gfortran.path 的重要变化：

  • NetCDF 不再只靠 $NETCDF/lib 与 $NETCDF/include。
  • 改为使用 nc-config --cflags、nf-config --fflags、nc-config --libs、nf-config --flibs。
  • IOIPSL include/lib 从旧 ioipsl/modipsl/lib 风格转向 sibling ../IOIPSL/inc 和 ../IOIPSL/lib。

复现时如果旧脚本只设置 $NETCDF，6.3 gfortran 构建可能找不到正确 flags。

IOIPSL scripts

新增：

install_ioipsl_ADASTRA-gnu.bash
install_ioipsl_ADASTRA-ifort.bash
install_ioipsl_BAO_YGG.bash
install_ioipsl_ifort_MESOIPSL.bash
install_ioipsl_MESOIPSL-gnu.bash
install_ioipsl_MESU-gnu.bash

删除旧脚本：

install_ioipsl_centos-ifort.bash
install_ioipsl_ciclad-ifort.bash
install_ioipsl_mesu.bash
install_ioipsl_occigen.bash

SVN URL 从 forge.ipsl.jussieu.fr 系列迁到 forge.ipsl.fr 系列；部分脚本会把 IOIPSL 内部脚本 shebang 从 /bin/ksh patch 到 /bin/bash。

复现检查

  1. 先确认 arch 名是否属于 6.3 新增/修改平台。
  2. gfortran 构建要确认 nc-config 与 nf-config 可执行。
  3. IOIPSL 目录应按 6.3 ../IOIPSL/inc、../IOIPSL/lib 布局。
  4. 不再使用已删除的 centos/ciclad/occigen install 脚本。
  5. 保存 makelmdz_fcm --version 或 --version 输出用于构建溯源。
